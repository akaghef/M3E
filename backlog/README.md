# Backlog / Inbox

未整理の入力と、後続判断を待つ backlog の共通受け皿。

## 入力規則

- 行き先が未確定のメモ、調査候補、機能案は inbox としてまずここへ置いてよい。
- ファイル名と形式は固定しない。既存ファイルを上書きせず、新規ファイルとして追加する。
- 仕様、運用規則、idea、task など正規の置き場が確定したら、内容を該当先へ移し、ここに二重正本を残さない。
- 実行可能な TODO の正本は `docs/06_Operations/Todo_Pool.md`。backlog に未整理入力を置いても、確定 TODO として二重管理しない。
- 秘密情報、資格情報、private/public-danger material は置かない。
