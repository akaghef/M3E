第一に、構造の不足です。現行のAIは、与えられた文脈の中でもっともらしい次の出力を生成することには強い一方で、問い・仮説・根拠・反証・保留事項といった関係を、明示的な世界モデルとして内部に保持しているわけではありません。そのため、見かけ上は推論していても、実際には局所的な整合性をその都度作っていることが多いです。知的活動では、単に候補を広く試すだけでなく、何を固定し、何を変数とみなし、どの失敗から何を学んだかを構造として蓄積する必要があります。ここが弱いです。

第二に、評価の不足です。パラメータスイーピングは、評価関数が比較的明確であれば回ります。しかし、研究や設計のような知的活動では、何が良い問いか、どの仮説が有望か、どの枝を切るべきかという評価基準自体が未確定です。つまり、探索対象だけでなく評価軸も動く。このとき必要なのは、固定された目的関数ではなく、目的関数そのものを見直す能力です。現行のAIはここに弱く、結局は与えられた指示や表面的な最適化目標に従いやすいです。

第三に、主体性の不足です。知的活動では、探索の途中で問題設定そのものを書き換える必要があります。これは単なる試行回数の増加ではなく、「この問いは悪い」「この分解は粗い」「この変数は本質でない」と判断して、探索空間そのものを再設計することです。現行のAIは、基本的には与えられた枠内で答える機械であり、自分で問題の枠組みを作り替えることを安定しては行えません。できたように見える場合もありますが、多くは局所的な言い換えに留まります。

第四に、記号操作と意味理解の接続の弱さです。現行のAIは、言語的には滑らかに振る舞えますが、その背後で扱っている対象が、本当に一貫した対象として結ばれているとは限りません。研究では、ある記号が前の議論のどの定義に依存し、どの仮定を受け、どの例外で壊れるかを保ち続ける必要があります。ここが崩れると、AIは知的に見えても、実際には浅い組合せに留まります。

第五に、失敗の使い方が浅いことです。知的活動では、失敗は単なる不正解ではなく、モデル更新の材料です。どこで壊れたかによって、仮説空間の形が変わる。しかし現行のAIは、失敗を構造的に蓄積して次の探索方針を変えることが苦手です。そのため、何度も似たような誤りを別表現で繰り返しやすいです。これは、探索が学習に接続していないことを意味します。

そしておそらく、ここで重要なのは完全解を目指さないことです。あなたの言う通り、他のアーキテクチャも、複雑な知識横断を低コストかつ直観的に扱う問題をまだ十分には解いていません。ならば、最初に Tree を選ぶことは後退ではなく、制御可能な複雑性から始めるという戦略です。研究用 OS として重要なのは、最初から世界の全複雑性を表すことではなく、人間が継続的に使える骨格を先に作ることです。

言い切るなら、Tree は完全だから採るのではなく、不完全さが管理可能だから採る、です。そこから越境が必要になったときだけ、限定的に Network 性を足すのが自然です。
