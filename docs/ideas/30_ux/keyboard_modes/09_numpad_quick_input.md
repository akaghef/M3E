# 09. テンキー（Numpad）による属性高速入力

「カラーリング・緊急度・ステータスを一括でテンキーから入れたい」を起点に、
テンキー専用設計のメリット・配置案・運用パターンを整理する。

## なぜテンキーが特別か

通常の数字キー列（DigitX）と物理的に独立しており、ブラウザイベントでも
`event.code === "Numpad1"` のように別判定が可能。これにより:

- **既存ホットキー一切と衝突しない** — 横取り設計の必要がほぼ無い
- **左手はマウス／トラックパッド、右手はテンキー** の片手分業ができる
- **3×3 グリッド + 0** という空間配置がそのままメニューになる
- **テンキーだけで完結するモード**を作れる（モードに入る必要すらない可能性）
- **+ / - / * / / / Enter / .** も追加で使えて拡張性が高い

弱点:
- **ノートPCにはテンキーが無い** ことが多い → フォールバック必須
- **Mac の小型キーボードにも無い**
- 慣れない人には「数字＝意味」の対応学習がいる

## 用途別の配置案

### 用途A. カラーリング（10色プリセット）

```
NumLock OFF/ON 関係なく、Numpad 0〜9 を10色に固定:

  7: 紫     8: 青       9: 水色
  4: 緑     5: 黄緑     6: 黄
  1: オレンジ 2: 赤      3: ピンク
        0: グレー（無色化）

  +: 次のカラーパレットへ切替
  -: 透明度切替（はっきり / 薄く / 完全透明）
  *: 色を子ノードに伝播
  /: 色を消す（reset）
  .: 直前の色を再適用（リピート）
  Enter: 確定（soft-commit していた場合）
```

**配置の理由（候補）**:
- A1案: 上記のように **左下→右上で暖→寒** に並べる（直感的）
- A2案: マンセル色相環の順
- A3案: マップ内で頻度の高い色を中央寄せ（5,8,2,4,6 が頻用）
- A4案: ユーザがマップ毎にカスタマイズできる

### 用途B. 緊急度・優先度

**5段階案**:
```
  7: 緊急上位     8: 高い      9: 最高
  4: 中（中位）   5: 標準      6: 中（再）
  1: 低い         2: 様子見    3: いつか
        0: 優先度無し（リセット）

  +: 1段上げる
  -: 1段下げる
  *: 期限を「今日」にする
  /: 期限解除
  .: 直前の優先度をリピート
```

**9段階案**:
```
  数字 N をそのまま「優先度 N」に対応
  1=最低 〜 9=最高
  Enter: 確定
```

**配置の理由（候補）**:
- B1案: 物理位置と優先度を一致（上ほど高優先度＝直感的）
- B2案: 単純に 1〜9 = 優先度数値
- B3案: アイゼンハワー2x2マトリクスを Numpad の4象限に対応
  - 7: 緊急かつ重要
  - 9: 緊急だが重要でない
  - 1: 重要だが緊急でない
  - 3: どちらでもない

### 用途C. ステータス

```
  7: blocked   8: review   9: done
  4: paused    5: wip      6: cancelled
  1: backlog   2: todo     3: ready

  +: 次の状態へ進める（todo→wip→done）
  -: 前の状態へ戻す
  Enter: 状態履歴を確定
```

**配置の理由**:
- C1案: 右に行くほど完了に近い（左→右で進行）
- C2案: 上に行くほど止まっている（blocked が上）
- 列ごとに意味（左列=未着手系、中列=進行中系、右列=完了系）

### 用途D. 複合（用途A〜C を切り替えて使う）

テンキーは1セット9個しかないので、用途を切替える仕組みが必要:

| 切替方式 | 操作 |
|---------|------|
| D1. NumLock の状態で意味を変える | NumLock ON=色、OFF=ステータス（OS依存で罠） |
| D2. 直前にプレフィックスキー | `c` 押した直後 numpad → 色、`s` 直後 → ステータス |
| D3. モード切替 | 色モード / 緊急度モード を明示的に切替 |
| D4. 修飾キー | Numpad + Shift → 色、+ Ctrl → 緊急度、+ Alt → ステータス |
| D5. 連続2打 | Numpad 2回押で意味確定（5,5 = 緊急度5、3,5 = 色5、等）|
| D6. 用途固定 + メニュー切替 | 画面上のタブで用途切替、テンキーは現在用途のみ |

**それぞれの相性**:
- D1: OS と戦うことになるので非推奨
- D2: 最速、ただし「いま何モード」が見えないと迷子
- D3: モード概念と整合、見える化しやすい
- D4: 修飾キー併用で右手だけで完結しない（テンキーの強みを殺す）
- D5: 学習コスト高、が **モードレス** なのは魅力
- D6: 視認性◎、切替の物理操作が増える

**推し**: D2 + D3 のハイブリッド。デフォルトはモード固定だが、`c`/`s`/`p` で
プレフィックス付きの一発上書きも可能。

## バルク操作との統合

「**選択中の複数ノードに対して** テンキー一発で属性付与」が最高に強い。

### バルク選択 + テンキー

```
1. 選択モードで複数ノードを選ぶ（Shift+クリック / lasso / 検索結果）
2. テンキー押下 → 全選択ノードに即適用
3. 自動で次の selection にフォーカス、続けて入力可能
```

**派生1: ペイントブラシ式**
- テンキーで「現在ブラシ色」を確定
- 以後ノードをクリックすると即その色になる
- `Esc` でブラシ解除

**派生2: 進行型バルク**
- 選択中の集合に対して numpad 押下
- ノードが "完了" になると自動で集合から外れる
- 集合空になったら自動でモード抜ける

**派生3: スプレッドシート的な縦更新**
- Tab/Enter で次のセルへ移動するように、ノード間を移動
- 各ノードでテンキー1打 → 即次へ
- 「20件のステータスを5分で更新」みたいな業務に強い

## 視覚フィードバック

テンキーの価値は **見ずに打てること**。それを最大化する補助 UI:

| 案 | 良い点 | 悪い点 |
|----|------|--------|
| F1. 画面隅にテンキーレイアウト常時表示 | 学習中◎ | 場所取る |
| F2. 用途切替時の1秒トースト | 邪魔にならない | 見逃すと迷う |
| F3. ノード周囲に「選択肢9個」を3×3で重ねる | 直感的、テンキー位置と一致 | クリックも要る場合に競合 |
| F4. 押下時に画面端の対応マスが光る | 物理マッピング学習に最適 | 派手 |
| F5. 何も出さない（覚えるまで耐える） | 静か | 学習困難 |

**推し**: F2（切替トースト）+ F4（押下フィードバック）+ `?` で F1（チートシート）。

## ノートPCフォールバック

テンキー無しの環境では:

| 案 | 説明 |
|----|------|
| L1. ホームポジションキーをテンキー風に再マップ | `u i o / j k l / m , .` の3×3 |
| L2. Fn+数字 で代用 | 既存の数字キーを使う |
| L3. 専用パレット UI | テンキー機能をマウスで触れるパネル |
| L4. 機能無効化 | 「テンキー必須機能です」と諦める |

**推し**: L1 を「Numpad emulation mode」として実装。
テンキー有る人にも、左手で済ませたい時の代替として使える。
hjkl の3×3拡張版（`yuio / hjkl / nm,.`）もアリ。

## 連続操作のリズム設計

テンキーの強みは「リズム」。1秒1ノード処理を目標にする:

- 押下 → **即適用**（確認ダイアログ無し）
- 適用後 → **自動で次のノードへ**（Tab 不要）
- 直前操作は **常に undo 可能**（`*` か `Numpad-` あたりで）
- 同じ値を連打する場合は **`.` でリピート**

タイピング音を出すと「打てている感」が増すが、好みが分かれる。
オプション化推奨。

## 用途を超えた組み合わせ案

ユーザの「カラーリングや緊急度を一括で」は実は **複数属性を同時更新** したい
意図かもしれない。複合パターン:

| パターン | 操作例 |
|---------|------|
| P1. 色＝緊急度の視覚化 | 緊急度を入れると色も自動で変わる（連動） |
| P2. 1打で複数属性 | Numpad 5 = 「中優先・黄色・wip」のプリセット |
| P3. プリセット駆動 | 「会議メモ用」「タスク用」「アイデア用」の3プリセットを切替、各プリセットで Numpad 配置が違う |
| P4. レイヤー切替 | 色レイヤー / 優先度レイヤー / ステータスレイヤーをタブで切替、Numpad は常に「現在レイヤーの値」を入れる |

**P1（連動）** が最も「業務効率化」の趣旨に合う。
緊急度を入れる → 自動で赤くなる → マップを俯瞰すると赤い場所が緊急エリア → 二度手間ゼロ。

## オープンな論点

- **N1. 用途切替方式** — D1〜D6 のどれ、または組み合わせ
- **N2. 緊急度の段階数** — 5 / 9 / 任意 / 4象限
- **N3. 色と緊急度の連動** — P1 を採用するか、独立させるか
- **N4. NumLock 状態の扱い** — 無視 / 利用 / OS依存で危険視
- **N5. テンキー無しユーザのデフォルト体験** — L1〜L4 のどれ
- **N6. プリセットのカスタマイズ機能** — マップ毎 / グローバル / ロック
- **N7. 履歴の見せ方** — 「直近5回の操作」を画面端に出すか

## ミニマム実装イメージ

最小実装で「色か緊急度のどっちかだけ」テンキー対応するなら:

```
1. viewer.ts に numpad キーリスナを追加（`event.code` 判定）
2. デフォルト用途を「緊急度」に固定（最も業務効率に効きそう）
3. Numpad 1〜9 → 選択中ノードに priority=N をセット
4. Numpad 0 → priority クリア
5. Numpad . → 直前ノードに priority をリピート適用
6. 適用と同時に色（薄→濃）を attributes.color にも自動反映（P1連動）
7. 一括: 複数選択中なら全部に適用
```

これだけで「緊急度を見ながらマップ俯瞰できる」が実現する。
50〜100行程度で書けそう。色プリセット・モード切替などは後段。
