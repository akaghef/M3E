# Slideshow / Guided Tour for M3E

M3E マップ内で「順番に決定事項やノードをガイドツアー」する機能の設計ブレインストーミング。
レビュー保留のため、各論点について **複数の選択肢を並べ、決定はしない**。

## 目的（一言）

マップを「読む順序」を作者が指示し、視聴者が次へ／前へで辿れるようにする。
プレゼン用途・オンボーディング用途・意思決定レビュー用途を兼ねる。

## ファイル構成

- [01_concept.md](01_concept.md) — 何を作るか・想定ユースケース・既存ツールとの差分
- [02_authoring.md](02_authoring.md) — 「ツアー」を定義する方法（どこに順序情報を置くか）
- [03_runtime_ui.md](03_runtime_ui.md) — 再生時のカメラ挙動・UI コントロール・表示モード
- [04_data_model.md](04_data_model.md) — 永続化・エクスポート・他形式との互換
- [05_mvp_path.md](05_mvp_path.md) — 最小実装案と未決の質問リスト

## 整理されている論点

各ファイル末尾に「論点ID」を打ってある。MVP に進む段階で
ユーザーが論点ID単位で選択肢を選ぶ → 設計確定、という流れを想定。

論点一覧:

- A. ステップの粒度（ノード／ノード+カメラ／スライド本文）
- B. ツアー定義の置き場（マップ内ノード／マップ属性／別ファイル）
- C. 順序の決め方（DFS／手動配列／タグ番号／リンク鎖）
- D. 複数ツアーの管理（1マップ1ツアー／名前付き複数）
- E. カメラ挙動（中心+ズーム／フィット枠／パン演出）
- F. 周辺ノードの扱い（フェード／非表示／そのまま）
- G. UI 配置（オーバーレイ／フルスクリーン／サイドバー）
- H. ナレーション欄（details 流用／専用 note／markdown 本文）
- I. 操作系（矢印キー／ボタン／自動進行／リモート）
- J. 永続化先（マップ JSON 内／別 .tour ファイル／URL パラメータ）
- K. エクスポート（無し／PDF／動画／共有リンク）
