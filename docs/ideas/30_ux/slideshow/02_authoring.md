# 02. オーサリング（ツアーの定義方法）

「どのノードを、どの順番で見せるか」をどこに、どう書くか。
ここが最も判断を要する論点。

## 論点B. ツアー定義の置き場

### B1. マップ内の「ツアー専用ノード」配下に並べる

```
root
├─ strategy
├─ research
└─ tour:advisor-2026Q2     ← 専用ノード
   ├─ → links to: n_xxx (Step 1)
   ├─ → links to: n_yyy (Step 2)
   └─ → links to: n_zzz (Step 3)
```

- 良い点: 既存のリンク機構（`links` フィールド）で表現可能、UI 上も見える
- 悪い点: マップが「ツアー用ノード」で散らかる、リンクの順序保証が必要

### B2. 各ノードの `attributes` にツアー番号を打つ

```
node A: attributes = { "tour": "1" }
node B: attributes = { "tour": "2" }
node C: attributes = { "tour": "3" }
```

- 良い点: 既存ノードに後付けが超軽い、複数ツアーは `tour:advisor`, `tour:onboarding` で名前空間化
- 悪い点: 同じノードを複数ツアーで使うと属性が肥大、順序の入れ替えが面倒（番号振り直し）
- 緩和: 番号は `010, 020, 030` のように間隔を空ける慣習にする

### B3. マップ JSON の `state` に `tours: { name: [nodeId...] }` を追加

```json
"tours": {
  "advisor-2026Q2": ["n_xxx", "n_yyy", "n_zzz"],
  "onboarding": ["n_root", "n_strategy", "n_done"]
}
```

- 良い点: ツアー定義がノードと完全分離、順序入れ替え自由、複数ツアー自然
- 悪い点: マップ UI 上では見えない（編集するには専用エディタが必要）
- 補足: ノード削除時に dangling reference が発生 → クリーンアップ処理が必要

### B4. 別ファイル `*.tour.json` として独立

- 良い点: マップ本体に変更を加えず実験可能、Vault でも .md ペアで持てる
- 悪い点: ファイル管理が増える、マップとの整合性チェック必須

### B5. details/note 内マークダウンに `<!-- tour:1 -->` のようなコメント

- 良い点: マップを汚さない、grep 可能
- 悪い点: パースが面倒、見えないので作者が混乱する

### おすすめ組み合わせ

- **MVP**: B3（state に tours フィールド）+ B2（attributes 番号）どちらか
- **本命**: B3。理由は順序入れ替え・複数ツアー・dangling 検出のしやすさ
- **B1 は UI が育ってから** 「ツアー編集モード」として表に出す形が綺麗

## 論点C. 順序の決め方

ツアー定義 = 単なる nodeId の配列、と単純化した場合でも
**配列をどう生成するか** にいくつか選択肢がある。

| 方式 | 入力 | 例 |
|------|------|----|
| C1. 完全手動 | ノードIDを並べる | 配列直書き |
| C2. DFS 自動 | ルートノードと深さ | strategy 配下を pre-order |
| C3. タグ番号順 | 属性の `tour` 値 | 1,2,3,... を昇順 |
| C4. リンク鎖 | 始点ノード + `next` リンク | A→B→C と辿る |
| C5. 検索クエリ | 条件式 | `status=done` 順序は created_at |

- C1: 自由度最大、編集 UI が必要
- C2: 既存マップから「読む順」を生成する初心者向け、手動微調整は別途
- C3: B2 と相性、付箋感覚で順序付与
- C4: 物語的（フローチャート的）に作りたいときに自然、リンク機能と重複しない
- C5: 動的ツアー（「今週 done になったタスク」を毎週自動生成）に強い

複数方式を共存させて良い。例えば「default = C2 で生成、上書きは C1」みたいな。

## ツアー編集 UI の選択肢（実装は後）

- E1. ノード右クリックメニュー「ツアーに追加」→末尾に append
- E2. サイドパネルにドラッグ＆ドロップ並び替え可能なリスト
- E3. ホットキー: ノード選択中に `T+1`〜`T+9` で番号付与
- E4. 「ツアー編集モード」を切り替え、地図上のノードクリック順に追加

これらは段階導入。最小は E1。
