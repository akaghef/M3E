# 03. 再生時の UI とカメラ挙動

ツアー再生中の見た目・操作・演出。

## 論点E. カメラ挙動

| 方式 | 動き | 良い点 | 悪い点 |
|------|------|--------|--------|
| E1. 中心+ズーム | 各ノードを画面中央に置き、固定ズーム | 実装最小、既存 `centerOnNode` 流用可 | 文脈が見えない |
| E2. フィット枠 | ノード+子孫を含む矩形にフィット | 文脈付き、サブツリーレビューに強い | フィット計算と演出が必要 |
| E3. パン演出 | 前ノード→次ノードを補間移動（300ms） | 連続性が伝わる、迷子を防ぐ | 遠距離だとめまい、計算重い |
| E4. ズームアウト→パン→ズームイン | ジオブラウザ風 | 全体地理感、印象的 | やりすぎると疲れる |
| E5. 静止＋ハイライトのみ | カメラ動かさず、ノードを光らせる | 軽い、聞き手の視線追跡が容易 | 大マップでは見つけにくい |

- 既存コード: `centerOnNode(nodeId, zoom)` がある（viewer.ts:6148）→ E1/E3 は流用容易
- **推し**: デフォルト E3（パン）+ Shift で E1（瞬時）の選択。E2 は将来のオプション

## 論点F. 周辺ノードの扱い

| 方式 | 効果 |
|------|------|
| F1. そのまま | 編集中のレビューに最適、地図感が残る |
| F2. 暗くフェード（opacity 0.3） | スポットライト効果、発表向け |
| F3. ステップ対象 + 直接の親子のみ表示 | フォーカスドラスティック、ストーリー強い |
| F4. 完全非表示でカードだけ表示 | スライド化、マップを見せたくない発表向け |

F2 が無難。F4 は「もはやマップを使っている意味がない」のでスコープ外候補。

## 論点G. UI 配置

| 方式 | 説明 | 用途 |
|------|------|------|
| G1. 下部オーバーレイバー | 進捗バー + Prev/Next + 注釈テキスト | 編集中レビュー |
| G2. フルスクリーン takeover | 全 UI 隠して F キーでスライドモード | 発表 |
| G3. サイドパネル（右） | 注釈・進捗・サムネイル | 講義動画録画 |
| G4. PiP（小ウィンドウ） | コントロールだけ浮かべて他は触れる | デモ・編集併用 |

ユースケース毎に切り替えられる方が綺麗。MVP は G1 のみ。

## 論点I. 操作系

| 方式 | 入力 | 備考 |
|------|------|------|
| I1. 矢印キー Right/Left | 標準 | 既存 hotkey と衝突注意 |
| I2. Space / Shift+Space | プレゼン慣習 | 既存と衝突する可能性あり |
| I3. PageUp/PageDown | プレゼンリモコン互換 | 物理リモコン対応 |
| I4. 画面上 Prev/Next ボタン | 触る／視聴 | タッチ対応 |
| I5. 自動進行（N秒ごと） | 無人デモ | ループも可 |
| I6. 番号ジャンプ（1〜9） | 高速移動 | 番号表示が必要 |

既存 viewer.ts のホットキーと衝突しないものを優先。
- スライドショー起動中はモードに入れて、衝突するキーを横取りする方針が安全

## 論点H. ナレーション欄

各ステップに「説明文」を表示する場所と書く場所。

| 方式 | データ源 | 表示場所 |
|------|---------|---------|
| H1. ノードの `details` をそのまま | 既存フィールド流用 | 注釈バー |
| H2. ノードの `note` 専用 | 既存だが用途分け | 注釈バー |
| H3. 専用フィールド `tourNote` | スキーマ追加 | 注釈バー |
| H4. ツアー定義側に注釈を持つ | `tours[name].steps[i].note` | 注釈バー |
| H5. なし（ノードテキストだけ） | 何もしない | — |

- H1: 既に details にメモを書く文化があれば最短。details が技術的なら不向き
- H4: 同じノードでもツアー毎に異なる注釈が書ける（一番表現力高い）が、編集 UI が要る
- **推し**: MVP は H1（既存流用）、表現力が足りなくなったら H4 に進化

## ツアーモードの状態遷移

```
[idle] --start--> [playing] --next--> [playing]
                  | pause                ^
                  v                      |
                [paused] --resume--------+
                  | exit
                  v
                [idle]
```

- 状態は viewer.ts のグローバル state に1つ追加するだけで済むはず
- exit 時にカメラ位置を「開始前」に戻すか、現在ステップ位置を保つかは選択肢
  - **推し**: 戻す（編集に戻りやすい）+ Shift+Esc で「現在位置で抜ける」
