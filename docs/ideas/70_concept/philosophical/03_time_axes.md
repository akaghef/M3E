# Q3. 複数の時間軸（線形 vs 円環 vs 結晶）

研究者の体感時間は単一の線形タイムラインで表現できない。
**異なる時間メタファ** をマップに重ねる仕組みをブレストする。

time_history I3 差分アニメ / I5 履歴 AI 要約 / cognitive_science J3 Compounding と接続。

## なぜ複数時間軸か

- 線形時間（締切・カレンダー）と思考の時間（湧きたい時に湧く）は別物
- 季節・年次イベント（年度末・学会）は **円環** で繰り返す
- 理論研究の概念は **結晶** のように並列に育ち、特定タイミングで結合する
- "5 年前の自分" と "明日の予定" が同じタイムラインで並ぶのは違和感
- automation_obstacles N14「待つ時間」は線形時間の隙間に存在する

## ユースケース

- **U1.** 「学会が近づくと書く」研究者が、年次サイクル view で前年の同時期を見る
- **U2.** 理論研究者が、独立に育ててきた 3 つの概念がある朝に "結晶化" する瞬間を可視化
- **U3.** 締切ベースで動く実験系研究者が、線形タイムラインを主に使う
- **U4.** 学生が「指導教員のリズム（学期サイクル）」と自分のリズムを重ね見る
- **U5.** 5 年プロジェクトで「いま何年目」を直感的に把握する
- **U6.** 過去の自分の "考え始めた瞬間" を辿りたい（線形以外の探索）
- **U7.** 「論文が腐る前に出す」タイミング感覚を視覚化する

## 3 つの時間メタファ

### 線形時間（Linear / 締切時間）

- 性質: 単調増加、不可逆、客観的
- 用途: 締切・スケジュール・実験ログ・履歴再生
- メタファ: タイムライン・ガントチャート・カレンダー
- 既存接続: time_history I1 タイムトラベル, audit_log.ts

### 円環時間（Cyclic / 季節時間）

- 性質: 周期的、繰り返し、リズム的
- 用途: 学期/年度/学会サイクル、習慣、レビュー周期
- メタファ: 時計盤・年輪・Mandala・Wheel of the Year
- 既存接続: personal_productivity G1 デイリー儀式, G2 週次レビュー

### 結晶時間（Crystalline / 概念時間）

- 性質: 非線形、並列成長、突然の結合
- 用途: 理論的概念の育成、長期テーマ、突然のアハ体験
- メタファ: 結晶構造・星座・銀河・キノコ菌糸
- 既存接続: thought_physics P1 物理エンジン, automation_obstacles N7 創発の場

## サブ論点

### Tx1. UI メタファの選択肢

| 案 | 表現 | 強い時間種 | 実装難度 |
|---|---|---|---|
| Tx1.a | 横スクロールタイムライン | 線形 | 低 |
| Tx1.b | カレンダー grid（年×月） | 線形+円環 | 中 |
| Tx1.c | 時計盤 / Wheel | 円環 | 中 |

### Tx6. 「時計が複数ある」 UX

時計を 3 つ並べる比喩:

- 左上: 線形クロック（今日の日時 + 締切まで N 日）
- 中央: 円環クロック（今は年度のどこか / 学期のどこか）
- 右上: 結晶クロック（並走テーマの熟成度ゲージ）

## 反対意見

- **R1.** 1 つの時間軸でも頭がパンクする。複数は混乱
- **R2.** 結晶時間は思想的すぎて使いどころが不明
- **R3.** 円環時間はカレンダーで十分
- **R4.** 時間メタファの実装コストが高い
- **R5.** 結局見るのは線形だけになる

→ 答え:
- R1 → デフォルトは線形、円環/結晶はオプション
- R2 → 長期プロジェクト（半年以上）でのみ価値が出る → 全員必要ではない
- R3 → カレンダーは "予定" 表示に偏る、研究の "リズム" 視覚化はマップ独自
- R4 → Tx1.b/Tx1.c は中難度。Tx1.e/Tx1.f は遅らせる
- R5 → それで構わない。結晶は意識的選択時のみ

## 既存ツール差分

| ツール | 時間軸の扱い |
|---|---|
| Notion カレンダー | 線形のみ |
| Google Calendar | 線形 + 弱い円環（毎週繰り返し） |
| Gantt ツール | 線形のみ |
| Roam daily note | 線形（日付ベース） |
| Anki | 円環（復習サイクル）に特化 |
| M3E | 線形 + 円環 + 結晶の重ね合わせ（差別化） |

## 半年実用化への寄与

- **短期**: 線形タイムライン view を 1 つ作るだけで「思考の時間」可視化が始まる
- **中期**: 円環 view で年次サイクル理解 → 助成申請の "毎年同じこと" を効率化
- **長期**: 結晶 view で 5 年プロジェクト全体感 → 大型助成申請書の物語構築に効く

## キーメッセージ

- **時間メタファは認識フレーム。** 同じデータでも見方で意味が変わる
- **デフォルトは線形・拡張で円環/結晶。** 哲学を強要しない
- **「待つ時間」を肯定する仕組み** は M3E 独自の差別化点
- **3 メタファの併存** は実装難度より UX 設計が重い（混乱回避がカギ）
