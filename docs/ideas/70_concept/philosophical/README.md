# 哲学的・実験的テーマ（Philosophical / Experimental）

00_topic_pool.md カテゴリ Q（横断的・哲学的テーマ）を素材に、
M3E の **思想的バックボーン** となる 5 サブトピック（Q1〜Q5）を網羅展開する。

哲学のままで終わらせず **「実装可能性」を必ず並列に置く** ことを原則とする。
project_projection_vision（半年で「世界モデル→射影」実用化）への寄与は
"短期：実装に効く小機能 / 中期：体験設計 / 長期：思想ブランド" の 3 層で評価する。

## 方針

- 採否判断はしない、複数案を並べる
- 実装詳細は最終ファイルでのみ参考レベルに留める
- 哲学命題と **「研究者がいつ嬉しいか」** をペアで書く
- 既存事例（automation_obstacles N7/N9/N14/N8、time_history I4/I5、
  thought_physics、cognitive_science）と **接続点** を必ず示す
- 思想原理主義を避ける — 「実用優先 / 哲学はバックボーン」を逸脱しない範囲

## ファイル構成

|---|---|---|---|---|

| Q4 | 自己発見 | マップは "鏡" — 自分が知らない自分を映す装置 | マップは作業台に過ぎない | 中（メタ層 / AI 視点） |
| Q5 | ノード死と再生 | 死を仕組み化することで生も意味を持つ | 全ノードは平等に永続 | 中（ライフサイクル設計） |

## 5 つの相互関係

```
                Q1 未完成原則 (常に開いた構造)
                       │
                       ↓
    Q2 忘却 ────────── Q5 死と再生 ────── Q4 自己発見
   (能動的見ない化)   (ライフサイクル)    (鏡として読む)
                       │
                       ↓
                Q3 時間軸 (線形/円環/結晶 = 認識フレーム)
```

- **Q1 は基盤思想** — 残る 4 つは Q1 の "未完成性" を異なる軸（時間・忘却・死・自己）で具体化したもの
- **Q2/Q5 は連続体** — Q2 は「忘れる」ソフト、Q5 は「死ぬ」ハード。閾値の違い
- **Q3 は認識フレーム** — Q1〜Q5 全体が「どの時間で見るか」で意味が変わる
- **Q4 は出口** — Q1〜Q3/Q5 の仕掛けの結果として「気づき」が生まれる

## キーメッセージ（先取り）

- **哲学は機能を選別する物差しになる。** 「これは Q1 に反するから採用しない」
  と言える基準として使うと、機能爆発を抑える効果がある
- **「実用優先・哲学バックボーン」のため、各テーマは "1 機能でも実装に降ろせる"
  最小翻訳を必ず添える。** 思想だけのブレストにしない
- **Q1 の "未完成" は M3E の差別化軸になり得る。** Notion/Obsidian は
  「整理ツール」であり、M3E は「未整理を価値化するツール」と位置付けると独自性が立つ
- **Q3 の時間軸は研究者にとって既知の問題。** "実験の時間（線形）" "理論の時間（結晶）"
  "生活の時間（円環）" を分けて表示できると、研究のリズム理解が大きく進む
- **Q4/Q5 は "マップとの長期関係" を設計する。** 短期 UX ではなく
  「3 年後にこのマップを見て何を感じるか」を設計対象にする
