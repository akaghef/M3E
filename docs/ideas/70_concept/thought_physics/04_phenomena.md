# 04. 創発現象 — 気象予報・地震・季節・生態系

物性（02）と挙動（03）の上に立ち上がる **集団的・時間的現象**。
本ファイルは「個別ノードを越えた、マップ全体に起こる出来事」を扱う。

メタファとしては最も派手で、ユーザー価値も高い。
一方で **誤検出のコスト** が高いので閾値設計が肝。

## P1. 気象予報（今日のマップ動向）

### P1.1 気象の構成要素
| 要素 | 計算 | 例 |
|------|------|-----|
| 気温 | 全ノード平均温度 | 高い=活発、低い=停滞 |
| 湿度 | 未決ノード比率 | 高い=モヤモヤ、低い=スッキリ |
| 風向 | 編集の重心移動方向 | "東から西へ" = 関心が移っている |
| 風速 | 単位時間の編集量 | 強い=活発 |
| 降水確率 | "新規ノード発生" の予測 | 高い=アイデア降ってきそう |
| 雷注意報 | 構造変化の予兆 | リンク変動兆候 |

### P1.2 予報の表示形式
- P1.2.a: 天気アイコン（晴/曇/雨/雪/雷）+ 一行コメント
- P1.2.b: 天気図（マップ上に等温線・等圧線オーバレイ）
- P1.2.c: 文章レポート（"今日の研究天気予報"）— AI 生成
- P1.2.d: 音声ナレーション（朝のラジオ風）

### P1.3 配信タイミング
| ID | タイミング | 用途 |
|----|------------|------|
| P1.3.a | 朝 1 回（デイリー） | "今日何やる" の意思決定 |
| P1.3.b | マップを開いた時 | 久しぶりに開いた時の状況把握 |
| P1.3.c | 週次サマリ | retro 用 |
| P1.3.d | リアルタイム（常時更新） | 気が散る可能性、要オフ機能 |

### P1.4 予報の信頼度
- データ量が少ないと "予報不可"
- 信頼度を % で添える
- 外れた予報を後日比較表示（"先週の予報的中率"）

## P2. 地震（突然の構造変化検出）

### P2.1 地震の発生条件
- 短時間（例: 30分以内）に閾値以上のノード／リンク変動
- 大きなサブツリーの再編
- 多数ノードの status 一括変更
- 重要ノードの削除

### P2.2 マグニチュード（規模）算出
| 要素 | 寄与 |
|------|------|
| 変動ノード数 | 比例 |
| 影響範囲（接続グラフの広がり）| 大きいほど高M |
| 重要度の総和 | 重要ノードが動くほど高M |

→ M1〜M9 のような直感的尺度を用意

### P2.3 地震イベントの記録
- 震源（最初に動いたノード）
- 震度マップ（影響を受けたノードの度合い）
- 発生時刻
- AI が "原因仮説" を生成

### P2.4 通知設計
- 軽微（M1-3）: 履歴に記録のみ
- 中規模（M4-6）: バッジ表示
- 大規模（M7+）: 即通知 + ナレーション

### P2.5 地震履歴ビュー
- 過去の地震を時系列で見る
- "あの日の研究の転換点" を可視化
- I4（意思決定の履歴）と接続

## P3. 季節（中長期トレンド）

### P3.1 季節の定義
- 春: 新規ノード生成多、温度上昇
- 夏: 全体的に熱い、活発
- 秋: 整理・収束フェーズ、冷却開始
- 冬: 停滞、化石化進行

### P3.2 季節の判定アルゴリズム
- 過去 N 日の平均統計から推定
- ユーザーの "学期" "プロジェクト期" と相関する想定

### P3.3 季節 UI
- 背景色や枠線の季節テーマ
- 朝ログイン時のメッセージ "あなたの研究は今 "晩秋" です"
- 季節ごとに推奨アクションを AI が提案

### P3.4 過剰演出のリスク
- "ずっと冬" だとモチベ下がる
- ポジティブ寄りに翻訳する仕掛け（"冬は次の春の準備期"）

## P4. 生態系（ノード種の共存）

### P4.1 種の概念
- ノード type を "種" として扱う
- 同種 = 群れ、異種 = 共生／競合関係
- 例: question 種 と answer 種 は共生（互いに増やし合う）

### P4.2 種の動態
- 種の人口推移（過去 N 日の生成数）
- 種ごとの絶滅危惧（最近全く生まれていない type）
- "外来種" 検出（新しく登場した type）

### P4.3 生態バランス指標
- 多様性指数（Shannon 等）
- "task ばかりで question が少ない" のような偏り検出
- 健康な研究は多様性が高い、という仮説

### P4.4 視覚化
- 円グラフでの種構成
- 時系列での種人口グラフ
- マップ上で種ごとに色分け

## P5. 火山・噴火（爆発的アイデア生成）

### P5.1 火山の定義
- 短時間に大量の子ノードを生んだ親ノード
- "ブレストの種" になったノード

### P5.2 噴火イベント
- 1時間で 5+ ノードを子に持ったら "噴火"
- マップに "火山" マーク表示
- 後日振り返り材料

### P5.3 マグマ溜まり（休火山）
- 過去に噴火した火山ノード
- 再加熱されると再噴火の予兆
- "休火山リスト" として AI が提示

## P6. 化石化（凍結ノードの保存と発掘）

### P6.1 化石化の条件
- 温度が下限到達
- N ヶ月以上未触
- archive と異なり、マップ上に残る（"地層に埋もれる"）

### P6.2 化石化の表現
- 灰色・半透明
- 専用「化石」アイコン
- 周囲の物理から除外（軽量化）

### P6.3 発掘ビュー
- "発掘モード" で地層を掘る UI
- 古い化石を一覧
- 発掘したものは即解凍可能

### P6.4 化石の楽しみ
- "5年前のノード" としての懐かしさ
- 当時の文脈（周辺ノードの温度・地震履歴）も同時表示
- I1（time travel）と接続

## P7. 通知の頻度・トーン設計

創発現象は通知しないと意味がない。だが過剰通知は離脱を生む。

### P7.1 通知頻度
| ID | 頻度 | 適用 |
|----|------|------|
| P7.1.a | 1日1回（朝） | 気象予報 |
| P7.1.b | イベント駆動 | 地震・噴火 |
| P7.1.c | 週次 | 季節変化 |
| P7.1.d | 月次 | 生態系レポート |

### P7.2 通知トーン
- ニュートラル（"温度が下がりました"）
- 文学的（"あなたの研究マップは晩秋を迎えました"）
- ユーモア（"地震警報！3章で大規模再編が発生中"）
- 研究者向け事実ベース（"30 nodes affected, M5.2"）

### P7.3 通知のオフ／カスタマイズ
- 種別ごとにオン／オフ
- "週次以下のみ" モード
- 完全オフ（履歴は残す）

### P7.4 通知 UI 配置
- マップ上のバッジ
- ヘッダーの天気アイコン
- 別パネル（ログ風）
- システム通知（ブラウザ／デスクトップ）

## P8. 創発現象の組み合わせ

複数現象が組み合わさるとさらに強い物語が生まれる。

| 組み合わせ | 物語 |
|------------|------|
| 地震 + 季節転換 | "晩秋に大地震、構造の冬支度" |
| 噴火 + 高温多湿 | "活発期にアイデア爆発" |
| 寒冷化 + 化石増加 | "氷河期、整理日推奨" |
| 風向反転 + 種の入替 | "研究テーマがシフト中" |

## 横断観察

- **P1（気象予報）は最も投資対効果高い** — 朝の意思決定支援は実用直結
- **P2（地震）はメタ認知価値が高い** — "何が起きたか" を後から振り返れる
- **P6（化石化）は感情的価値** — 楽しさを出すフックとして良い
- **P3（季節）は外しやすい** — 短期スパンの研究では合わないので、デフォルト OFF が安全
- **P7（通知設計）が成功失敗を分ける** — 過剰通知 = 即離脱
- **AI ナレーションが現象を "物語化"** — 単なるデータ表示から体験に昇華
