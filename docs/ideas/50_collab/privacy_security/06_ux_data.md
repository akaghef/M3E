# 06. UX・データモデル — 体験設計と永続化の選択肢

技術的に暗号化されていても、UI で見えなければ研究者は確信を持てない。
UX 選択肢とデータモデル選択肢を並べる。

## U1-U6. 機微度の視覚表示

ノードの機微度を **一目でわかる** ようにする UI 候補:

| ID | 表示方法 | 良い点 | 悪い点 |
|---|---|---|---|
| **U1** | 南京錠アイコン | 既存 UI 規範 | 状態数（鍵あり/なし）少ない |
| **U2** | ノード色帯（左端線） | 5段階色で表現可 | 色覚多様性配慮要 |
| **U3** | ノード背景色 | 強い視覚 | 文字読みづらく |
| **U4** | 半透明 / blur | 非アクティブ時に隠す | 目に優しいが説明的でない |
| **U5** | バッジ表記 (`S4`) | 明示的 | UI 雑然 |
| **U6** | アイコン色違い | 鍵の色で区別 | 解像度低い |

組合せ案:
- **基本**: U2 色帯 + U1 南京錠（公開可ノードは鍵なし）
- **強化**: 上記 + U4 blur on focus loss（離席時自動）

### 色帯の色配分（推し）

| 機微度 | 色 | 補助 |
|---|---|---|
| S1 公開 | なし（背景同色） | アイコンなし |
| S2 準公開 | 薄い緑 | 小さい点 |
| S3 内部 | 黄 | 注意マーク |
| S4 機密 | オレンジ | 鍵アイコン |
| S5 極秘 | 赤 | 鍵 + blur |

## U7-U12. 操作系 UX

| ID | 操作 | 内容 | 良い点 | 悪い点 |
|---|---|---|---|---|
| **U7** | unlock 期間 | 復号セッションの有効期限 | バランス | 中断時の再認証 |
| **U8** | 再認証頻度 | 高機微度操作前の追加認証 | 安全 | 確認疲れ |
| **U9** | panic ロック | キー一発で全画面ロック | 即時防御 | 誤打のリスク |
| **U10** | readonly モード | 高機微度時は編集禁止 | 誤操作防止 | 編集できず不便 |
| **U11** | 共有禁止フラグ | export / share 時に強制除外 | 漏洩防止 | フラグ忘れリスク |
| **U12** | 機密フォーカス | 機密ノード以外を完全フェード | 集中・周囲対策 | 操作しづらい |

### unlock セッションの選択肢（U7 詳細）

| 案 | 期間 | シーン |
|---|---|---|
| **U7-a 30秒** | 短い | 1ノード見たら閉じる |
| **U7-b 5分** | デフォルト | 通常の作業ブロック |
| **U7-c 1時間** | 長い | 集中作業時 |
| **U7-d セッション** | タブ閉じるまで | 作業中ずっと |
| **U7-e 操作なし○分** | 動的 | 自動ロックタイマー |
| **U7-f 機微度別** | S4 は5分、S5 はノード閉じたら | きめ細かい |

推し: **U7-e + U7-f**（離席自動ロック + 機微度別）

### panic ロック（U9）の発動方式

| 方式 | 良い点 | 悪い点 |
|---|---|---|
| ホットキー（`Cmd+Shift+L`） | 速い | 誤打 |
| タブ非アクティブ | 自動 | 切替で頻発 |
| マウスマウントの隅へ | 直感 | 誤発動 |
| 音声「隠して」 | 自然 | プライバシー的に矛盾 |
| 顔認証離脱 | 完全自動 | カメラ常時必要 |

## U13-U18. AI 送信前 UX

| ID | UI | 良い点 | 悪い点 |
|---|---|---|---|
| **U13** | 送信プレビュー | 「送る内容はこれ」表示 | 透明 | 操作1段増 |
| **U14** | マスク差分表示 | 「ここをマスクして送ります」赤字表示 | 教育的 | 表示混雑 |
| **U15** | dry-run モード | 「送信したらこうなる」シミュレーション | 安心 | 操作多 |
| **U16** | 機微検出時赤枠 | プロンプト入力欄が機微検出で赤 | 即時 | 集中阻害 |
| **U17** | LLM 切替インジケータ | 「いまローカル LLM」を常時表示 | 透明 | 画面占有 |
| **U18** | 送信履歴 (audit) | 過去の送信内容を見られる | 事後追跡 | 保存量増 |

組合せ: **U13 + U14 + U17** が最低ライン

## D1-D8. データモデル選択肢

機微度・暗号化情報をどこに持たせるか:

| ID | 場所 | 内容 | 良い点 | 悪い点 |
|---|---|---|---|---|
| **D1** | node.attribute | `node.attrs.sensitivity = "S4"` | 既存機構 | 暗号化対象との分離曖昧 |
| **D2** | node.metadata | 別フィールド `node.meta.sensitivity` | 概念明確 | スキーマ拡張 |
| **D3** | sidecar file | 別ファイルでメタ管理 | クリーンセパレーション | 同期が複雑 |
| **D4** | vault container | vault 単位で機微度継承 | 階層的 | ノード単位 override 必要 |
| **D5** | attribute + vault | 二重管理 | 柔軟 | 不整合リスク |
| **D6** | 別 store（IndexedDB） | DB 上のフラグ | パフォーマンス | sync と分離 |
| **D7** | node.tag | `#S4` タグで表現 | 既存タグ流用 | UI 雑 |
| **D8** | id prefix | `secret_xxx` ID で表現 | 検索容易 | リネーム不可 |

推し: **D2 + D4 の組合せ**（vault でデフォルト、ノードで override）

## D9-D14. 暗号化データの構造

暗号化されたノード本体をどう持つか:

| ID | 構造 | 内容 | 良い点 | 悪い点 |
|---|---|---|---|---|
| **D9** | inline | `node.text = "encrypted:base64..."` | シンプル | json サイズ膨張 |
| **D10** | 別フィールド | `node.text_encrypted, node.text` 二重 | 互換性 | 平文側がうっかり |
| **D11** | 別ファイル | `secrets/<node_id>.bin` | 漏洩独立 | 同期複雑 |
| **D12** | 暗号化 segment | `node.text` 内に `[ENC:...:END]` 埋込 | フィールド単位 | パーサ必要 |
| **D13** | 完全別 vault | secret vault 自体がファイル分離 | 防御強度高 | 既述 V5 |
| **D14** | DB 暗号化 | SQLite EAR 等で透過暗号 | 実装楽 | 外部依存（policy_privacy 違反） |

推し: **D9 inline + D11 別ファイル の使い分け**（ノード単位は D9、機密 vault は D11）

## D15-D20. 互換性

| ID | 観点 | 内容 | 選択肢 |
|---|---|---|---|
| **D15** | 既存 m3e-map.json 互換 | 機微情報なしユーザーで動くか | 互換維持 / 破壊的変更 |
| **D16** | backup 互換 | backup ファイルが暗号化を保てるか | 同等暗号 / 平文 backup |
| **D17** | export 互換 | Markdown / JSON export 時の扱い | 暗号化排除 / マスク export / 拒否 |
| **D18** | 別端末同期互換 | 鍵同期の方法 | 手動 / QR / 紙 |
| **D19** | バージョンアップ互換 | スキーマ変更時の暗号データ移行 | 自動マイグレーション |
| **D20** | 災害復旧 | 鍵紛失 / ファイル破損時の復旧 | バックアップ戦略 |

特に **D17 export 互換** は要注意。export 時に暗号化データが平文で出ないようにする必要。

## U25-U30. 「視認性」最重要 UX 候補

key insight: 暗号化されていても、**いま画面に何の機微度のものが映っているか** が
わからなければ事故が起きる。視認性の選択肢:

| ID | UI | 内容 |
|---|---|---|
| **U25** | 画面端の vault バー | 常時 vault 表示の縦バー |
| **U26** | 上部 banner | 高機微度時は赤帯 banner |
| **U27** | windows title | ブラウザタイトルに `[SECRET]` |
| **U28** | favicon 変化 | tab favicon が南京錠に |
| **U29** | カーソル変化 | 機密領域でカーソル形状変化 |
| **U30** | 周辺フェード | アクティブ機密ノード以外フェード |

組合せ: **U25 + U26 + U27** が「いま何 vault」明示の標準パッケージ候補

## アクセシビリティ・キーボード操作

| ID | 観点 | 内容 |
|---|---|---|
| **A1** | 色だけに依存しない | U2 色帯 + 形状（点・三角・鍵） |
| **A2** | スクリーンリーダー対応 | 機微度を音声で読み上げ |
| **A3** | キーボードのみで vault 切替 | VS3 ショートカット |
| **A4** | キーボードのみで機微度設定 | 既存 keyboard_modes と整合 |

## モバイル・他デバイス UX

| ID | 観点 | 内容 |
|---|---|---|
| **MO1** | スマホで機密 vault を開けるか | V16 desktop only との衝突 |
| **MO2** | 生体認証の活用 | Face ID / Touch ID で unlock |
| **MO3** | スクショ防止フラグ | OS 機能の活用（Android secure flag） |
| **MO4** | 通知に機密混入を防ぐ | preview 隠し |

## 既存 keyboard_modes との接続

- Cleanup mode との接続: 機密ノードを「presentation hide」フラグで一括非表示
- Capture mode で vault 切替ショートカット
- Status mode で機微度を attribute として変更

## キーポイント

- **視認性が物理層の最終防衛線** — U25-U30 は暗号化以上に重要
- 色帯（U2）+ 南京錠（U1）+ blur（U4）が標準パッケージ
- panic ロック（U9）は「同僚が来た時」用で必須
- AI 送信 UX は **送信プレビュー（U13）+ マスク差分（U14）+ LLM 切替表示（U17）** が最低
- データモデルは **D2 + D4（vault デフォルト + ノード override）** が推し
- export 互換（D17）が見落とされやすい事故源
- アクセシビリティ（色覚 / スクリーンリーダー）配慮を初期から入れる
