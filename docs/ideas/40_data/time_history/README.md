# 時間軸／履歴（Time & History）

M3E マップを「時間方向にも探索可能」にするためのブレスト。
**今のマップ** だけでなく、**過去の任意の瞬間**・**もし別の選択をしていたら**・
**最近の変化** ・**忘却の演出**・**履歴 AI 要約** といった、
時間軸に関する 5 つの切り口（I1〜I5）をまとめて掘る。

00_topic_pool.md カテゴリ I（時間・履歴・バージョン）を素材とする。
既存 `backup.ts`（SQLite snapshot, 1h 間隔, 10 世代）と
`audit_log.ts`（ring buffer 500 件 + JSONL 永続化, add/edit/delete/move）が
データソースの土台になり得る。

## 方針

- 採否判断はしない、複数案を並べる
- 実装詳細は決めない（コードは MVP ファイル参考程度）
- I1〜I5 は **独立に検討しつつ、共通基盤（履歴ストア）を意識** する
- 各サブトピックの **誰が／いつ／どの研究シーン** を最低 3 個具体化
- **時間軸 UI のメタファ**（タイムライン／世代スライダ／ブランチ Tree／カレンダー）を比較
- **何を保持するか × どこから再現するか × どう見せるか** を分離
- 既存資産（backup.ts / audit_log.ts / m3e-map.json）と接続
- 半年で「世界モデル→射影」実用化（project_projection_vision）に
  **過去の自分の思考過程を見直す手段** として効く設計を優先

## 全体俯瞰

5 つのサブトピックは **時間軸の使い方** で 3 グループに分かれる:

| グループ | サブトピック | 主動作 | 主な視聴者 |
|---|---|---|---|
| **過去への移動** | I1 タイムトラベル / I2 What-if | 任意時点へ「跳ぶ」「枝分かれる」 | 自分（振り返り・反実仮想） |
| **時間の表示** | I3 Diff アニメ | 変化を「再生する」 | 自分・共同研究者・指導教員 |
| **時間の操作・要約** | I4 忘却 / I5 AI 要約 | 「見せない」「圧縮する」 | 自分（認知負荷管理） |

```
       過去 ←──────── 現在 ────────→ 未来（仮想）
        │                                │
   I1 任意時点復元           I2 what-if 分岐
        │                                │
        └────── I3 差分アニメで結ぶ ──────┘
                       │
              I5 AI 要約（時間窓を読む）
                       │
              I4 忘却（古いノードをぼかす）
```

## キーメッセージ（先取り）

- **5 つは "時間の使い方の異なる側面"。** 共通基盤は履歴ストアだが、UI は別物として
  設計するほうが筋がいい（時間スライダ vs ツリー vs アニメ vs 霞 vs テキスト要約）
- **「保持」と「再現」と「見せ方」を分離せよ。** 既存 backup.ts は 1h × 10 世代（粗）、
  audit_log.ts は操作単位（密）。**密＋粗のハイブリッド** が再現コストを下げる
- **What-if は "ブランチ" ではなく "サンドボックス"。** Git 風 branch を持ち込むと
  認知コストが爆発する。**現在マップを汚さない一時オーバーレイ** が研究者向けに最適
- **忘却は破壊ではなく演出。** ノードは消えず、**透明度・距離・色彩** で「忘れている感」
  を出す。思い出した時に復活できる可逆性が必須
- **AI 要約は "週報" メタファが最強。** 「先週何があった」は研究者にとって
  毎週リアルに必要。Slack の summary や Granola の議事録 AI と接続イメージ
