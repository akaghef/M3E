# 03. 商用 / クラウド API 系 OCR ツール候補

従量課金・SaaS として提供されるもの。OSS より精度・運用面で勝ることが多いが、ランニングコストとデータ秘匿性が課題。

## C1. Mathpix Convert / Snip

- **提供**: mathpix.com
- **API**: Convert API（PDF → Markdown+LaTeX / MMD / Docx / TeX ZIP）、Snip API（画像 → LaTeX）
- **料金**: Convert は従量、1 ページあたり $0.005〜0.01 程度（プラン次第）。Snip は月額
- **強み**: **数式 OCR のデファクト**、学術界で最も実績、MMD (MathPix Markdown) は LaTeX 拡張で記法豊か
- **弱み**: 有料、大量処理だとコストが効く、データは API 送信
- **数式精度**: ◎ (業界トップ級)
- **図版**: 抽出される（ラスタ）
- **構造**: 見出し・数式・表・リストは認識
- **推し度**: ★★★★★ (商用の第一候補、1 冊単位なら十分許容コスト)

## C2. LlamaParse (LlamaIndex)

- **提供**: cloud.llamaindex.ai
- **料金**: 無料枠 1000 pages/day (個人)、超過は $0.003〜0.01/page
- **強み**: RAG 向け設計、表・数式・Markdown・JSON 出力、最近数式精度が向上
- **弱み**: ベンダー LlamaIndex への依存、数式は Mathpix ほどではない
- **数式精度**: ◯〜◎
- **図版**: 抽出あり
- **推し度**: ★★★★☆ (無料枠が広い、個人利用なら事実上タダで回せる)

## C3. Reducto

- **提供**: reducto.ai
- **料金**: 従量、$0.008〜/page 前後
- **強み**: 精度評価で上位常連、数式・表・レイアウト全部強い、API シンプル
- **弱み**: 有料、日本でのマーケ薄い
- **数式精度**: ◎
- **図版**: 抽出 + 分類
- **推し度**: ★★★★☆ (Mathpix の対抗馬、1 冊スポット利用候補)

## C4. Unstructured.io

- **提供**: unstructured.io
- **料金**: 無料 OSS ライブラリ + 商用 API、API は従量
- **強み**: 入力形式多彩 (PDF/docx/html/epub)、RAG 定番、Python SDK 充実
- **弱み**: 数式対応は弱め、表は中程度
- **数式精度**: △〜◯
- **図版**: bbox
- **推し度**: ★★★☆☆ (汎用性は高いが数学書特化には弱い、他形式の文書も扱うなら)

## C5. Chunkr

- **提供**: chunkr.ai
- **料金**: 無料 OSS 版 (chunkr-ai/chunkr) + ホスト版、ホストは従量
- **強み**: OSS 版もホスト版もある、数式・表対応進行中
- **弱み**: 新しいので実績蓄積中
- **数式精度**: ◯
- **図版**: 抽出 + bbox
- **推し度**: ★★★☆☆ (将来ウォッチ)

## C6. Adobe PDF Extract API

- **提供**: Adobe Document Services
- **料金**: 無料枠あり (月 500 ドキュメント)、超過は $0.05/doc 程度
- **強み**: 表抽出の精度が高い、Adobe 純正
- **弱み**: **数式は完全に無視** (画像として埋め込まれる)、高機能だが数学書には向かない
- **数式精度**: ×
- **図版**: 抽出あり
- **推し度**: ★★☆☆☆ (数学用途では非推奨、一般 PDF なら優秀)

## C7. Google Document AI

- **提供**: Google Cloud
- **料金**: $1.50/1000 pages （Form Parser 等）〜高額
- **強み**: 汎用 OCR 精度、多言語、日本語 OK
- **弱み**: **数式対応なし**（数式は崩れる）、GCP 課金セットアップ必要
- **数式精度**: ×
- **図版**: bbox
- **推し度**: ★★☆☆☆ (数学には向かない)

## C8. Azure Document Intelligence (旧 Form Recognizer)

- **提供**: Azure
- **料金**: $1.50/1000 pages 〜
- **強み**: Layout モデルが強い、表・段落検出
- **弱み**: **数式対応限定的**
- **数式精度**: △
- **図版**: bbox
- **推し度**: ★★☆☆☆ (数学用途では弱い)

## C9. AWS Textract

- **提供**: AWS
- **料金**: $1.50/1000 pages (Document)、$15/1000 pages (Analyze)
- **強み**: 表・フォーム強い、AWS 統合
- **弱み**: **数式対応なし**
- **数式精度**: ×
- **推し度**: ★☆☆☆☆ (数学には使わない)

## C10. Mistral OCR API

- **提供**: mistral.ai
- **料金**: 従量 ($0.001〜/page 帯、新規参入価格)
- **強み**: 新興で価格競争力、数式・表・レイアウト対応を謳う、2024 末ローンチで評価は蓄積中
- **弱み**: 新しいので比較データ薄
- **数式精度**: ◯〜◎（評価次第）
- **図版**: 抽出あり
- **推し度**: ★★★★☆ (価格／精度ともに注目、実測必要)

## C11. Firecrawl Document Parsing

- **提供**: firecrawl.dev
- **料金**: 1 credit/page（サブスク次第で実質 $0.001〜/page）
- **強み**: Web スクレイピングと統合、URL 指定で PDF も処理、Markdown 出力
- **弱み**: **数式・LaTeX 保持は明言されていない**（前会話で確認済み）
- **数式精度**: 低〜中
- **図版**: 不明瞭
- **推し度**: ★★☆☆☆ (Web 上 PDF を URL で拾う前段として使える程度)

## C12. Anthropic / OpenAI Vision API 直呼び

- **提供**: Anthropic Claude (Opus 4.7 / Sonnet 4.6)、OpenAI GPT-4o / GPT-5 系
- **料金**: トークン従量、1 ページ数円〜十数円
- **強み**: LLM が直接画像を解釈、構造や意味理解が柔軟、数式も LaTeX で吐ける
- **弱み**: ページごとに毎回 LLM 呼び = 高コスト、構造整合性のブレ
- **数式精度**: ◎ (モデル次第)
- **図版**: 認識はするが抽出は別作業
- **推し度**: ★★★★☆ (スポット利用、難しい数ページだけ LLM に渡す使い方が強い)

## C13. InftyReader (商用数学 OCR 老舗)

- **提供**: www.inftyproject.org（日本発）
- **料金**: 有償 (パッケージ販売、数万円)
- **強み**: **数学 OCR 特化 20 年以上**、数学書の数式・レイアウト精度が非常に高い、MathML/LaTeX 出力
- **弱み**: UI が古い、Windows 中心、API というよりデスクトップアプリ、スキャン PDF 向け
- **数式精度**: ◎ (スキャン PDF に対して独走)
- **図版**: △
- **推し度**: ★★★★☆ (**スキャン和書 / 古書の数式** なら事実上の最適解。04 ファイルで詳述)

## 論点

- **論点 Z1**: **スポット商用利用 vs 定常サブスク** — 頻度低いなら Mathpix/Reducto のプリペイド、定常なら LlamaParse 無料枠活用
- **論点 Z2**: **LLM Vision 直呼び vs 専用 OCR** — コストと精度のバランス。難ページだけ LLM もあり
- **論点 Z3**: **Mistral OCR の実測評価** — 新興なので価格魅力だが 2026 時点での精度は要 PoC
- **論点 Z4**: **InftyReader** はスキャン和書最強だが、PDF パイプラインにどう組み込むか (GUI 前提)
- **論点 Z5**: 商用 API に送っていい **プライバシー判断** — 自分の研究ノートを含む場合は要注意 (policy_privacy と整合確認)
- **論点 Z6**: Firecrawl は数式弱いが **「Web 公開論文を URL で拾う」前段** としては便利、組み合わせ可
