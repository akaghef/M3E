# 04. N3: 教師向け「学生のマップ」一覧

複数の学習者マップを教員が横断俯瞰する。
教育用途で **最もプライバシー設計が重い** 領域。policy_privacy / L1〜L6 と直交。

## 目的

- 教員が学生群の理解度を一覧
- クラス全体の躓き箇所を発見
- 個別指導の優先順位付け
- 教材改善のフィードバック

## 1. 集約方式の選択肢（N3.A）

| 案 | 集約 | 教員負担 | 学生プライバシー | 推し |
|---|---|---|---|---|
| N3.A1 各学生マップを個別閲覧 | なし | 重 | 高（個別共有許諾） | ⭐⭐ |
| N3.A2 ヒートマップ集約（匿名統計） | 集約 | 軽 | 高 | ⭐⭐⭐ |
| N3.A3 ピア集約（学生同士見せ合う） | グループ | 中 | 中 | ⭐ |
| N3.A4 教員が「教材マップ」だけ管理 | しない | 軽 | 最高 | ⭐⭐ |
| N3.A5 機械学習で躓きパターン抽出 | 高度 | 軽 | 中（生データ要） | ⭐ |
| N3.A6 学生から「質問ノード」だけ収集 | 部分 | 中 | 中 | ⭐⭐ |

## 2. 比較軸（N3.B）

学生マップを横断して何を見るか:

### N3.B1. 進度（どこまで読んだか）
- ステータス「読了」ノード数 / 全ノード数

### N3.B2. 理解度（クイズ正答率）
- N2 連携、ノード単位の正答率ヒートマップ

### N3.B3. 質問数
- 「質問」ステータスのノードが多いトピック = 難所

### N3.B4. 滞在時間
- どのノードを長く見ていたか
- 重い解析、プライバシー強

### N3.B5. ノード追加数
- 学生が独自に増やしたノード = 興味の深さ

### N3.B6. リンク密度
- 学生が概念間のリンクをどれだけ張ったか = 理解の構造化度

### N3.B7. 用語使用頻度
- 自由記述で使った専門用語の頻度

## 3. 表示形式（N3.C）

### N3.C1. 教材マップ上のヒートマップオーバーレイ
- 教材ノードの色 = クラス平均の理解度
- 推し: ⭐⭐⭐

### N3.C2. 表形式（学生 × ノード）
- 行: 学生名（or ID）、列: ノード、セル: ステータス
- スプレッドシート的

### N3.C3. 個別マップ並列表示
- 4〜9 学生のミニマップを同時表示
- 重い、scale 問題

### N3.C4. 時系列グラフ
- 学生の進度を時系列で

### N3.C5. ランキング / リーダーボード
- 進度・正答率順位
- 教育的に賛否両論

### N3.C6. 異常検出アラート
- 「全クラスで詰まっているノード」を自動通知

## 4. プライバシー設計（N3.D）

最重要セクション。policy_privacy / L1〜L5 と整合必須。

### N3.D1. オプトイン必須
- 学生が明示的に「教員に共有」したマップだけ閲覧可

### N3.D2. 匿名統計のみ
- 個別マップは見れない、集計だけ

### N3.D3. 仮名化（学籍番号でなく仮名）
- 教員も誰のマップか分からない（評価バイアス回避）

### N3.D4. ノード単位 ACL
- 「公開」「教員のみ」「非公開」をノードごとに

### N3.D5. 教員側の閲覧ログ
- 学生が「いつ誰に見られたか」を確認可能

### N3.D6. 評価切り離し
- ダッシュボードと成績評価は別システム
- 「監視されている」感の回避

### N3.D7. 卒業後自動消去
- 学期終了で教員アクセス権失効

## 5. データフロー（N3.E）

### N3.E1. 学生 → クラウド → 教員（SaaS）
- リアルタイム
- プライバシー懸念最大

### N3.E2. 学生 → ローカルファイル提出
- 学期末に export して提出
- バッチ的

### N3.E3. 学生 → 暗号化共有 → 教員
- 学生の鍵で暗号化、教員のみ復号
- E2E 暗号化

### N3.E4. 同期しない（学生だけが見る）
- N3.A4 と同じ。最も安全。
- 推し: ⭐⭐（MVP として）

## 6. 教材改善ループ（N3.F）

### N3.F1. 学期末レポート
- 「このノードでクラスの 60% が詰まりました」
- 教員が次年度教材を改善

### N3.F2. リアルタイム介入
- 講義中に学生のマップを見て質問を投げる
- 重い、運用難

### N3.F3. 過去年度比較
- 「去年と比べて今年は3章で躓きが多い」

## 7. ユースケース

### UC1. 大学講義（30人クラス）
- 学生は任意で自分のマップを N3.E2（学期末提出）
- 教員は N3.A2 ヒートマップで全体俯瞰
- 翌年度の講義改善に使う

### UC2. ゼミ（5人）
- 全員 opt-in、N3.A1 個別閲覧
- 週次面談で各人マップを一緒に見る

### UC3. オンライン講座
- 受講者数千人、N3.A2 統計のみ
- 講義動画の改善材料

### UC4. 塾の個別指導
- 1教員 vs 1〜10 学生
- N3.A1 個別閲覧、N3.D6 評価切り離しは塾だと適用しにくい

## 論点 ID

- N3.A. 集約方式（個別閲覧 / 統計 / ピア / 教材のみ / ML）
- N3.B. 比較軸（進度 / 正答率 / 質問数 / 滞在 / 構造化度）
- N3.C. 表示形式（ヒートマップ / 表 / 並列 / 時系列 / ランキング / アラート）
- N3.D. プライバシー設計（オプトイン / 匿名 / 仮名 / ACL / ログ / 評価切り離し）
- N3.E. データフロー（クラウド / 提出 / 暗号化 / 同期しない）
- N3.F. 教材改善ループ（学期末 / リアルタイム / 過去比較）
- N3.G. 教員間の共有（同僚教員と教材を共有するか）
- N3.H. 教員アシスタント AI（TA agent）
