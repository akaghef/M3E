# 03. N2: 学生用クイズ生成

マップ上のノード内容から自動的にクイズを生成する。
自習用・教員の小テスト作成・SRS（J1）と連携。

## 目的

- 学習者が「自分が理解しているか」を即時確認
- 教員が小テスト作成にかける時間を削減
- 同じマップから無限にバリエーションを生成

## 1. 問題形式の選択肢（N2.A）

| 形式 | 自動生成容易さ | 採点容易さ | 教育効果 | 推し |
|---|---|---|---|---|
| N2.A1 穴埋め（Cloze） | 高（用語マスク） | 高 | 中 | ⭐⭐⭐ |
| N2.A2 多肢選択（MCQ） | 中（ダミー選択肢必要） | 高 | 中 | ⭐⭐ |
| N2.A3 〇× | 高 | 高 | 低 | ⭐ |
| N2.A4 短答記述 | 中 | 中（AI 採点） | 高 | ⭐⭐ |
| N2.A5 自由記述 | 低 | 低 | 最高 | ⭐ |
| N2.A6 用語定義（フラッシュ） | 高 | 高 | 中 | ⭐⭐⭐ |
| N2.A7 順序並べ替え | 中 | 高 | 中 | ⭐⭐ |
| N2.A8 図表ラベル | 低 | 中 | 高 | ⭐ |
| N2.A9 概念マップ補完 | 中（M3E ネイティブ） | 中 | 最高 | ⭐⭐⭐ |
| N2.A10 計算問題（数式） | 低 | 中 | 高 | ⭐ |

### 特に N2.A9 が M3E らしい
マップから一部ノード/リンクを伏せて「埋めよ」とする問題。
他ツールには無いネイティブ形式。

## 2. 生成方式（N2.B）

### N2.B1. AI（LLM）に委ねる
- ノード内容を渡して「この内容からクイズを5問作って」
- 品質: 中〜高、ばらつき大
- コスト: API 課金
- 関連: ai_subagent.ts

### N2.B2. ルールベース（テンプレ）
- 「{用語} とは何か」「{A} と {B} の違いは」など定型
- 品質: 単調だが安定
- コスト: ゼロ

### N2.B3. 教員が手動作成 + AI 補助
- 教員が骨子、AI が選択肢ダミー生成
- 品質: 高、コスト: 中

### N2.B4. ハイブリッド（教員レビュー必須）
- AI 生成 → 教員チェック → 公開
- 推し: ⭐⭐⭐（教育用途では責任問題回避必須）

### N2.B5. 学習者自身が生成
- 「自分で問題を作る」=深い学習効果
- 関連: cognitive_science（メタ認知）

## 3. 採点方式（N2.C）

### N2.C1. 完全一致（穴埋め・〇×）
- 簡単、表記揺れに弱い

### N2.C2. 正規表現・同義語辞書
- 「ベイズの定理 / Bayes' theorem」を同一視

### N2.C3. AI 採点（記述問題）
- 模範解答 + 学習者解答 → AI が部分点判定
- 信頼性課題

### N2.C4. 自己採点
- 解答後に正解表示、学習者が「正解 / 半正解 / 不正解」を自己申告
- SRS（Anki 方式）

### N2.C5. ピア採点
- 学習者同士が採点
- 教育効果あり、運用重い

## 4. 出題ロジック（N2.D）

### N2.D1. 全問ランダム
- 最も簡単
- 復習効率悪い

### N2.D2. SRS（間隔反復）
- 忘却曲線に沿って出題
- J1 連携、cognitive_science ブレストに既存
- 推し: ⭐⭐⭐

### N2.D3. 弱点重点
- 過去の不正解箇所を多めに出題

### N2.D4. 章節絞り込み
- 「今日は2章だけ」のような範囲指定

### N2.D5. 適応的（IRT 風）
- 解答結果から推定能力を更新、難易度調整
- 重い、研究領域

### N2.D6. 教員指定
- 「次回小テスト用にこの10問」を選ぶ

## 5. UI / フロー（N2.E）

### N2.E1. マップ内オーバーレイ
- 現在のノード上に問題を表示
- マップから離れない

### N2.E2. 別ビュー（フラッシュカード風）
- フルスクリーンで1問1画面

### N2.E3. キーボード駆動
- スペースで次、矢印で選択
- M3E の keyboard_modes と接続

### N2.E4. モバイル特化
- 通学中の隙間時間 → F2 連携

## 6. クイズの永続化（N2.F）

### N2.F1. ノード属性として埋め込む
- 各ノードに `quiz: [...]` 属性
- マップと一体化

### N2.F2. 別 JSON ファイル
- `<workspace>.quiz.json`
- マップ純度保つ

### N2.F3. 別 workspace（クイズプール）
- 「教科書」ws と「クイズ」ws を別管理

### N2.F4. 永続化しない（毎回 AI 生成）
- 鮮度高い、コスト高

## 7. ユースケース

### UC1. 学部生の試験対策
- 「線形代数」マップから N2.B1 で穴埋め50問生成
- N2.D2（SRS）で1週間反復
- 試験当日には弱点ノード可視化

### UC2. 教員の小テスト作成
- 講義マップから N2.B3 で骨子生成 → 教員が10問選定 → 印刷配布
- 配布形式: PDF / Google Forms 連携

### UC3. ゼミ前の確認テスト
- 担当者が章マップから N2.A9（概念マップ補完）を作成
- ゼミ冒頭5分で全員回答 → 弱点共有

## 論点 ID

- N2.A. 問題形式（穴埋め/MCQ/記述/概念マップ補完...）
- N2.B. 生成方式（AI / ルール / 手動 / ハイブリッド / 学習者自作）
- N2.C. 採点方式（完全一致 / AI / 自己 / ピア）
- N2.D. 出題ロジック（ランダム / SRS / 弱点 / 範囲 / 適応 / 指定）
- N2.E. UI（オーバーレイ / 別ビュー / キーボード / モバイル）
- N2.F. 永続化（属性 / 別ファイル / 別 ws / 都度生成）
- N2.G. 多言語問題（同じマップから日英版生成）
- N2.H. 難易度自動推定
- N2.I. 共有・配布（マップ非公開でクイズだけ配布）
