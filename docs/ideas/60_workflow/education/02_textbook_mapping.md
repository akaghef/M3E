# 02. N1: 教科書のマップ化

教科書（PDF / EPUB / 紙→OCR / Web 教科書）を M3E のマップに変換する。
A3「PDF 取り込み」と密接、ただし教育コンテキストに特化。

## 目的

- 章節構造を木構造として保持
- 用語・概念を別ノード化、相互リンク
- 演習問題を独立ノードとして status 管理可能に
- 学習者が「読書ペース」を可視化できる

## 1. 入力源の選択肢（N1.A）

| 案 | 入力 | 自動化容易さ | カバー範囲 | 推し |
|---|---|---|---|---|
| N1.A1 | PDF（テキスト埋め込み） | 高 | 大学教科書の多く | ⭐⭐ |
| N1.A2 | PDF（スキャン）→OCR | 中 | 古い教科書 | ⭐ |
| N1.A3 | EPUB / Kindle | 高 | 商業書籍 | ⭐⭐ |
| N1.A4 | LaTeX ソース | 最高 | 数学・物理書籍 | ⭐⭐⭐ |
| N1.A5 | Markdown / Web 教科書 | 最高 | OSS 教科書 | ⭐⭐ |
| N1.A6 | 紙→写真→OCR | 低 | 個人ノート | ⭐ |
| N1.A7 | 動画講義の字幕 | 中 | Coursera/YouTube | ⭐⭐ |
| N1.A8 | 手入力（章タイトルだけ） | 最高 | あらゆる書籍 | ⭐ |

## 2. 構造抽出の粒度（N1.B）

### N1.B1. 章単位
- ノード数: 5〜20
- 適: 索引・全体俯瞰
- 不適: 概念単位の進度管理

### N1.B2. 節単位
- ノード数: 30〜200
- 適: 一般的な大学講義の進度
- 推し: ⭐⭐⭐

### N1.B3. 段落単位
- ノード数: 500〜5000
- 適: 精読・反復学習
- 不適: 一覧性、scale issue（M1）

### N1.B4. 概念単位（用語抽出ベース）
- ノード数: 100〜1000
- 適: 概念マップ、SRS（J1 連携）
- 構造: 章節とは別レイヤー、リンクで紐付け
- 推し: ⭐⭐

### N1.B5. ハイブリッド（章節＋概念の二層）
- 章節木 ＋ 概念ノードの relation
- 推し: ⭐⭐⭐
- 実装: 重い

## 3. 演習問題の扱い（N1.C）

### N1.C1. 本文ノードの details に含める
- 簡単
- status 管理ができない

### N1.C2. 演習を独立ノード（child）として作る
- 「2.3.演習1」「2.3.演習2」をノード化
- status: 未着手 / 解答済み / 正解 / 不明
- 推し: ⭐⭐⭐

### N1.C3. 別 workspace に演習だけ集める
- 復習・SRS 用途
- マップ複製問題

### N1.C4. 演習はクイズ生成（N2）に流す
- 元教科書の演習はマップ化せず、N2 でクイズ化
- メリット: 重複しない

## 4. メタデータ（N1.D）

教科書ノードに付ける属性:

- 著者・出版年・ISBN
- 章のページ範囲（PDF だと p.45-67）
- 難易度（初級/中級/上級/院生）
- 重要度（必須/推奨/参考）
- 推定読了時間
- 言語（日/英/独 etc.）

## 5. 図表・数式の扱い（N1.E）

### N1.E1. 画像として埋め込み
- 簡単
- 検索不可

### N1.E2. 数式は LaTeX 文字列
- KaTeX/MathJax レンダリング必要
- 検索可能
- 推し: ⭐⭐

### N1.E3. 図表に独立ノードを与える
- 「Fig 2.3 ベイズの定理」をノード化
- 引用しやすい

### N1.E4. AI で alt-text 生成
- 視覚障碍対応・検索向上

## 6. 既存教科書のテンプレ化（N1.F）

人気教科書のマップを **公式テンプレ** として配布する案:

- 数学: 「線形代数（齋藤）」「解析入門（小平）」
- 物理: 「ファインマン物理学」
- CS: 「SICP」「TAOCP」
- ML: 「PRML」「Bishop」「ESL」

メリット: 入力コスト 0 で学習開始
デメリット: 著作権・配布権

## 7. ユースケース具体化

### UC1. 学部生が「線形代数」を独学
- 教科書 PDF を取り込み → 章節マップ生成
- 各節を読了するごとに status を「読了」に
- 演習問題ノードに解答メモ
- 試験前に「未読了」「演習未着手」をフィルタ

### UC2. 院生がゼミ輪読
- 各週担当の章を別workspace で展開
- 議論で出た疑問は質問ノード追加
- 章末で全員のマップをマージ

### UC3. 教員が講義資料準備
- 採用教科書のマップを取り込み
- 自分の講義順に sort、不要章を archive
- 補足資料を子ノードとして追加

## 論点 ID

- N1.A. 入力源（PDF/LaTeX/EPUB/Web/手入力）
- N1.B. 抽出粒度（章/節/段落/概念/ハイブリッド）
- N1.C. 演習問題の扱い（details / 独立ノード / 別ws / N2 連携）
- N1.D. メタデータの種類（難易度・推定時間・重要度）
- N1.E. 図表・数式の表現（画像 / LaTeX / 独立ノード / alt生成）
- N1.F. 公式テンプレ配布の有無
- N1.G. 著作権処理（ユーザーローカルのみ / 公開禁止 / fair use）
- N1.H. 翻訳機能（英語教科書を日本語で読む / バイリンガル）
