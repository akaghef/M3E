# 03. カタログ — 思考・論理系

「考えを整理する・深掘りする」系のフレームワーク群。
M3E は木構造そのものなので、この系統は特に親和性が高い。

## T1. MECE （漏れなくダブりなく）

**用途**: 分解時のチェック原則。スニペットというより**検査観点**。

**スニペットとしての形**:
```
親
├── カテゴリA
├── カテゴリB
├── カテゴリC
└── その他（MECE 残余）
```

**ポイント**: 「その他」を最初から置くとカバー率が上がる。AI レビューで MECE 性を評価する拡張も面白い。

## T2. ロジックツリー（一般形）

**用途**: イシューや原因を階層的に分解。

**構造**: 親 → 子群 → 孫群（深さは自由、パラメタ化）

**バリエーション**:
- 要素分解（WHAT ツリー）
- 原因分解（WHY ツリー）
- 手段分解（HOW ツリー）
- イシュー分解（IF ツリー）

## T3. Why ツリー / なぜなぜ分析

**用途**: 原因追求。トヨタ式「なぜを 5 回」。

**構造**:
```
親（事象）
└── Why 1
    └── Why 2
        └── Why 3
            └── Why 4
                └── Why 5 → 真因
```

**パラメタ**: 深さ N（3〜7）、分岐数（単線型 vs 樹形型）。

**推し**: M3E で最も映える候補の1つ。1キーで深掘り展開できるとバグ調査が激速になる。

## T4. How ツリー / 手段展開

**用途**: 「どうやって？」を手段に展開。How-How 法とも。

**構造**: Why ツリーの向きを逆にしたもの。目的 → 手段 → さらに手段...

## T5. What ツリー / 要素分解

**用途**: 対象を構成要素に分解。例: 「コスト」 → 「固定費/変動費」 → さらに細目。

**用途区別**: Why が原因、How が手段、What が構成。3 者セットで覚える。

## T6. ピラミッド原則（Minto Pyramid）

**用途**: 結論 → 根拠 → データの階層で論理構造化。プレゼン・文書構成に。

**構造**:
```
親（結論）
├── 根拠 1
│   ├── データ 1a
│   └── データ 1b
├── 根拠 2
└── 根拠 3
```

**補助**: 下から上へ帰納 / 上から下へ演繹 の 2 向き。

## T7. 演繹 / 帰納 / アブダクション

**用途**: 推論の3形式。ノードに「推論タイプ」ラベルを付けるだけでも思考が整う。

**スニペット形**:
```
親（主張）
├── 大前提（演繹用）
├── 小前提（演繹用）
├── 観察事例（帰納用）
└── 仮説（アブダクション用）
```

## T8. 仮説思考

**用途**: 仮説を先に立てて検証する思考。

**構造**:
```
親（問い）
├── 仮説 1
│   ├── 検証方法
│   ├── 予想される結果
│   └── 反証条件
├── 仮説 2
└── 仮説 3
```

## T9. イシューツリー

**用途**: 問題を「解くべき問い」に分解。コンサル系で定番。

**構造**: Key Question → Sub Questions → Sub-sub Questions
**差分（vs ロジックツリー）**: ノードが全て疑問文になる。

## T10. 特性要因図（フィッシュボーン / 石川ダイアグラム）

**用途**: 原因の網羅的洗い出し。製造業 QC で標準。

**構造（4M / 5M1E）**:
```
親（結果・問題）
├── Man （人）
├── Machine （機械）
├── Material （材料）
├── Method （方法）
├── Measurement （測定）
└── Environment （環境）
```

各カテゴリの下に具体的原因を追加。

**表示**: 魚の骨形状（map_views で横向きレイアウト案）。

## T11. パレート分析 / 80-20 ルール

**用途**: 効果の偏りを発見。

**スニペットとしての形**:
```
親（対象集合）
├── 上位20%（高影響）
│   ├── 要因A
│   └── 要因B
└── 下位80%（ロングテール）
```

**ポイント**: 数値評価スロットを持つノード種別と組合せると自動ソート可能。

## T12. マインドマップ（古典）

**用途**: 中心概念から放射状に連想展開。

**構造**: 中心 → 主枝 → 副枝（分岐数無制限、ラベル自由）

**M3E での位置づけ**: M3E 自体がマインドマップに近いので、「放射状展開モード」として意味を持つ。

## T13. KJ 法 / 親和図法

**用途**: ブレストで出た大量のアイデアをグルーピング。

**構造**:
```
親（テーマ）
├── グループA（命名）
│   ├── 付箋1
│   ├── 付箋2
│   └── 付箋3
├── グループB
└── グループC
```

**ワークフロー**: (1) 付箋散らす → (2) 近いもの集める → (3) グループに名前 → (4) 関係線引く。

**スニペットが活きる場面**: (1)の時点で空カードを N 枚用意する「発散スニペット」、(3) でグループ化後に構造を与える「収束スニペット」。

## T14. 系統図法

**用途**: 目的・手段の連鎖を階層で。ロジックツリーと似るが QC 手法。

**差分**: 特に構造は同じだが QC 7つ道具文脈で呼ばれる。

## T15. マトリクス図法 / クロス分析

**用途**: 2 軸で対象を分類。

**スニペット形**:
```
親（マトリクス名）
├── 軸 X （定義）
├── 軸 Y （定義）
└── セル
    ├── X1Y1
    ├── X1Y2
    ├── X2Y1
    └── X2Y2
```

**パラメタ**: 軸の段階（2/3/N）、セルを自動生成。

## T16. ブレインライティング / 635 法

**用途**: 集団発散。6人×3案×5分を繰り返す。

**構造**:
```
親（テーマ）
├── Round 1
│   ├── Participant1 （3案）
│   ├── Participant2
│   └── ...
├── Round 2
└── Round 3
```

**用途区別（vs ブレスト古典）**: 書く形式。黙って書くので声の大きい人に流されない。

## T17. 逆転の発想 / 逆ブレスト

**用途**: 「どうすれば失敗するか」を考えてから裏返す。

**構造**:
```
親（目標）
├── 失敗するには？
│   ├── 失敗要因1
│   ├── 失敗要因2
│   └── 失敗要因3
└── 成功策（反転）
    ├── 対策1
    ├── 対策2
    └── 対策3
```

**兄弟**: プレモーテム（05 D15）と近い。

## T18. 前提疑い / Assumption Busting

**用途**: 暗黙の前提を洗い出して潰す。

**構造**:
```
親（命題）
├── 暗黙前提1 — それを取り除くと？
├── 暗黙前提2 — それを取り除くと？
└── 暗黙前提3 — それを取り除くと？
```

## T19. ラダリング（梯子登り法）

**用途**: 表層的回答から深層動機まで段階的に「なぜ？」を辿る（消費者調査系）。

**構造**: Attribute → Consequence → Value の 3 層（Means-End Chain）。

## T20. OODA ループ

**用途**: 意思決定の循環プロセス（軍事・経営）。

**構造**:
```
親
├── Observe （観察）
├── Orient （状況判断）
├── Decide （意思決定）
└── Act （行動）
```

**リンク**: 循環なので Act → Observe へのリンクを自動で張る。

---

## 横断論点（T）

- **Ta. 再帰展開** — Why ツリー・ロジックツリーは深さがパラメタ。呼び出し時に「深さ3」指定可能に
- **Tb. 縮退形** — MECE みたいに「検査のみ」モードがあってもいい（構造展開はしないが警告だけ）
- **Tc. 自動命名** — 「グループA/B/C」をAI に名付けさせるオプション
- **Td. 無限分岐** — マインドマップ（T12）は分岐数無制限、他は固定。両対応が必要
- **Te. 循環構造** — OODA, PDCA は「Act → Observe」のリンクを張るか。木構造を外れるのでリンク機能と接続
