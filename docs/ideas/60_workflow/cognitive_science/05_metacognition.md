# J5. メタ認知支援（自分の思考パターンを可視化）

J1〜J4 が **学習行動を変える** のに対し、J5 は **学習している自分自身を観察する** レイヤー。
編集パターン・改稿頻度・確信度推移などをログ化し、本人にフィードバックする。

## なぜ要るか

研究者は「自分の思考の癖」に気づきにくい。
- 同じ問題を何度も書き直しているのに気づかない
- 確信度が高い領域 / 低い領域の自覚がない
- 朝に書いたノードと夜に書いたノードの質の違いを把握していない
- 「考えていると思い込んで悩んでいるだけ」の時間が見えない

→ 行動データを **本人に返す** ことで、自己改善の出発点を作る。
N5（学習を自動化しない）原則に最も忠実な機能。AI は答えを出さず、本人に気づきを促す。

## ユースケース具体化

1. **週次レビュー** — 「今週は J3 領域に時間を使ったが、結論まで出ていない」と気づく
2. **執筆スランプ時** — 「過去の生産的だった日のパターン」を見て条件を再現
3. **共著論文後** — 「自分のどの仮説が論文に活きたか」をマッピング
4. **指導側として** — 学生に「あなたの思考の癖はこれ」と示す（教育応用）
5. **長期キャリア視点** — 「過去半年で確信度が変動した領域」を見て興味の流動を観察
6. **AI 依存度の自己点検** — 「AI に頼ったノード比率」を見て N5 原則違反を防ぐ

## J5.A. 観察対象

| 案 | 観察項目 | データ取得 | 推し度 |
|---|---|---|---|
| J5.A1 編集行動 | 編集頻度 / 編集時間帯 / 連続編集 | viewer 計装 | ★★ |
| J5.A2 思考時間 | ノード滞在時間 | viewer 計装 | ★★ |
| J5.A3 改稿パターン | 同ノードの再編集回数 / 文字差分 | history 必要 | ★★★ |
| J5.A4 確信度推移 | confidence 属性の時系列 | 明示入力 | ★★★ |
| J5.A5 リンク作成パターン | 直リンク / 間接 / 遠距離 | history 解析 | ★ |
| J5.A6 文体の変化 | 文長・抽象度の推移 | NLP 解析 | ★ |
| J5.A7 AI 依存度 | AI 生成 / 人間生成の比率 | tag 必要 | ★★ |
| J5.A8 執筆速度 | 文字/分 | viewer 計装 | ★ |

**観察**: J5.A3 と J5.A4 が研究的に最重要。両方とも他属性で代替できない独自データ。

## J5.B. フィードバック頻度

| 案 | When | 強さ | 推し度 |
|---|---|---|---|
| J5.B1 即時 | 行動直後にコメント | 高、邪魔 | ★ |
| J5.B2 セッション終了時 | 「今日のまとめ」 | 中 | ★★ |
| J5.B3 週次レポート | 月曜朝 | 弱、振り返り向き | ★★★ |
| J5.B4 月次レポート | 月初 | 弱、長期傾向向き | ★★ |
| J5.B5 オンデマンド | 「メタ認知ビュー」開いた時のみ | 最弱 | ★★ |
| J5.B6 イベント駆動 | 「執筆スランプ検知時」のみ | 中 | ★ |

**観察**: J5.B3 + J5.B5 の併用が押し付けず気づきやすい。

## J5.C. フィードバック表現

| 案 | 表現 | 推し度 |
|---|---|---|
| J5.C1 ダッシュボード | グラフ・指標一覧 | ★★ |
| J5.C2 物語型 | 「あなたは今週○○の領域で…」 | ★★★ |
| J5.C3 比較型 | 「先週 vs 今週」「自分 vs 過去平均」 | ★★ |
| J5.C4 ヒートマップ | 時間帯×領域マトリクス | ★★ |
| J5.C5 質問型 | 「気づいたことはありますか？」と問う | ★★ |
| J5.C6 未介入 | データだけ提示、解釈は本人 | ★★ |

**観察**: J5.C2 + J5.C5 が「気づきを促す」原則に合う。AI 解釈過剰は N5 違反リスク。

## J5.D. 確信度（confidence）の入力

J5.A4 を成立させるには確信度入力が必要:

| 案 | 入力方法 | 摩擦 | 推し度 |
|---|---|---|---|
| J5.D1 1〜5 数値 | テンキー、書いた直後 | 低 | ★★★ |
| J5.D2 0〜100% スライダー | 細かい | 中 | ★ |
| J5.D3 タグ式 | `#hypo` `#fact` `#guess` | 中 | ★★ |
| J5.D4 自動推定 | LLM が文体から推定 | 低（自動） | ★ |
| J5.D5 日次サマリ | 1日1回まとめて | 中 | ★ |
| J5.D6 リンク付与時 | リンクの種類で表現 | 低 | ★ |

**観察**: J5.D1 を既定、J5.D3 を補助。

## J5.E. 「思考パターン」の分類軸

何を「パターン」として抽出するか:

| 案 | 軸 | 例 |
|---|---|---|
| J5.E1 時間帯 | 朝/昼/夜の生産性差 | 朝は J3、夜は J5 |
| J5.E2 トピック領域 | カテゴリ別の集中度 | A 入力に偏重 |
| J5.E3 思考モード | 発散 / 収束 / 整理 | 発散時間が短い |
| J5.E4 中断パターン | どこで思考が止まるか | J1.B あたりで停滞 |
| J5.E5 リンク化傾向 | 孤立 vs 多リンク | 孤立ノード過多 |
| J5.E6 確信度分布 | 高確信 vs 低確信比率 | 低確信領域が多い = 探索期 |
| J5.E7 改稿率 | 「最初に書いて満足」vs「何度も改稿」 | 改稿少 = 鵜呑み傾向 |

**観察**: J5.E3 + J5.E4 の組み合わせが「自分の研究スタイル」を浮かび上がらせる。

## J5.F. 「気持ち / 状態」のログ

事実データだけでなく、本人の主観も記録対象:

| 案 | 何を | 入力タイミング | 推し度 |
|---|---|---|---|
| J5.F1 ノード単位の気持ち | 書いた時の感情1語 | 編集後 | ★★ |
| J5.F2 セッション気分 | 朝/夜のチェックイン | G1 デイリー儀式と接続 | ★★★ |
| J5.F3 エネルギーログ | 1〜5 でエネルギー入力 | 任意 | ★★ |
| J5.F4 困りごと自由記述 | 「今 stuck している」 | スランプ時 | ★ |
| J5.F5 達成感 | ノード完成時に「達成感1〜5」 | 完了時 | ★★ |

**観察**: J5.F2/F3 は G1（デイリー儀式）と統合運用が自然。

## J5.G. メタ認知 → 行動変容

ログるだけでは意味がない。**気づき → 次の行動** の繋ぎ:

| 案 | 仕組み | 推し度 |
|---|---|---|
| J5.G1 「今日の改善案」提示 | 過去パターンから提案 | ★★ |
| J5.G2 リマインダ | 「朝に J3 をやると良い」を朝に出す | ★★★ |
| J5.G3 反証提示 | 「あなたが避けている領域」を示す | ★★ |
| J5.G4 自己実験 | 「今週は朝執筆を試してみては」 | ★★ |
| J5.G5 介入なし | 観察のみ、行動は本人任せ | ★ |

**観察**: J5.G2 が日々の行動に直接影響、J5.G4 が中長期の自己理解を促す。

## J5.H. プライバシー考慮

メタ認知ログは **最も繊細なデータ**。気持ちログ、確信度、執筆速度は外に出てはいけない:

| 案 | プライバシー設計 |
|---|---|
| J5.H1 ローカルのみ | サーバ送信せず |
| J5.H2 暗号化必須 | policy_privacy 適用 |
| J5.H3 LLM 送信禁止 | L5 検閲（00_topic_pool）連携 |
| J5.H4 共有時マスク | 共有マップでメタ認知データは除外 |
| J5.H5 自己消去オプション | N日後自動削除 |

**観察**: 全部必須レベル。J5 は L1〜L5（プライバシー）と必ずセットで設計する。
