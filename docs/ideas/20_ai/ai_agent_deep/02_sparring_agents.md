# 02. 壁打ち系エージェント — C1 / C2 / C3 / C9

研究の質を上げる「対話型 AI」候補。
共通方針: **AI に答えを出させない**。AI は質問・反論・視点提供のみ。

`automation_obstacles/02_solution_patterns.md` **P3 壁打ち相手** を全候補で踏襲する。

## C1. Devil's Advocate モード

任意ノードに対して反論・批判を生成する機能。

### S1. 発火タイミング

| 案 | 概要 | 良い点 | 悪い点 |
|---|---|---|---|
| S1-a 手動ボタン | ノード上で「反論ください」 | 意図的、無駄打ちなし | 忘れる、習慣化しにくい |
| S1-b ホットキー | "D" 等の 1 キー | 高速、reviewの流れに乗る | 暴発リスク |
| S1-c proposal/decision タグで自動発火 | 「決定」を書いた瞬間に AI が反論を準備 | 重要箇所に自動介入 | 邪魔、押しつけがましい |
| S1-d 確信度高ノードに自動発火 | 「絶対」「明らか」等の語に反応 | バイアス検出に効く | 文体への過反応 |
| S1-e 静的レビュー | 週次で全 decision を巡回反論 | 漏れない | レポートが膨大化 |

### S2. 反論の深さ

- S2-a 1 行コメント（「本当に N=10 で十分？」）
- S2-b 構造化（前提 / 反例 / 代替案 の 3 ブロック）
- S2-c 多段論証（複数のレベルで批判）
- S2-d 数値シミュレーション付き（「もし X が違ったら結論が反転する」）

**観察**: 深さは固定でなく、**ユーザがホットキーで段階的に深めていく** 形が良い。

### S3 関連: 反論の「種類」リスト

- 論理: 飛躍 / 循環 / 偽の二分法
- 経験: サンプル過小 / cherry-picking / 反例
- 概念: 定義曖昧 / 用語混同
- 文脈: 先行研究無視 / 反対派視点
- 実務: 実装可能性 / コスト / 倫理

→ S1〜S3 の組み合わせをマップ内 **プロンプトノード**（G7）として保存・育成

## C2. 質問生成器

ノード内容を読んで「次に問うべき質問」を提案する。

### S3. 問い方のスタイル

| スタイル | 例 | 適用シーン |
|---|---|---|
| Socratic | 「なぜそれが本当だと思う？」 | 概念検討 |
| Five Whys | 「なぜ？」を 5 段 | 原因探究 |
| SCAMPER | Substitute / Combine / Adapt 等 | アイデア展開 |
| Premortem | 「失敗したとしたら、何が原因？」 | 計画段階 |
| Steelman | 「最強の反対意見は？」 | 議論段階 |
| Inversion | 「逆を考えると？」 | 思考停止打破 |

複数スタイルを並列に出して、**ユーザが選んで子ノード化** する流れが本命。

### 反パターン（避ける）

- ❌ 1 つの「正しい質問」を出す → N7 創発の場の侵食
- ❌ 質問→自分で答えてしまう → P3 壁打ち違反
- ❌ 同じ質問を何度も出す → キャッシュ＋既出質問の差分管理が必要

## C3. 多視点エージェント

実験家 / 理論家 / 批評家 が同時にコメントする。

### S4. 視点リスト案

| 視点セット | 構成 | 良い点 | 悪い点 |
|---|---|---|---|
| 学術 4 役 | 実験家・理論家・批評家・統計家 | 自然科学に汎用 | 人文では合わない |
| 役割 5 役 | 提案者・反対者・調停者・観察者・記録者 | 議論一般 | 学術特化が薄れる |
| 時間軸 3 役 | 過去の自分・現在・未来の自分 | 内省的 | 客観性が弱い |
| ステークホルダ | 学生・指導教員・査読者・産業利用者 | 利害対立を可視化 | 数が多すぎ |
| ユーザ定義 | プロジェクト毎にカスタム | 最適化可 | 設計コスト |

→ G1-e ハイブリッドペルソナと同じ構造。固定セット＋カスタム。

### S6. コメント保存場所

| 案 | 場所 | 良い点 | 悪い点 |
|---|---|---|---|
| S6-a 子ノード | `[実験家] N が小さい...` を子に追加 | マップに残る、議論可能 | マップが膨張 |
| S6-b メタ属性 | ノードの hidden field | マップ清潔、後参照可 | 視認性が低い |
| S6-c 別レイヤ | 「コメント layer」を ON/OFF | 切替で清潔保持 | レイヤ管理が複雑 |
| S6-d 一時バルーン | hover/select で表示、保存しない | 軽快 | 後で参照不可 |
| S6-e ハイブリッド | 重要なものだけ子ノード化 | 良いとこ取り | 「重要」の判定が必要 |

## C9. エージェント・ペルソナ

「指導教員ならどう言う？」「査読者ならどう？」を 1 アクションで切替。

### S5. ペルソナの人格データ

| データソース | 内容 | 課題 |
|---|---|---|
| プロンプトのみ | 「あなたは厳しい査読者...」 | 表層的 |
| 過去発言ログ | 実在の指導教員の発言を学習 | プライバシー |
| 役職プロファイル | 査読基準・採点ルーブリック | 入手難 |
| マップ内ノード | 「指導教員のコメント集」サブツリー | 自前で育成必要 |

**現実解**: マップ内ノードでペルソナを育てる（G7 プロンプト資産化）。
「先週のレビューで先生はこう言った」をノード化していくと、
そのまま AI ペルソナのコンテキストになる。

### S8. ペルソナ切替 UI

- pill バー（画面上部に「教員/査読者/学生」ボタン）
- コマンドパレット（`Ctrl+P` → "as supervisor" 等）
- モード（keyboard_modes M27 Assign mode の派生）
- マウス hover トグル（軽快だが事故りやすい）

## S7. 「無視する権利」

すべての壁打ち系で必須:
- AI コメントを **デフォルト非表示** にできる
- ノード単位で「このノードへのコメントは出さない」設定
- 「うるさい時はミュート」がワンクリック
- N9 自己観察・内省の領域を侵食しないため

## 共通: 壁打ちが守るべき原則

| 原則 | 由来 | 説明 |
|---|---|---|
| 答えを出さない | P3 / N7 | 提案・問い・反論まで。結論は人間 |
| 出典を示す | P9 / G6 | 「○○論文によると」のように根拠提示 |
| 確信度を表示 | G6 | 強い断言を避ける、確信度を 1〜5 で |
| 取り消し容易 | G8 | コメントは 1 キーで消せる |
| 検閲を経る | G3 | プライベートノードは送らない |

## 比較: 4 候補の使い分け

| | 入力 | 出力 | 主用途 | 使う頻度 |
|---|---|---|---|---|
| C1 Devil's Advocate | 確信ノード | 反論 | 決定前のチェック | 中 |
| C2 質問生成 | 任意ノード | 次の問い | 探索段階 | 高 |
| C3 多視点 | 主張ノード | 複数立場のコメント | 設計レビュー | 中〜低 |
| C9 ペルソナ | 任意ノード | 特定人物視点 | 提出前模擬レビュー | 低 |

→ **C2 質問生成 が最も日常使い**。C1 が研究の質に効く。
C3/C9 は「重要場面」の時々使う重火器。
