# 03. マップ衛生・気づき系 — C4 / C5 / C7 / C10

マップが大きくなった時に AI に「整理」「気づき」「自走」を任せる候補。
`automation_obstacles/` の **D系（Hygiene）障害** と直結する。

共通方針: **L1〜L2 から始め、信頼が貯まったら L3 へ**。
**「夜中にマップを書き換える小人」は怖いので、「朝に報告書を出す秘書」に近い形** が安全。

## C4. 自動 link 提案

似た内容のノード間にリンク候補を提示する。

### H1. 判定軸

| 軸 | 方法 | 良い点 | 悪い点 |
|---|---|---|---|
| H1-a Semantic | embedding 類似度 | 概念近接を捕える | コスト中、誤検出 |
| H1-b Structural | 共通親・共通リンク先 | 軽量、確定的 | 表層的 |
| H1-c Temporal | 同時期に編集 | 思考の連続性 | 偶然の同時編集に弱い |
| H1-d Lexical | 共通語・固有名詞一致 | 軽量、理解容易 | 同義語に弱い |
| H1-e Hybrid | a+b+c+d をスコア合算 | 精度最高 | パラメタ調整が必要 |

### H2. 表示方法

| 案 | 概要 | 良い点 | 悪い点 |
|---|---|---|---|
| H2-a 候補 inbox | 「リンク候補 7 件」が scratch に貯まる | 後でまとめ処理 | inbox 詰まり |
| H2-b 仮リンク（pending） | 透明線で表示、確定で実線化 | 視覚的、即判断 | 画面ゴチャゴチャ |
| H2-c hover オーバーレイ | ノード hover で「このノードに似た 3 件」 | 軽快 | 能動的に見ないと気づかない |
| H2-d daily report | 朝に「昨日の編集と関連しそうなリンク N 件」 | 習慣化しやすい | 即時性なし |
| H2-e サイドバー | 常時表示の「今のノードの関連候補」 | 気づきやすい | 場所占有 |

**観察**: H2-a + H2-d の組み合わせが最も無理がない（inbox に貯めて朝にレビュー）。

### リンク種別の自動分類

候補リンクに「種類」を AI が推定:
- 関連 / 反例 / 派生 / 引用 / 矛盾 / 同義
- 種類があると merge（D1）判定との連携も楽

## C5. insight 抽出

マップ全体から「あなたが気づいていないパターン」を発見する。

### H3. スコープ

| スコープ | 計算量 | 出てくる insight |
|---|---|---|
| 単一ノード | 小 | そのノードの含意・盲点 |
| Subtree | 中 | テーマ内の偏り・ギャップ |
| Workspace | 大 | プロジェクト横断パターン |
| 全 vault 横断 | 巨大 | 研究人生レベルの傾向 |

→ 段階的にスコープを上げる UI が必要。

### insight の例

- 「過去 1 ヶ月、`方法` ノードは増えたが `結果` ノードが増えていない」
- 「proposal が 12 個ある中で 3 個だけ decision に進んでいる」
- 「最近 `不安` というキーワードを含むノードが増えている」
- 「subtree A と B で同じ用語が違う意味で使われている」
- 「決定済みノードのうち N1 触れる議論を経ていないものが N 件」

### H4. insight の保存

| 案 | 場所 | 良い点 | 悪い点 |
|---|---|---|---|
| H4-a daily report ノード | `insights/2026-04-16` のような時系列 | 振り返り資産化 | 数が多くなる |
| H4-b 専用 insight ペイン | サイドバー常時表示 | 即気づき | 邪魔 |
| H4-c 通知 | OS 通知 / メール | プッシュ強い | うるさい |
| H4-d ノード属性 | 関連ノードに「AI insight: ...」属性 | 文脈密着 | 散らばる |
| H4-e weekly digest | 週次 1 通の総括 | 集約された価値 | 即時性なし |

**観察**: H4-a + H4-e が「報告書を出す秘書」モデルと整合。

### N1/N9 との緊張

C5 は **「あなたはこう感じている」「あなたはこう考えている」を断言してはいけない**。
出力は必ず **「観察」レベル** に留める（「N が小さいノードが増えています」は OK、
「あなたは焦っています」は NG）。

## C7. 要約の階層化

同じ subtree を 1 行 / 3 行 / 1 段落 / topic-list で切替表示。

### H5. 粒度設計

| 粒度 | 用途 | 文字数目安 |
|---|---|---|
| label | hover ツールチップ | 〜30 字 |
| 1 行 | リスト表示 | 〜80 字 |
| 3 行 | カード表示 | 〜200 字 |
| 1 段落 | プレビュー | 〜500 字 |
| topic-list | 章節骨子 | 5〜10 項目 |
| full | 子孫全展開 | 制限なし |

→ ホットキー or zoom レベルで自動切替（map_views と接続）。

### H6. キャッシュ戦略

| 案 | 概要 | 良い点 | 悪い点 |
|---|---|---|---|
| H6-a 都度生成 | 表示時に毎回 LLM 呼び出し | 常に最新 | 高コスト・遅い |
| H6-b 編集時生成 | ノード保存時に要約も更新 | 表示は高速 | 古い要約が残る |
| H6-c 定期再生成 | 1 日 1 回バッチ | コスト低 | 半日古い |
| H6-d Lazy + invalidate | 表示時にチェック、子孫変更があれば再生成 | バランス良 | 実装やや複雑 |
| H6-e ローカルモデル即時 | 軽量 LLM で都度、たまに重い LLM で再 | 実用的 | 品質ばらつき |

### N12 との緊張

要約は **下書きを「綺麗に」してはいけない**。
「下手に書く体験」を侵食しないよう、要約は **別レイヤ** に保存し、
原文は絶対に書き換えない。

## C10. 自走 AI（夜間整理）

夜中にマップを掃除・整理・要約する。本ブレストの中で最も野心的、最も危険。

### H7. トリガ

| 案 | 概要 | 良い点 | 悪い点 |
|---|---|---|---|
| H7-a cron（毎晩 2:00 等） | 固定時刻 | 予測可能 | 編集中に当たることも |
| H7-b idle 検出 | 無操作 N 分後 | 邪魔しない | 起動タイミング不安定 |
| H7-c 閾値超過 | scratch が N 件超えたら | 必要時のみ | 起動が偶発的 |
| H7-d 手動キック | 「今夜やっといて」ボタン | 完全制御 | 自走感が薄れる |
| H7-e 起動時バッチ | 朝 viewer 開いたら 30 秒で実行 | 必ず実行 | 起動が遅くなる |

### H8. 権限レベル

| レベル | できること | 危険度 |
|---|---|---|
| L8-1 read-only | 読むだけ、insight 出力のみ | 低 |
| L8-2 proposal-only | 提案を inbox に積む | 低 |
| L8-3 write-with-undo | 変更するが 1 アクションで全戻し | 中 |
| L8-4 write-with-diff | 変更し、朝に diff レビュー | 中 |
| L8-5 silent commit | 黙って書き換え | 高（推奨しない） |

**推し**: L8-2 + L8-4 の組み合わせ。
**「夜のうちにこれを提案しました／これを変更しました」を朝に必ず diff で見せる**。

### H9. 朝の事後レポート

「夜のうちに何をしたか」を 1 ノードに集約:
- マージ提案: 3 件（D1 と連動）
- リンク提案: 7 件（C4 出力）
- 孤立ノード警告: 2 件
- 要約再生成: 12 ノード
- 検出 insight: 「proposal 滞留が増加」

→ **viewer 起動時に 1 通だけポップアップ** が現実解。

### H10. 自走と「やらない」境界

| やらせて良い | やらせない |
|---|---|
| 重複 / 孤立 / stale 検出 | ノードの **思想内容** を書き換える |
| リンク候補生成 | リンクを **確定** する（人間承認） |
| 要約の再生成 | 原文の書き換え |
| done→archive 移動候補の提示 | 自動 archive（誤判定で消える） |
| 用語の不統一指摘 | 用語の **強制統一** |
| 「気づき」観察 | 「あなたはこう考えている」断言 |

**N1 思考の核心 / N7 創発の場 / N9 内省 / N11 創造的逸脱** には触らない。

## 4 候補の依存関係

```
C7 要約 ─┬→ C5 insight（要約があるとマップ全体読みが安く済む）
         └→ C10 自走 AI（夜間に要約再生成）

C4 link 提案 ─→ C10 自走 AI（夜間に link 候補を貯める）

C5 insight ─→ C10 自走 AI（夜間に insight を計算）
```

→ **C7 要約が基礎レイヤ**。
これがあると C5/C10 のコスト・品質が大きく改善する。

## 既存 D 系障害との対応

`automation_obstacles/` の D 障害群（衛生）と本候補の対応:

| 障害 | 対応候補 |
|---|---|
| D1 重複検出 | C4 link 提案（同義リンク種別） + C10 自走 |
| D2 孤立ノード | C10 自走 |
| D3 stale | C5 insight + C10 自走 |
| D4 自動アーカイブ | C10 自走（提案のみ） |
| D5 健全性スコア | C5 insight |
| D6 過深ツリー | C5 insight |
| D7 巨大ノード | C5 insight + C7 要約での分割提案 |
| D8 GC UI | C10 自走 + keyboard_modes M11 Cleanup mode |
