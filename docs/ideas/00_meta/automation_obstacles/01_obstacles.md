# 01. 障害の網羅的分類

研究／業務自動化を阻む要因を10カテゴリ・約40項目に展開する。

## A. 認知・人的要因

### A1. 暗黙知の問題
研究者は **頭の中に膨大な文脈** を抱えている。
「あの論文の3章で言ってたあれ」「先週思いついた仮説の続き」が外部化されていない。
AI は外部化された情報しか扱えない → 自動化の入り口で詰まる。

### A2. 白紙の恐怖
最も難しいのは「何かをやる」ことではなく **始めること**。
AI が代わりにやってくれても、最初の指示が出せない。

### A3. 目標の曖昧さ
「論文を書く」は無限に分解できる。完了の定義が曖昧。
タスクを切り出せないので自動化対象を特定できない。

### A4. 自己欺瞞
「論文を 50 本読んでいる」≠「論文を書いている」。
作業のフリをしている時間を AI で効率化しても何も生まない。

### A5. エネルギー管理
深い思考には特定の身体的・時間的条件が必要。
朝の頭が冴えた30分は何にも代えがたい。自動化はこれを増やせない。

### A6. 興味の漂流
研究テーマは時間とともに変わる。
昨日の自動化パイプラインが今日には合わない。

### A7. インポスター症候群
「自分にはこれを書く資格がない」と感じると、AI 出力すら使えない。
精神的障壁は技術では取り除けない。

### A8. 学習機会の喪失
「文献整理」を全部 AI に任せると **読解力そのもの** が育たない。
自動化が長期的な能力低下を招く危険。

## B. AI の限界

### B1. 幻覚（Hallucination）
存在しない論文を引用する、計算間違いを自信満々に出す。
研究では1つの嘘が信用を破壊する。

### B2. 文脈窓の限界
論文1本まるまる読ませて議論しても、別の章を見直すと忘れている。
長文・大量資料との相性問題。

### B3. 専門知識の不足
ニッチな分野ほど AI は弱い。一般論しか出ない。
「言うだけは詳しそう」が一番たち悪い。

### B4. IP 汚染リスク
未公開アイデアを LLM に送る → 学習データに乗る恐れ。
研究の核を出せない。

### B5. 一貫性の欠如
同じ質問でも回答が変わる。論文化の根拠として使えない。

### B6. 「分からない」と言えない
不確実な領域でも自信満々。**メタ認知が無い**ことが致命傷。

### B7. 出力の検証コスト
AI 出力が正しいか検証する時間が、自分でやる時間を超えることがある。
ネット loss になる自動化。

## C. ツール断片化

### C1. データ分散
Zotero に文献、Overleaf に原稿、Slack に議論、Notion にメモ、
Drive にデータ、IDE にコード、メールに約束。横断検索不能。

### C2. 文脈再構築コスト
ツール A から B に移動するたびに **頭の中で文脈を作り直す**。
「あの件何だっけ」のロード時間。

### C3. インテグレーション地獄
全ツールに API があるが、繋ぐ仕事が永遠に終わらない。
繋いだ瞬間にどれか壊れる。

### C4. 認証・権限の壁
SSO、2FA、API token 管理。自動化しようとした瞬間に認証が切れる。

### C5. 標準化の欠如
学会ごとに投稿フォーマット、助成団体ごとに申請書フォーマット。
正規化が永遠に終わらない。

## D. ワークフロー

### D1. 引き継ぎコスト（人 ↔ AI）
AI に頼んだ → 結果を受け取る → 自分の文脈に戻る。
切替の摩擦が小さくないと、自動化のメリットが消える。

### D2. 検証ループの長さ
書いた → 実験 → 結果 → 解釈 → 書き直し。
1サイクルが数日〜数ヶ月。改善が遅い。

### D3. 並行作業の管理
3本の論文、5つの実験、2つの講義準備。
全部の状態をどこにも記録しないと頭がオーバーフロー。

### D4. 中断・再開
電話・会議・予定外で中断。再開時に前の状態が思い出せない。

### D5. 進捗の見えなさ
研究は「何時間やったか」では測れない。
やる気が出ず、やる気が出ないことを正当化できない。

## E. データ・知識

### E1. フォーマット問題
論文は PDF、図はビットマップ、式は画像、データは Excel/CSV/binary。
意味を持つ構造で機械可読でないものが多すぎる。

### E2. 出典・provenance の喪失
「この数字どこから持ってきたっけ」が頻発。
半年後の自分が再現できない。

### E3. アクセス障壁
ペイウォール、図書館経由、機関認証。
自動化しようとすると DRM に阻まれる。

### E4. 古いデータの腐敗
3年前のスクリプトが動かない。データ形式が変わった。
過去資産の負債化。

### E5. データの私物化
共著者が「自分の手元にしかないデータ」を持っていて、
頼まないと出てこない。

## F. 協調

### F1. 共著者の道具差
自分は M3E、共著者は Word。最終的に Word に揃える。
自動化を強行できない。

### F2. 指導教員の不在 / 多忙
レビュー待ちで止まる。エージェントは指導教員の代わりにはならない。

### F3. 査読の遅さと不確実性
投稿 → 半年 → reject。学習信号が遅すぎる。

### F4. ミーティング駆動文化
非同期で進められず、毎週ミーティングまで決定が止まる。

### F5. 役割境界の曖昧さ
「これは誰がやるんだっけ」が頻発。AI に振りたくても owner 不明。

## G. 品質・検証

### G1. 品質の定義不能
コードは「コンパイルする／しない」だが論文は違う。
自動化目標を立てづらい。

### G2. 評価の主観性
査読者によって評価が分かれる。
自動化された QA は意味あるか不明。

### G3. 後追い検証
書いた時は正しいと思っていたものが、半年後に間違いと分かる。
事前 QA に限界。

### G4. 「良い研究」の暗黙的基準
論文は **書き方の暗黙ルール** に支配されている。
AI は表層は真似られるが本質は外す。

## H. 制約

### H1. プライバシー
被験者データ、医療データ、機密データ。
そもそも AI に渡せない。

### H2. 知的財産
共同研究の成果を勝手に LLM に流せない。
所属組織の規定にも縛られる。

### H3. 倫理審査
データ取扱い規程が AI 経由を想定していない。
新しい運用に IRB の承認が必要。

### H4. 法規制
GDPR / 個人情報保護法 / 各国規制。
自動化フローが越境すると合法性が崩れる。

## I. 管理事務

### I1. 助成金事務
申請書、報告書、経費精算、予算管理。
研究そのものとは無関係なのに時間を吸う。

### I2. 教育義務
講義準備、学生指導、卒論指導。
自動化対象と研究自動化が衝突する。

### I3. 学内会議・委員会
拒否しにくい、減らない、自動化困難。

### I4. メール・スケジューリング
日々の20%以上を消費するのに、自動化が難しい（人間関係が絡む）。

### I5. 身辺管理
出張手配、健診、税務、家事。
研究時間を侵食する。

## J. 持続性

### J1. 燃え尽き
効率化を進めると、空いた時間にさらに仕事を入れがち。
自動化が休息に向かわず、過剰労働に向かう。

### J2. モチベーション維持
研究の意味を見失う瞬間がある。
自動化は意味を与えない。

### J3. 関係性の維持
共同研究者・指導教員・学生との **顔を合わせる時間** は自動化できない。
むしろ自動化で減ると関係が薄まる。

### J4. ツール・パイプラインの保守
作った自動化が壊れる、保守者がいなくなる、依存先が消える。
自動化自体が負債化する。

### J5. キャリアの不確実性
「この自動化を作っても、5年後の自分はそれを使うか」が分からない。
投資判断が困難。

## 障害の優先度マッピング（ヒューリスティック）

縦軸 = 解決時インパクト、横軸 = 解決容易性

```
高        │ A1 暗黙知       │ C1 データ分散
イン      │ B1 幻覚         │ D5 進捗の見えなさ
パクト    │ G1 品質定義     │ I1 事務
          │ H1 プライバシー │ E2 出典管理
          │                 │
低        │ A7 インポスター │ I3 会議
          │ J1 燃え尽き     │ I4 メール
              低 ─── 解決容易性 ─── 高
```

「右上＝速攻で着手」「左上＝戦略的長期投資」「右下＝見送り可」「左下＝諦め」。
