# 研究・ビジネス活動の自動化: 障害と解決のブレスト

研究活動（広義のビジネス活動）を **自動化／半自動化** する上で立ちはだかる障害と、
それぞれに対する解決アプローチを網羅的にブレストする。

M3E は研究思考支援ツールとして、この障害群の **どこにどう介入できるか** を考える素材。

## 方針

- 障害は「困っている人の声」として具体的に書く
- 解決策は1障害につき複数案、採否は決めない
- 「自動化しない方がいい領域」も同等に扱う（自動化原理主義を避ける）
- M3E が直接できることと、外部ツールに任せるべきことを区別

## 全体俯瞰

研究自動化を考える時の **論点フレーム**:

1. **何を自動化したいか** — 思考？情報収集？意思決定？執筆？管理事務？
2. **誰の手間を減らすのか** — 自分／指導教員／共著者／チーム
3. **完全自動 vs 半自動** — AI が全部やる vs 人間判断を残す
4. **失敗の許容度** — 失敗が致命的（捏造論文）vs 軽微（メモ整理）
5. **学習機会の喪失** — 自動化することで身につけ損なう能力は何か
6. **依存リスク** — その自動化が止まった時に研究も止まるか

## 障害カテゴリ俯瞰

| 大分類 | 代表的障害 | 自動化難度 |
|-------|----------|----------|
| 認知・人的要因 | 暗黙知、白紙の恐怖、自己欺瞞 | 高 |
| AI 自身の限界 | 幻覚、信頼性、IP汚染 | 中（改善中） |
| ツール断片化 | データ分散、文脈再構築コスト | 中 |
| ワークフロー | 引き継ぎ、検証、フィードバック遅延 | 中 |
| データ・知識 | フォーマット、出典管理、アクセス権 | 中〜高 |
| 協調 | 共著者、指導教員、査読 | 高 |
| 品質・検証 | 定義困難、後追い検証 | 高 |
| 制約 | プライバシー、IP、ペイウォール | 高 |
| 管理事務 | 助成、報告、教育義務 | 中 |
| 持続性 | 燃え尽き、モチベ管理 | 高 |

## キーメッセージ（ブレスト前提）

- 「自動化＝速くなる」ではない。**自動化＝認知負荷を解放して別のことに使える**
- 全部自動化は不可能・不要。**70点を80点にするのが現実的目標**
- 自動化の最大の敵は AI の精度ではなく、**人間が AI を信用しきれないこと**
- 研究は探索的なので、**自動化対象は探索の周辺（事務・整理・検索）から始まる**
