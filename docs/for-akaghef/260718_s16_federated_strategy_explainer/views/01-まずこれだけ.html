M3E Strategy S16

Neo4j 化ではなく、
「正本の置き方」を組み替える

結論は、SQLite を捨てて Neo4j に移すことではない。Git・Obsidian・provider が所有する情報は source の近くに残す。Neo4j 内では、その materialization と M3E-owned accepted Deep graph の canonical record を role で分ける。

現在地: Phase 0 — 仕様確定、Neo4j 実装はまだ始めない
一つの concern に対して確定権を持つ owner は一つ。Neo4j にあるだけでは正本か派生物か決まらない。owner と record role を見て、書くときは必ず owner へ戻す。

まず覚える三つの判断

1. 正本は局所化

repository の code と設計は Git、Live Markdown は Obsidian、Rapid 文書は M3E-local。全内容を一つの SQLite に押し込まない。

重要度 high難易度 mediumconfidence 1.0

2. Neo4j は二つの role

source-owned record は再構築可能な materialization。M3E-owned accepted graph は Gate 後の canonical runtime。proposal は journal に残す。

重要度 high難易度 mediumconfidence 1.0

3. write は owner へ

人間・UI・AI・bot・CI は同じ Command boundary を使う。Neo4j や cache への raw write は確定変更にしない。

重要度 high難易度 highconfidence .95

今やる

  • canonical owner、identity、authority、proposal journal の契約を固定
  • repository-local semantic source を agent / CI が file read できる specimen を作る
  • record role、rebuild・stale write・exposure・ownership transfer の test case を固定
  • S2 collaboration と S3 save / recovery を優先する

Gate 後にやる

  • 実需 cross-source query を三件採取
  • Neo4j shadow adapter を他候補と比較
  • edition、deployment、backup、authentication を別 Architecture Decision Record で決定
  • record role schema、Cypher schema、canonical subgraph recovery を実装

混ぜると壊れる境界

Scope ≠ authority
Scope は見える範囲と編集範囲。owner と write route は authority root で解決する。authority を跨ぐ reparent は通常 move ではなく、途中状態を journal に残す ownership transfer になる。
Rapid occurrence ≠ Deep entity
Rapid occurrence は文書内 wording・順序・文脈を所有し、Deep entity は文書横断 identity と typed semantics を所有する。many-to-many の entity binding で接続し、同じ本文を二重所有しない。
materialization ≠ canonical record ≠ 射影
source-materialized record は source から再構築できる copy。M3E-owned accepted record は Gate 後の canonical runtime 対象。無修飾の「射影」は Deep から Rapid 文書を切り出す操作で、Mermaid / TOON の一時表現は context projection と呼ぶ。
proposal ≠ accepted assertion
AI や bot の推定は proposal journal へ即時保存する。承認されて owner に適用されるまで、確定 relation として扱わない。

止めるべき failure mode

Dual-canonsource-owned record と Neo4j copy の双方を canonical と扱う。
Graph exposuresource の閲覧権限を超えた relation や metadata が漏れる。
Lost proposalNeo4j 全削除で未承認判断まで消える。
Half transferownership transfer の片側だけが確定する。
