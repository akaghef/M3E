# daily/

**役割**: 日次作業ログ。**追記のみ・改変禁止**。

## 読むタイミング

- セッション開始時（直近 3 日分を読んで経緯を把握）
- 過去の作業経緯をさかのぼる時

## 書き方

- ファイル名: `YYMMDD.md`（例: `260415.md`）
- 日付が変わったら新規ファイル
- 追記のみ。過去の記述を書き換えない

## 収録内容

- 今日の作業内容・成果
- 決定事項
- 発見した問題・TODO

## 置かないもの

- 設計判断の本体 → `09_Decisions/` への ADR
- 未着手タスク → `06_Operations/` or `tasks/`
