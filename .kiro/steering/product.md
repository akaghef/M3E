# Product Overview

M3E is a local-first knowledge base for research and development work. It combines rapid mind-map input, visual collaboration, Obsidian-style Markdown knowledge, and Git-based operational discipline into one map-centered workflow.

The product is built for Akaghef's daily use first: reading, writing, planning, implementation review, handoff, and research synthesis should happen on a shared `map / scope / node` substrate that both humans and AI workers can inspect.

## Core Capabilities

- Structured knowledge editing through `map`, `scope`, `node`, `edge`, and `alias`.
- Rapid-band tree editing with browser-based viewer interaction, local persistence, and recovery-oriented behavior.
- Local-first storage with SQLite-backed workspace data, backups, audit artifacts, and optional sync layers.
- Human + AI coordination through explicit task, handoff, review, and status boundaries.
- External-format integration, especially Markdown / Obsidian-style linear text, without losing M3E structural semantics.

## Target Use Cases

- Daily development coordination across strategy, tasks, handoffs, reviews, and implementation evidence.
- Research knowledge accumulation from notes, papers, discussions, and generated artifacts.
- Conversion between linear text and graph/tree structures for reading, writing, and synthesis.
- Multi-agent or multi-user collaboration where scope ownership, conflicts, and provenance must stay visible.

## Value Proposition

M3E treats the knowledge map as an operational substrate, not a static diagram. The same structure should support thinking, editing, AI collaboration, status tracking, synchronization, and later projection into documents or runtime contracts.

## Current Product Focus

- Active development target: `beta/`.
- Short-term strategy focus: `S2` Team Collaboration, `S3` save/sync/recovery reliability, and `S13` local-first independence.
- Prefer small validated changes over broad rewrites.
