# 03 Runtime Modes — モバイル体験モードの列挙

「モバイルで何をするか」のモード（体験パターン）を網羅。
Pattern B 風の候補列挙、F1-F6 を Md1-Md12 に再配列・拡張。

各モードは **独立に実装可能** で、組み合わせで最終 UX を作る前提。
スマホ / タブレット / ウォッチで適合度が異なるので適用デバイスも明記。

---

## モード一覧サマリ

| ID | モード | 主用途 | 適合デバイス | F* 起源 |
|---|---|---|---|---|
| Md1 | 閲覧モード | 既存マップを見る | スマホ/タブレット | F1 |
| Md2 | 編集モード | テキスト編集 | タブレット主、スマホ可 | F1 派生 |
| Md3 | capture モード | 入力特化、scratch 投入 | スマホ | F2 ⭐ |
| Md4 | 読み上げモード | TTS 連続再生 | スマホ + イヤホン | F4 |
| Md5 | 通知モード | リマインド、簡易応答 | スマホ + ウォッチ | F5 |
| Md6 | 手書きモード | Pencil/Stylus | タブレット | F6 |
| Md7 | 検索モード | 全文検索特化 | スマホ | 新規 |
| Md8 | Widget モード | ロック画面・コントロール | スマホ + ウォッチ | F5 派生 |
| Md9 | 今日のノードモード | ジャーナル風 1 画面 | スマホ | 新規 |
| Md10 | 音声会話モード | AI と口頭対話 | スマホ + イヤホン | 新規（AlgLibMove 親和） |
| Md11 | コーチ通知モード | メタ認知促し | ウォッチ + スマホ | 新規（J5 接続） |
| Md12 | プレゼン補助モード | スマホで他人に見せる | スマホ | 新規 |

---

## Md1. 閲覧モード — 既存マップをタッチで見る

**目的**: PC で作ったマップをスマホで眺める。電車・ベッド・カフェ。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md1.1 タッチ操作 | ピンチズーム / 2 本指パン / 1 本指タップ専用 |
| Md1.2 ノード表示密度 | 全表示 / scope で間引き / Lv1 のみ |
| Md1.3 ナビ手段 | パン+ズーム / リスト / breadcrumb / 検索 |
| Md1.4 縦向き vs 横向き | グラフは横、リストは縦 |
| Md1.5 ノード詳細表示 | inline 展開 / モーダル / 別ページ |
| Md1.6 戻るジェスチャ | iOS 標準スワイプ / カスタム / 戻るボタン |

### バリエーション

- **マップ縮小版**: SVG をそのまま縮小、ピンチで拡大
- **リスト変換版**: マップ構造を縦リストに変換、パンキング不要
- **focus-only 版**: 1 ノード + 子ノードだけ、画面遷移
- **ハイブリッド**: 上半分マップ、下半分選択ノード詳細

### 推し: ハイブリッド（リスト + ノード詳細）+ ピンチでマップ全景切替

**理由**: スマホでフルマップ操作は無理。リスト主、マップ補助。

---

## Md2. 編集モード — モバイルでもテキスト編集

**目的**: 既存ノードのテキスト・タグ修正。深い編集は PC で。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md2.1 編集対象 | テキストのみ / 構造（追加・削除）も / attribute も |
| Md2.2 キーボード | OS 標準 / カスタム数字バー追加 |
| Md2.3 ショートカット | 不可 / 長押しメニュー / カスタムバー |
| Md2.4 リンク作成 | 検索→選択 / @ メンション / 不可 |
| Md2.5 同期タイミング | 即 / 「保存」ボタン / 自動間欠 |
| Md2.6 編集競合 | 楽観的同期 / 編集中はロック / オフラインキュー |

### バリエーション

- **最小編集**: テキスト変更のみ、構造は触れない
- **フル編集**: PC と同等、ただし操作系は別
- **コメント専用**: 既存ノードに「コメント」しか追加できない（安全）
- **音声編集**: STT で音声入力

### 推し: コメント専用 + 最小編集 のハイブリッド

**理由**: モバイルで構造変更は事故りやすい。安全側。

---

## Md3. capture モード ⭐ — 入力に特化した別 UI（F2）

**目的**: 1 タップ起動 → 即入力 → scratch に投入。思考停止で動く。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md3.1 起動経路 | アプリ起動 / 共有シート / Shortcut / Widget |
| Md3.2 入力種別 | テキスト / 音声 / 写真 / URL / 全部 |
| Md3.3 投入先 | scratch flat / scratch 階層 / 指定ノード子 |
| Md3.4 確認動作 | なし即投入 / 1 タップ確認 / プレビュー |
| Md3.5 投入後動作 | 閉じる / 連続投入 / マップに飛ぶ |
| Md3.6 オフライン挙動 | キュー / エラー表示 / ローカル保存 |
| Md3.7 メタデータ | 位置情報 / 時刻 / バッテリー / 端末名 |

### バリエーション

- **テキスト爆速版**: 起動→キーボード即出現→改行で送信→閉じる。3 秒。
- **音声録音版**: 起動→マイク即起動→停止で送信。会話可能。
- **写真+OCR 版**: カメラ→撮影→OCR テキスト+画像 scratch 投入。
- **マルチモーダル版**: 1 画面に文字/音声/写真/URL の 4 ボタン。

### 推し: テキスト爆速 + 音声録音 の 2 モード切替

**理由**: capture_ingest の A1（音声）と A9（モバイル）のクロス点。

---

## Md4. 読み上げモード — マップを耳で聞く（F4）

**目的**: 散歩・通勤で「自分のマップを聞く」。手は使えない前提。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md4.1 何を読むか | 1 ノード / subtree / today / 検索結果 / 全 done |
| Md4.2 読む順序 | DFS / BFS / 優先度順 / ランダム / 古い順 |
| Md4.3 読み上げ内容 | text のみ / details も / コメントも / 全部 |
| Md4.4 速度・声 | 速度可変 / 声色選択 / 多言語切替 |
| Md4.5 操作 | 前へ/次へ/停止のヘッドホン操作 / 音声コマンド |
| Md4.6 マーキング | 「気になった」を口頭でマーク → 後で見る |
| Md4.7 BGM | なし / 環境音 / 音楽との重畳 |
| Md4.8 オフライン TTS | OS 標準（オフライン可）/ クラウド（高品質） |

### バリエーション

- **ポッドキャスト版**: 全 done を順に読む、シンプル
- **会話版**: 「次」「飛ばす」「コメント」などを音声操作
- **AI ナレーション版**: AI が補足コメント挟みつつ読む（C5 insight 接続）
- **学習版**: SRS（J1）と組み合わせ、復習対象を読む

### 推し: ポッドキャスト版 → 会話版 → AI ナレーション版 と段階拡張

**理由**: ポッドキャスト版なら Web Speech API + 既存マップで MVP 可能。

---

## Md5. 通知モード — リマインド・タスク確認（F5）

**目的**: 「触ってないノード」「期限ノード」「スケジュール」を通知。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md5.1 トリガー | 時刻 / 位置 / 滞在時間 / イベント |
| Md5.2 通知先 | スマホ / ウォッチ / 両方 |
| Md5.3 通知内容 | テキスト / 質問 / 写真 / 統計 |
| Md5.4 アクション | 開く / done / スヌーズ / コメント |
| Md5.5 頻度 | 毎日 / 週次 / カスタム / 適応的 |
| Md5.6 静音時間 | 夜間 / 集中時 / 自動学習 |
| Md5.7 通知 UX | バナー / バッジ / 振動のみ |

### バリエーション

- **朝のチェックイン**: 7:00「今日のノード 3 つ」通知
- **夕方の振り返り**: 18:00「今日触ったノード一覧」
- **stale アラート**: D3 と接続「30 日触ってない: 〇〇」
- **コーチ通知**: J5 と接続「最近こんな傾向です」
- **クエスト**: K3「今日のミッション: 〇〇を整理」

### 推し: 朝のチェックイン + stale アラート（週 1）

**理由**: 通知過多は離脱原因。最小から始める。

## Md6. 手書きモード — Pencil/Stylus（F6）

**目的**: タブレットで図示・スケッチ・式書き。研究者の現場感。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md6.1 入力先 | ノードに添付 / ノード本文 / 別ノード型「sketch」 |
| Md6.2 認識 | 手書きそのまま画像 / OCR 経由テキスト化 / 数式認識 |
| Md6.3 描画範囲 | ノード内のみ / マップ全体に書き込み / 別レイヤー |
| Md6.4 ツール | ペン / 蛍光ペン / 消しゴム / 図形 |
| Md6.5 圧力・傾き | 使う / 使わない |
| Md6.6 連動 | 手書き図 → AI で構造解析 → ノード自動生成 |

### バリエーション

- **添付画像版**: 既存ノードに「手書きメモ」画像を添付するだけ
- **sketch ノード版**: 「手書き専用ノード型」を新設
- **マップ書き込み版**: マップ自体に注釈を書き込める（FigJam/Heptabase 風）
- **数式認識版**: LaTeX に変換して text に入れる

### 推し: 添付画像版（MVP）→ sketch ノード版（将来）

**理由**: 手書きは深いトピック。本ブレストでは位置づけだけ。
将来 `idea/handwriting/` で深掘り推奨。

## Md7. 検索モード — 全文検索特化

**目的**: マップが大きくなった時、目的のノードに即到達。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md7.1 検索方式 | 単純 substring / fuzzy / semantic |
| Md7.2 検索対象 | text / details / コメント / attribute / リンク |
| Md7.3 検索 UX | フルスクリーン / モーダル / 上部バー |
| Md7.4 候補表示 | リスト / プレビュー付き / マップハイライト |
| Md7.5 履歴 | 残す / 残さない / 同期 |
| Md7.6 音声検索 | あり / なし |

### 推し: モーダル + プレビュー付き、音声検索オプション

**理由**: モバイルでは検索が主要ナビ手段になる。

## Md8. Widget モード — ロック画面・コントロール

**目的**: アプリを開かずに最低限の操作 / 表示。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md8.1 Widget 種別 | 表示のみ / アクション付き |
| Md8.2 表示内容 | 今日のノード / 統計 / ランダム / 直近編集 |
| Md8.3 サイズ | 小 / 中 / 大 / ロック画面 |
| Md8.4 更新頻度 | リアルタイム / 1 時間 / 手動 |
| Md8.5 OS 対応 | iOS / Android / 両方 |
| Md8.6 ウォッチ Widget | あり / なし |

### バリエーション

- **「今日」Widget**: 今日触るべきノード 3 件
- **「ランダム」Widget**: マップから 1 ノード（J1 SRS 接続）
- **統計 Widget**: ノード数、今週の差分
- **クイック capture Widget**: 1 タップで capture モード起動

### 推し: クイック capture Widget + 「今日」Widget

## Md9. 今日のノードモード — ジャーナル風 1 画面

**目的**: 「今日」だけにフォーカス、認知負荷ゼロ。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md9.1 「今日」の定義 | 今日触ったノード / 今日期限ノード / 今日作成ノード |
| Md9.2 並び | 時系列 / 優先度 / カテゴリ |
| Md9.3 操作 | 閲覧のみ / done チェック / コメント |
| Md9.4 グラフ表示 | あり / なし |
| Md9.5 翌日への引継ぎ | done でない物を明日に / リセット / 残す |

### 推し: 時系列 + done チェック + 翌日引継ぎ

**理由**: G1（デイリー儀式）と接続。

## Md10. 音声会話モード — AI と口頭対話

**目的**: 散歩中に AI が思考パートナーになる。AlgLibMove の認知層。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md10.1 起動 | 音声コマンド / アプリ / イヤホンボタン |
| Md10.2 文脈 | マップ全体 / 今いるノード / 直近編集 |
| Md10.3 出力 | 質問 / 要約 / コメント / 提案 |
| Md10.4 入力 | STT / 直接タッチ |
| Md10.5 履歴 | 自動でマップに保存 / オプトイン |
| Md10.6 privacy | 全マップを LLM 送信 / 該当部分のみ / オフライン LLM |

### バリエーション

- **散歩 AI**: 「今日のマップ要約して」「気になる点ある？」
- **Devil's Advocate 音声**: C1 を音声で
- **Coach 音声**: J5 メタ認知を口頭で促す
- **Brain dump 受け**: ユーザーが話すだけ、AI が構造化

### 推し: Brain dump 受け（MVP）→ 散歩 AI（拡張）

**理由**: 入力が圧倒的に難しい場面（移動中）を AI で楽にする。

## Md12. プレゼン補助モード — スマホで他人に見せる

**目的**: 学会・打ち合わせで自分のマップを相手に見せる、または共有。

### サブ論点

| 論点 | 選択肢 |
|---|---|
| Md12.1 共有方法 | 画面見せる / QR / 一時 URL / AirDrop |
| Md12.2 表示 | 全マップ / 該当 subtree のみ |
| Md12.3 機微情報 | 自動マスク / 手動指定 / 公開専用 vault |
| Md12.4 受け側 | 読み専用 web / アプリ / 画像 |

### 推し: subtree のみ + 一時 URL + 自動マスク（L2 接続）

## モード組み合わせの相性

| 組合せ | 効果 | 例 |
|---|---|---|
| Md3 + Md10 | 移動中フル捕捉 | 散歩で口頭ブレスト |
| Md4 + Md1 | 聞きながら見る | 通勤で読み上げ + 気になったらタップ |
| Md5 + Md9 | 朝の儀式 | 通知 → 今日のノード画面 |
| Md7 + Md2 | 検索→即編集 | 出先で議事録更新 |
| Md8 + Md3 | ゼロタップ捕捉 | Widget タップで即入力 |
| Md11 + Md4 | 振り返り | コーチ通知から読み上げ |

## デバイス別優先モード

| デバイス | 優先モード |
|---|---|
| **スマホ** | Md3 capture, Md4 読み上げ, Md1 閲覧, Md9 今日 |
| **タブレット** | Md2 編集, Md6 手書き, Md1 閲覧, Md12 プレゼン |
| **ウォッチ** | Md5 通知, Md8 Widget, Md11 コーチ |
| **イヤホン+スマホ** | Md4 読み上げ, Md10 音声会話 |
| **車（CarPlay/Android Auto）** | Md4 読み上げのみ（運転中） |
| **E-Ink（Kindle 等）** | Md1 閲覧（epub 経由、Pf8） |

## 横断的観察

- **モード間切替が多すぎると認知負荷** — 1 アプリ内で 12 モードは多すぎ。
  「主要 4 モード（capture / 閲覧 / 読み上げ / 通知）」を表に出し、残りは設定の奥。
- **音声系（Md4/Md10）は M3E の独自性が出る領域** — 他のメモアプリにない。
  競争優位を狙うならここ。
- **Md3 capture は最優先** — F2 が ⭐ なのは正しい。MVP 候補筆頭。
- **Md6 手書きは別ブレスト** — 設計が深く、本ブレストでは触りだけ。
- **Md9 今日のノードはジャーナル系の入口** — G1 デイリー儀式と統合可能。
- **Md12 プレゼン補助は L 系（privacy）と直結** — 機微情報マスク必須。
