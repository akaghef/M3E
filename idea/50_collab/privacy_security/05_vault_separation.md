# 05. Vault 分離 — 公開 / 私用 / 機密のレーンをどう作るか

00_topic_pool L4「マルチ Vault 切替」を深掘り。
**異なる機微度の情報を物理的・論理的に分離** する選択肢を並べる。

## V1-V10. 分離粒度の選択肢

### V1-V4. 単一 workspace 内の vault 分離

同じ `m3e-map.json` 内で論理分離する案。

| ID | 方式 | 内容 | 良い点 | 悪い点 |
|---|---|---|---|---|
| **V1** | タグ namespace | `vault:public` `vault:secret` 等のタグ | 既存タグ機構流用 | UI 隠ししても DB 漏洩 |
| **V2** | subtree 配下 | `/private/` `/secret/` ノード以下 | 階層的、視覚的明確 | ルート規約必要 |
| **V3** | attribute フラグ | `sensitivity: S4` 等の属性 | 細粒度、機微度と統合 | UI 表示制御が複雑 |
| **V4** | ノード type | `node.type = "secret"` | 型として明確 | 既存 type と衝突 |

### V5-V7. 複数 workspace 分離

別ファイル / 別キーで物理分離する案。

| ID | 方式 | 内容 | 良い点 | 悪い点 |
|---|---|---|---|---|
| **V5** | workspace 別 | `m3e-public.json` / `m3e-secret.json` | 物理分離、漏洩独立 | 切替操作必要、横断検索不可 |
| **V6** | キー別 | 同じファイル内に複数キーで暗号化された領域 | 1ファイル維持 | 実装複雑 |
| **V7** | デバイス別 | 機密 vault は特定端末のみ | 最高安全 | 利便性大幅低下 |

### V8-V10. UX 切替

| ID | 切替方式 | 内容 | 良い点 | 悪い点 |
|---|---|---|---|---|
| **V8** | 手動切替 | 明示ボタン / コマンドで vault 変更 | 透明、明確 | 操作負荷 |
| **V9** | 文脈自動 | URL / 時間帯 / 場所 で自動切替 | 摩擦最小 | 誤切替リスク |
| **V10** | session 分離 | ブラウザタブごとに別 vault | 並列作業可能 | リソース消費、混乱 |

V1-V3 は実質的な防御にならない（同じファイル内なので暗号鍵を持つ攻撃者には全部見える）。
**真の分離は V5 別 workspace 以降**。

ただし **段階的分離** は意味がある:
- S1-S2: V1 タグ（軽い分離）
- S3: V2 subtree（明示的分離）
- S4-S5: V5 別 workspace（物理分離）

## 既存 workspace パラメータとの整合

M3E は既に `?ws=foo` でworkspace 切替がある（recent commits: "URL params + standalone Home"）。
これを vault 機構に拡張する案:

| ID | 拡張方法 | 内容 |
|---|---|---|
| **W1** | workspace = vault | 既存の workspace を「機微度別 vault」として再解釈 |
| **W2** | workspace 内 vault | workspace の中に vault サブ概念を追加 |
| **W3** | 別概念 | vault は別レイヤとして workspace と直交 |

推し: **W1（既存機構の拡張）** がシンプルで policy_privacy 整合（複雑化を避ける）

## L4 の典型的 vault 構成

00_topic_pool L4「公開 / 私用 / 機密で分離」を具体化:

| Vault | 機微度 | 想定内容 | 暗号化 | AI 送信 | 同期 |
|---|---|---|---|---|---|
| **vault:public** | S1 | 文献まとめ、講義ノート | 不要 | 自由 | 通常 |
| **vault:work** | S2-S3 | 研究メモ、論文ドラフト、共同研究者 | 推奨 | マスク後 | E2E 暗号 |
| **vault:secret** | S4 | 査読、被験者、予算 | 必須 | ローカル LLM のみ | E2E 暗号 |
| **vault:credentials** | S5 | API キー、パスワード | 必須 | 送信禁止 | デバイス限定 |
| **vault:personal** | S2-S3 | 日記、感情、人間関係 | 推奨 | 警告付き | E2E 暗号 |

5 vault が多すぎるなら 3 vault（public / work / secret）に集約も。

| 案 | vault 数 | 良い点 | 悪い点 |
|---|---|---|---|
| **VC-a 3 vault** | public / private / secret | シンプル | 細分化要求あれば不足 |
| **VC-b 5 vault** | 上記5種 | 表現力高 | 切替負荷 |
| **VC-c 多数（自由）** | ユーザー定義 | 柔軟 | 統制不能、判断疲れ |

推し: **VC-a 3 vault スタート、後で拡張**

## 横断検索・横断ビューの問題

vault 分離すると **横断検索** が困難になる:

| 案 | 内容 | 良い点 | 悪い点 |
|---|---|---|---|
| **X1** | 横断不可 | 各 vault 内のみ検索 | 安全 | 不便 |
| **X2** | 横断可（明示） | パスフレーズで全 vault 開いた時のみ | 安全 | 操作負荷 |
| **X3** | 構造横断、内容ローカル | リンク・タグだけ全 vault 横断、本文は所属 vault 内 | バランス | 実装複雑 |
| **X4** | メタデータ index 共有 | タグ・タイトル等は共通 index | 検索可 | 部分漏洩 |

推し: **X2 + X3 の組合せ**（デフォルトはローカル、明示 unlock で横断）

## 切替時の操作デザイン

V8 手動切替の UI 候補:

| ID | UI | 良い点 | 悪い点 |
|---|---|---|---|
| **VS1** | 右上 vault selector dropdown | 標準的 | クリック2回 |
| **VS2** | URL パラメータ `?vault=secret` | プログラマティック | 共有時注意 |
| **VS3** | キーボードショートカット | 速い | 覚える必要 |
| **VS4** | コマンドパレット | 検索可 | 操作多 |
| **VS5** | 起動時選択画面 | 明示的 | 起動時のみ |
| **VS6** | vault badge 常時表示 + クリックで切替 | 視認性高 | 画面占有 |

推し: **VS6 常時表示 + VS3 ショートカット** の組合せ

### 切替時の安全動作

vault 切替時に **何が起きるか**:

| 動作 | 内容 |
|---|---|
| 現 vault のメモリクリア | secret → public に戻る時に必ず |
| パスフレーズ要求 | より高機密 vault に入る時 |
| 履歴クリア | undo 履歴に他 vault 情報を残さない |
| 通知 | 「いま vault: secret」を視覚通知 |
| audit log | 切替を記録 |

## V11-V15. 「壁」の堅さの選択肢

vault 間の **データ移動** をどこまで許すか:

| ID | 方式 | 良い点 | 悪い点 |
|---|---|---|---|
| **V11** | 完全隔離 | 移動不可 | 安全 | 不便、コピペで結局漏れる |
| **V12** | 一方向（昇格のみ） | 公開→機密は OK、逆は不可 | 安全寄り | 機密→公開できず使えない |
| **V13** | 一方向（降格のみ） | 機密→公開は OK、逆は不可 | 公開時クリーンアップ強制 | 機密に追加できず |
| **V14** | 双方向（警告付き） | 移動可、確認 | 柔軟 | 確認疲れ |
| **V15** | 自動降格（時限） | 半年後に査読中→公開可へ | 自動化 | 設定ミスで早期漏洩 |

推し: **V14 双方向 + 警告 + V15 時限** の組合せ

## V16-V20. デバイス分離（V7 詳細）

機密 vault を特定端末のみに置く案:

| ID | 方式 | 内容 |
|---|---|---|
| **V16** | desktop only | secret vault はデスクトップのみ、モバイル不可 |
| **V17** | air-gapped | sync しない、手動 export/import のみ |
| **V18** | hardware key 必須 | YubiKey 等を挿してる時のみ復号 |
| **V19** | session 限定 | 起動時パスフレーズなし → public のみ |
| **V20** | 場所限定 | IP / WiFi で勤務先のみ機密 vault 開ける |

## キーポイント

- **真の分離は V5 別 workspace（別ファイル + 別キー）以降** のみ
- V1-V4 同一ファイル内分離は事故抑制止まり、本物の防御ではない
- 段階的に **S1-S2:V1 / S3:V2 / S4-S5:V5** の組合せが現実的
- 既存 workspace パラメータ（W1）の拡張がシンプル
- 横断検索は **X2 明示 unlock + X3 構造のみ横断** が安全と利便のバランス
- vault 移動（V11-V15）は **V14 双方向 + V15 時限** で査読論文等の自然な公開化に対応
- credentials は **V16 desktop only + V17 air-gapped** が妥当
