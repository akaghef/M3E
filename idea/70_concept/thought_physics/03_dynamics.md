# 03. 動的挙動 — 引力・反発・冷却・流体

物性（02）を持ったノードが **時間とともにどう動くか**。
本ファイルは "挙動" のレイヤを扱う。
創発現象（気象予報・地震など）は次ファイル（04）で扱う。

## D1. 引力（Attraction）

ノード同士が引き寄せ合う条件。

### D1.1 引力源の選択肢
| ID | 引力源 | 強度の決まり方 |
|----|--------|----------------|
| D1.1.a | リンク（既存edge） | edge の重み |
| D1.1.b | 共通タグ | 共通タグ数 |
| D1.1.c | 親子関係 | 距離一定（バネ的）|
| D1.1.d | AI 類似度 | 意味類似度スコア |
| D1.1.e | 同温度 | 温度差が小さい同士 |
| D1.1.f | 共起編集（同セッションで触られた）| 共起頻度 |

### D1.2 引力の到達距離
- 全ノード ↔ 全ノード（O(n²)、重い）
- 近傍ノードのみ（半径 r 内）
- リンクのある相手のみ（疎、軽い）

### D1.3 推し度メモ
- 軽くて分かりやすい: D1.1.a + D1.1.c（リンク + 親子のみ）
- 創発を狙う: D1.1.d（AI 類似度）+ D1.1.f（共起）

## D2. 反発（Repulsion）

ノード同士が押しのけ合う条件。

### D2.1 反発源
| ID | 反発源 | 意味 |
|----|--------|------|
| D2.1.a | 全ノード一律 | 衝突回避（force-directed の標準）|
| D2.1.b | 異なる電荷（N4） | 立場の違いを物理表現 |
| D2.1.c | 異なる type | 種別ごとに領域分離 |
| D2.1.d | 矛盾タグ | "賛成派" vs "反対派" タグの分離 |

### D2.2 反発の強度カーブ
- 距離の逆二乗（自然）
- 距離の逆数（穏やか）
- 閾値外は 0（局所反発）

## D3. 冷却・凝固

時間とともに温度が下がるプロセスとその効果。

### D3.1 冷却トリガ
- 時間経過のみ
- "触られない" 期間
- マップ全体のアクティビティ低下（外気温連動）

### D3.2 凝固後の挙動
| ID | 挙動 | 説明 |
|----|------|------|
| D3.2.a | 動かなくなる | 物理シミュからは外す（軽量化） |
| D3.2.b | 半透明化 | 視覚的に控えめに |
| D3.2.c | 地層へ沈む | 画面下部に堆積 |
| D3.2.d | 化石化（→ 04 P6） | 専用「化石」アイコン |

### D3.3 再加熱（解凍）
- ユーザーが触ると即座に再加熱
- 周囲のノードが熱くなると伝導で温まる
- AI が "発掘候補" として一時的に温める

### D3.4 冷却の例外
- ピン留めノード = 冷却対象外
- 重要マーク付き = 冷却速度を遅くする
- ゴール／長期目標タグ = 冷却しない

## D4. 沈降・浮上

重力（N1）の結果としてのレイアウト変化。

### D4.1 沈降の発動条件
- 温度が閾値以下
- 重力スコアが閾値以上
- 一定期間アクセスなし

### D4.2 沈降の表現
| ID | 表現 | 説明 |
|----|------|------|
| D4.2.a | y 座標を下げる（連続的）| 滑らかなアニメ |
| D4.2.b | 別レイヤへ移動（地層モード）| 表層・深層を切替表示 |
| D4.2.c | リスト下部へ（リストビューの場合）| グラフでない時の代替 |
| D4.2.d | 専用「アーカイブ井戸」へ落下 | アニメ重視 |

### D4.3 浮上トリガ
- 触る・編集する
- AI による発掘提案
- 検索ヒット
- リンク追加（他ノードに引き上げられる）

### D4.4 沈降と既存 archive 機能の関係
- 既存 archive: 明示的にユーザーが移動
- 沈降: 自動・連続的・可逆
- 二段階モデル案: 沈降（自動・可逆） → archive（手動・明示）

## D5. 流体としての「思考の流れ」

複数ノードを流体として捉えるアプローチ。

### D5.1 流れの定義
- D5.1.a: 編集タイムライン上で連続して触られたノード列を "流れ" とする
- D5.1.b: リンクの方向性を "流路" とする
- D5.1.c: AI が "話題の遷移" を検出して流線を描く

### D5.2 流れの可視化
| ID | 表現 | 説明 |
|----|------|------|
| D5.2.a | 矢印アニメ | ノード間に流れる粒子 |
| D5.2.b | 等高線 | 関心の高さを地形図のように |
| D5.2.c | 色相グラデーション | 上流 → 下流 で色変化 |
| D5.2.d | 川の太さ | 流量 = 編集頻度 |

### D5.3 渦・滞留の検出
- 同じノード群を行き来している = "渦"（思考の停滞）
- 一方向に進んでいる = "順流"（思考の進展）
- → 04 の創発現象として通知化可能

## D6. 計算タイミング

### D6.1 リアルタイム
- 描画ループ内で物理計算
- メリット: 滑らか、生き物感
- デメリット: 重い、バッテリ消費

### D6.2 バッチ
- 数秒〜数分ごとに一括計算
- メリット: 軽量、安定
- デメリット: アニメが粗い

### D6.3 オンデマンド
- 「物理ビュー」を開いた時のみ計算
- メリット: 通常時の負荷ゼロ
- デメリット: 創発現象（気象予報）には不向き

### D6.4 ハイブリッド推奨
- 物性（属性）は永続化、リアルタイム算出不要
- 描画上の力場はリアルタイム or バッチ
- 創発現象（予報・地震）は日次バッチで十分

## D7. シミュレーションループの選択肢

### D7.1 物理エンジン
- d3-force（Web 標準・軽量）
- matter.js（剛体物理・重め）
- 自前実装（用途特化、軽量）

### D7.2 ループ単位
- フレーム単位（60fps）
- 秒単位（1fps）
- イベント駆動（編集時のみ更新）

### D7.3 シミュレーション停止条件
- "落ち着いた"（速度が閾値以下）と判定したら停止
- ユーザーが操作したら再開
- 永続的に動かす（生物感重視）

## D8. ユーザー操作との衝突

### D8.1 ドラッグ vs 物理
- ユーザーがドラッグ中は物理停止
- リリース後に物理再開（ばね的に元に戻る or 新位置に固定）
- ピン留め機能で物理対象から外す

### D8.2 オフ機能
- "物理 OFF モード" で完全静止
- 視覚的物性（色・大きさ）だけ残す案
- ユーザーごとの好み差は大きいので必須

## 横断観察

- **D1.1.f（共起編集）は M3E 独自の強み** — 編集セッションを記録できれば実装可能
- **D5（流体）は野心的だが研究的価値が高い** — "思考の流れ" 自体が研究対象になる
- **D6.4（ハイブリッド）が現実解** — リアルタイム全部はコスト高すぎ
- **D3 + D4 + 既存 archive で三段階モデル** が綺麗:
  1. 冷却（自動）
  2. 沈降（自動・可逆）
  3. archive（手動・明示）
- **D8 のオフ機能は最初から必須** — 物理感が苦手な人に強制すると離脱
