# 思考の物理学メタファー（Thought Physics）

ノードに「重力」「磁力」「温度」「鮮度」などの **物性** を持たせ、
マップ全体を **物理現象として観察する** ための隠喩設計ブレスト。

研究者の思考補助ツールである M3E に対し、
「気象予報」「地震」「冷却・凝固」「流体」のような自然現象アナロジーを与えて
**マップの状態を直観的に把握** することが目的。

## 方針

- 採用判断はしない（複数案を並列で残す）
- 実装詳細は決めない（force-directed layout や属性駆動アニメは "参考" 程度）
- **メタファとしての設計** が主、実装は二次
- project_projection_vision（半年で「世界モデル→射影」実用化）に対しては
  **遊び寄り** の位置づけ。ただし副産物として「マップ衛生」「振り返り」に効く可能性あり
- 既存 attribute システム（status/temperature/freshness 等）との接続を前提

## 全体俯瞰

```
                ┌─ 静的物性（02）─┐
ノード属性 ──→ │ 重力/磁力/温度/  │ ──┐
                │ 電荷/粘性/質量   │   │
                └────────────────┘   │
                                     ↓
                ┌─ 動的挙動（03）─┐
                │ 引力/反発/冷却/  │
                │ 沈降/流体/凝固   │
                └────────────────┘
                                     ↓
                ┌─ 創発現象（04）─┐
                │ 気象予報/地震/   │ ──→ ユーザー体験
                │ 季節/生態系      │
                └────────────────┘
```

物性 → 挙動 → 創発 → 体験 の階層で考える。
下層（物性）は属性として保存可、上層（創発）は計算結果。

## キーメッセージ

1. **物理隠喩は "観察ツール"** — レイアウトの最適化ではなく、状態の俯瞰が本命
2. **物性は属性、現象は計算** — 永続化するのは物性のみで現象は派生
3. **遊びと実用の二層構造** — 表層は遊び、深層は「マップ衛生」と「振り返り支援」
4. **既存 stale 検出 / freshness 系と連続的** — 全くの新機能ではなく、既存の延長で実装できる
5. **「予報」「地震」は副産物として研究的価値あり** — メタ認知ログ（J5）と接続可能
