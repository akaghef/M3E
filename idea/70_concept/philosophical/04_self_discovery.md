# Q4. 「マップで自分を発見する」体験設計

マップを **作業台** ではなく **鏡** として使う体験を設計する。
書いた内容から **「自分が知らない自分」を発見する** 仕掛け群のブレスト。

automation_obstacles N9 自己観察 / time_history I5 履歴 AI 要約 /
ai_agent_deep C5 insight 抽出 と接続。

## なぜ自己発見か

- 思考を外化する最大の価値は **「書いて初めて気づく」** ことにある
- 書いた本人は **書いた瞬間** には気づけない（事後に気づく）
- マップは時系列・関係性のメタデータを持つ → 自己観察に適している
- AI を「答えを出す存在」ではなく「鏡を提示する存在」と再定義する
- project_alglibmove_dogfood の「認知層が無い」批判への正面回答

## ユースケース

- **U1.** 月末に「今月、自分は何に時間を使っていたか」が分かる
- **U2.** 「自分が無意識に避けているテーマ」を AI が指摘する
- **U3.** 半年前の自分の予測が当たったか/外れたかを振り返る
- **U4.** 「同じ言葉を 50 回書いている」癖を発見する
- **U5.** 共著者から「あなたって○○な研究者だよね」を客観的にデータで確認
- **U6.** 助成申請書を書く時に「自分の研究の物語」が見える
- **U7.** 学生が自分の成長曲線を年単位で見る
- **U8.** メンタル不調の予兆（ノードの感情語の偏り）を察知する

## サブ論点

### Sd1. 「鏡」としての AI モード

| 案 | AI の振る舞い |
|---|---|
| Sd1.a | 答えを出さず質問だけ返す |
| Sd1.b | ユーザーの言葉を要約して "あなたはこう書いている" と返す |
| Sd1.c | パターンを指摘する "あなたは X が出ると必ず Y を書く" |
| Sd1.d | 矛盾を指摘する "ノード A と B は反対のことを言っている" |
| Sd1.e | 沈黙を指摘する "この領域は 30 日触れていない" |
| Sd1.f | 偏りを指摘する "ノードの 80% が悲観語" |
| Sd1.g | 成長を指摘する "1 年前と比べて○○の語彙が増えた" |

組み合わせ可。推し: **Sd1.b + Sd1.c + Sd1.g**（押し付けがましくない、観察ベース）。

### Sd2. 自己観察ダッシュボード

| 案 | 内容 |
|---|---|
| Sd2.a | "今月の自分" レポート（書いた量、テーマ分布、感情語） |
| Sd2.b | "頻出語ランキング"（自分の語彙特徴） |
| Sd2.c | "未解決問題の年齢分布"（古い問いほど赤） |
| Sd2.d | "成長グラフ"（テーマ別ノード密度の時系列） |
| Sd2.e | "活動ヒートマップ"（曜日×時間帯） |
| Sd2.f | "コラボ密度"（共著者ノードとの距離） |
| Sd2.g | "自分のテーマ星座"（タグ間の関係構造） |

automation_obstacles N9 と直結。

### Sd3. 偶然との出会い演出

自己発見は計画では起きない。**偶然性** を仕込む。

| 案 | 仕掛け |
|---|---|
| Sd3.a | 起動時に "今日の 1 ノード" をランダム表示 |
| Sd3.b | "こんな日に書いた" 1 年前リマインド |
| Sd3.c | ランダム 2 ノードを並べて「関係あるかも？」プロンプト |
| Sd3.d | "未読リンクツアー"（リンクを辿る一筆書き） |
| Sd3.e | "忘れていたタグ" を週次で 1 個提示 |
| Sd3.f | 検索結果に "意外な関連" を 1 件混ぜる |

Q2 忘却機能 (Fg4) と接続。

### Sd4. メタ認知ログ

書く時に "気持ち" を残す仕組み。
cognitive_science J5 と接続。

| 属性 | 例 |
|---|---|
| `confidence: 1-5` | この内容にどれくらい確信があるか |
| `emotion: hopeful/anxious/...` | 書いた時の気分 |
| `energy: 1-5` | 書いた時のエネルギーレベル |
| `with_whom: solo/discussion` | 1 人 vs 議論の中で |
| `triggered_by: paper/thought/talk` | きっかけ |

→ 後で「不安な時に書いたノードは 80% が破棄されている」のような自己分析可能。

### Sd5. 他者視点を借りる

| 案 | 視点 |
|---|---|
| Sd5.a | "指導教員ペルソナ" でマップを読む（C9 と接続） |
| Sd5.b | "5 年後の自分" 視点 |
| Sd5.c | "学生視点" |
| Sd5.d | "査読者視点" |
| Sd5.e | "競合研究者視点" |
| Sd5.f | "自分の最初のメンター" 視点 |

各視点は AI に system prompt で persona 注入。

### Sd6. 「気づき」のキャプチャ

発見した瞬間を残す仕組み。

- Sd6.a. "Insight" 専用ノードタイプ（気づきだけ集約）
- Sd6.b. 通常ノードに `was_insight_at: <date>` を付ける
- Sd6.c. 気づきタイムライン（時系列で並べる）
- Sd6.d. "気づきの種類" 分類（パターン発見/矛盾発見/沈黙発見/etc）
- Sd6.e. 気づきの頻度ダッシュボード（自分は週 N 個気づいているか）

### Sd7. 「答えを出さない」AI 設計

哲学的中核。AI に答えを言わせないための工夫:

| 案 | 制約 |
|---|---|
| Sd7.a | system prompt: "断定するな、観察を述べよ" |
| Sd7.b | system prompt: "質問だけで答えよ、最大 3 個" |
| Sd7.c | 答えを出すモード / 鏡モード を明示切替 |
| Sd7.d | 鏡モードでは「私の意見」を含む語を禁止 |
| Sd7.e | 鏡モードの応答は必ず "?" で終わる |

推し: **Sd7.c + Sd7.b**（モード分離が一番安全）。

### Sd8. プライバシーとの緊張

自己観察は **最も機微な情報の集合**。

- Sd8.a. メタ認知ログ（emotion 等）はローカルのみ、AI に送らない
- Sd8.b. AI に送る前に感情語を伏字化
- Sd8.c. 自己観察 view 自体に追加の認証要求
- Sd8.d. 共有マップでは自己観察データを必ず除外

policy_privacy 制約と直結。

### Sd9. 鏡モード ON/OFF

哲学押し付けを避けるため切替式に。

- 通常: 普通の作業マップ
- 鏡モード: AI が観察コメントを定期的に出す、自己観察 view が前面化
- 鏡モードの頻度: 月次（Sd2.a）、週次、起動時、明示要求のみ

### Sd10. 「気づき」を強制しない

- Sd10.a. AI からの気づき提示はサイドパネル、main view を侵食しない
- Sd10.b. 「あとで読む」キューに溜める、その場で読まなくていい
- Sd10.c. ユーザーが反応しなくても AI は責めない（押し付け禁止）

## 反対意見

- **R1.** 自己観察は心理セラピーで効果ある一方、ツールでやるのは越境
- **R2.** AI が「あなたはこう」と決めつけるのは不快
- **R3.** メタ認知ログは入力負荷が高すぎる
- **R4.** 統計が出ても「だから何」になる
- **R5.** 自分のことを知りすぎるとマップに書けなくなる（観察される恐怖）

→ 答え:
- R1 → 治療目的ではない、観察道具として位置付け
- R2 → Sd7.b（質問形式）で押し付け回避
- R3 → 任意、デフォルト OFF
- R4 → "気づき" は個人的、外から見て無意味でも本人には意味がある
- R5 → Sd9 鏡モード切替で逃げ道を用意

## 半年実用化への寄与

- **短期**: Sd2.a 月次レポート 1 機能だけでも「自分の研究の物語」が見える
- **中期**: Sd5 他者視点で助成申請書の客観性向上
- **長期**: Sd6 気づきタイムラインが研究人生のメタデータになる

## キーメッセージ

- **AI を「鏡」と再定義する。** 答えを出す存在から観察を提示する存在へ
- **自己観察はオプトイン。** 強制すると逆に書けなくなる
- **メタ認知ログは小さく始める** （confidence と emotion の 2 属性のみで十分）
- **automation_obstacles N9 を実装に降ろす最初の場所** がここ
- **押し付けないことが最重要。** "押し付けない AI" は M3E のブランドになり得る
