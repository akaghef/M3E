## 1. 横断観察

### Cx1. 5 テーマは "未完成性" の異なる側面

| サブトピック | 未完成性の発露 |
|---|---|
| Q1 未完成原則 | **構造** の未完成（穴を残す） |
| Q2 忘却 | **見え方** の未完成（一部を見ない） |
| Q3 時間軸 | **時間** の未完成（永遠に流れる） |
| Q4 自己発見 | **自己理解** の未完成（自分を発見し続ける） |
| Q5 死と再生 | **存在** の未完成（生死のサイクル） |

→ 哲学的に Q1 が基盤、残り 4 つは Q1 の **時間/見え方/自己/存在** への射影。

### Cx2. 共通するメカニズム

5 テーマすべてに共通する設計パターン:

| パターン | 説明 | 各テーマでの現れ |
|---|---|---|
| **段階性** | 二値ではなく多段階 | Uf3 未完成度 / Fg3 忘却段階 / Dr1 死段階 |
| **可逆性** | 戻れることが安心を生む | Uf2 完成解除 / Fg4 思い出す / Dr5 復活 |
| **ON/OFF 切替** | 哲学押し付け回避 | Uf7.d / Fg2 手動 / Sd9 鏡モード |
| **AI と人の役割分担** | AI は提案、人が決定 | Uf6 / Fg2.h / Sd7 / Dr3.g |
| **時間スケール選択** | 即時 / 日次 / 月次 / 年次 | Q3 時間軸 / Fg2.b / Sd2.a / Dr5.b |
| **メタ層と表層** | データは保持、見え方を変える | Fg vs 削除 / Dr1 fossilized |

### Cx3. 共通する反対意見の構造

各テーマに必ず "重い・複雑・押し付け" 系の反対意見がある:
- 「思想が重い」
- 「実装が複雑」
- 「ユーザーが混乱」
- 「既存機能で十分」

→ 共通の答え: **モード切替 + デフォルト OFF + 漸進導入**。

### Cx4. M3E の差別化軸

> 
> - Notion/Obsidian = 整理ツール

## 4. 哲学を機能選別の "物差し" として使う

ブレストでは複数案を並べたが、実運用では **どれを採用するか** の基準が必要。
哲学テーマを物差しに使うと:

- 「この機能は Q1 に反するか？」 → 反するなら却下
- 「この機能は Q4 を強化するか？」 → 強化するなら優先
- 「この機能は Q5 のどこに位置するか？」 → ライフサイクル不整合なら再設計

→ 機能爆発を抑える効果が大きい。
→ feedback_no_interpretation（勝手解釈禁止）と整合: 哲学は判断軸、解釈ではない。

## 6. 未決質問

### Cx12. 最重要の未決

- **Cx12.1.** 哲学を **デフォルト ON** にするか **デフォルト OFF** にするか
  → ON = ブランド強化だが新規ユーザー離脱リスク
  → OFF = 段階導入だがブランド希薄化
  → 中間案: 起動時 1 度 "哲学モードを試しますか？" プロンプト

- **Cx12.2.** 5 テーマすべてを実装するか、**重点投資** か
  → 全部 = 思想完結だがコスト大
  → 重点 = Q1 + Q4 + Q5（最も実用と接続）に絞る

- **Cx12.3.** 哲学を **akaghef 個人** で持つか、**M3E 公式** に出すか
  → 個人 = ブランド一貫性高い、依存リスク
  → 公式 = チーム拡張可、思想薄まりリスク

### Cx13. 設計上の未決

- **Cx13.1.** Q2 忘却 と Q5 死 の境界はどこか（dormant と dead の違い）
