# 05. N4: 講義スライド→マップ自動変換

PowerPoint / Keynote / PDF / Reveal.js 等の講義スライドを M3E マップに自動変換。
N1（教科書マップ化）と対称、入力フォーマットが違うだけ。
slideshow ブレストの「逆方向」（マップ→スライドではなく、スライド→マップ）。

## 目的

- 既存の講義資産を即マップ化
- 教員が新規マップを書かずに M3E を試せる
- 学生が配布スライドを自分のマップに統合できる
- スライドだと失われる「概念間の関係」を回復

## 1. 入力形式（N4.A）

| 形式 | 抽出容易さ | 普及度 | 構造保持度 | 推し |
|---|---|---|---|---|
| N4.A1 PPTX（OOXML） | 高 | 最高 | 中（タイトル/本文/ノート） | ⭐⭐⭐ |
| N4.A2 Keynote | 中（独自形式） | 中 | 中 | ⭐ |
| N4.A3 PDF（テキスト埋込） | 中（位置情報） | 高 | 低 | ⭐⭐ |
| N4.A4 PDF（スキャン）+OCR | 低 | 中 | 最低 | ⭐ |
| N4.A5 Google Slides | 中（API） | 中 | 中 | ⭐⭐ |
| N4.A6 Reveal.js / Markdown | 最高 | 低 | 高 | ⭐⭐⭐ |
| N4.A7 LaTeX Beamer | 高 | 中（学術） | 高 | ⭐⭐ |
| N4.A8 動画（自動文字起こし＋OCR） | 低 | 低 | 低 | ⭐ |

## 2. レイアウト→構造の推定（N4.B）

### N4.B1. 1スライド = 1ノード
- 単純、安全
- 推し（MVP）: ⭐⭐⭐

### N4.B2. タイトル階層を木構造化
- 「第1章」「1.1 節」のタイトルパターンから親子関係推定

### N4.B3. 箇条書き → 子ノード
- スライド内の bullet point を子ノードに展開

### N4.B4. 図表 → 別ノード
- 画像・グラフは独立ノードとして

### N4.B5. ノート（presenter notes）→ details
- スピーカーノートを M3E ノードの details に転記

### N4.B6. AI による意味的グループ化
- スライド本文を読んで、関連スライドを自動リンク
- 重い、品質ばらつき

### N4.B7. 番号・章記法ベース
- 「2.3」「Chapter 4」などの記法をパース

## 3. アセット（画像・動画・数式）の扱い（N4.C）

### N4.C1. 画像はそのまま埋め込み
- マップが重くなる

### N4.C2. 画像は別ストレージ参照
- マップは軽い、配布難

### N4.C3. 数式は LaTeX に変換（OCR）
- Mathpix 等使用

### N4.C4. 動画は URL リンクのみ

### N4.C5. アニメーション・遷移は無視
- スライドネイティブ機能は捨てる

## 4. 編集後のラウンドトリップ（N4.D）

スライド → マップ → スライド の往復:

### N4.D1. 一方向（取り込み後はマップが正）
- 簡単
- 元スライドは捨てる

### N4.D2. マップ編集 → スライド再生成
- slideshow ブレストの「マップ→スライド」と接続
- 元スライドは破棄

### N4.D3. 双方向同期
- 元 PPTX も更新
- 重い、競合多発

### N4.D4. マップ＝編集領域、スライド＝出力
- 推し: ⭐⭐⭐

## 5. メタデータ抽出（N4.E）

- スライドタイトル → workspace 名
- 作成日・更新日
- 著者
- フッター・著作権表記
- スライド番号（順序保持）
- セクション区切り

## 6. 既存 slideshow ブレストとの接続（N4.F）

`idea/slideshow/` は「マップを順序付きで読む（=プレゼン）」の設計。
N4 はその逆方向。両者を組み合わせると:

- スライド取り込み → マップ化 → 編集 → ツアー定義（slideshow） → 再エクスポート
- これが教育コンテンツの編集ワークフロー全体

### 共通化できる論点
- ノード順序の表現（slideshow のトピック C）
- カメラ挙動（slideshow E）
- ナレーション欄（slideshow H）

## 7. ユースケース

### UC1. 教員: 過去スライド資産のマップ化
- 過去5年分の講義 PPTX × 30コマを一括取り込み
- 章節構造で整理 → 講義改善
- 不要重複を発見

### UC2. 学生: 配布スライドを取り込み
- 講義後に配布された PDF を取り込み
- 自分の質問・気づきを子ノードで追加
- 試験前に SRS（J1）で復習

### UC3. オンライン講座作者
- Reveal.js で書いた講座をマップ化
- 受講者向けに構造を見せる
- 質問を集約

### UC4. ゼミ発表のアーカイブ
- 院生の毎週発表 PPTX を蓄積
- 数年分を横断検索可能に
- 過去の議論を再発見

## 8. 失敗モードとリカバリ（N4.G）

| 失敗 | 原因 | 対策 |
|---|---|---|
| OCR 文字化け | スキャン PDF | 教員に手動修正させる UI |
| 階層推定ミス | タイトル記法不統一 | drag でリパース |
| 画像欠損 | 埋込形式不対応 | 警告 + 手動アップロード |
| 著作権侵害 | 商用素材含 | ローカル保存・公開禁止デフォルト |
| 数式崩れ | OCR 限界 | LaTeX 手入力 fallback |

## 論点 ID

- N4.A. 入力形式（PPTX/Keynote/PDF/Reveal.js/Beamer...）
- N4.B. 構造推定（1:1 / 階層 / 箇条書き展開 / AI 意味的）
- N4.C. アセット扱い（埋込 / 参照 / 数式OCR / 動画リンク）
- N4.D. ラウンドトリップ（一方向 / マップ→スライド / 双方向）
- N4.E. メタデータ抽出範囲
- N4.F. slideshow ブレストとの統合度合い
- N4.G. 失敗モード（OCR / 階層 / 画像 / 著作権 / 数式）
- N4.H. バッチ処理（複数ファイル一括 vs 単発）
- N4.I. 取り込み中プレビュー（変換結果を確認しながら調整）
