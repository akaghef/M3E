# 01. 教育用途のコンセプト全体像

M3E を教育コンテキストに転用したとき、何が嬉しく、何が既存と違うのか。
個別機能（N1〜N5）の前提となる、共通の世界観を整理する。

## 1. なぜ教育用途を考えるか

### 1.1 akaghef の文脈
- 大学教員として **教育義務** がある（automation_obstacles I2）
- 講義準備・教材作成・学生指導に時間を取られ、研究時間を圧迫
- M3E を「研究」だけでなく「教育」にも使えれば、二重の dogfood になる
- ただし優先順位は **研究 > 教育**（project_projection_vision: 科研費が最優先）

### 1.2 M3E の特性が活きる場面
- グラフ構造 = 概念間の関係を表すのに自然
- ノード単位の status 管理 = 学習進度管理と相性
- viewer の SVG 描画 = スライドや教材として配布しやすい
- AI agent 連携 = クイズ生成・解説生成と相性

### 1.3 既存ツールとの差分
| 既存ツール | 強み | M3E が補える領域 |
|---|---|---|
| Anki / Quizlet | SRS・暗記特化 | 概念間の関係・全体構造 |
| Notion / Obsidian | ドキュメント | グラフ構造・順序ツアー |
| Moodle / LMS | 成績管理・配信 | 個人の理解構造の可視化 |
| YouTube / Coursera | 動画講義 | 視聴後の自己マップ作成 |
| 教科書 PDF | 静的内容 | インタラクティブ展開 |
| PowerPoint | 線形プレゼン | 非線形ナビゲーション |

## 2. ユースケース具体化

### UC1. 教員が来期の講義を準備
- いつ: 学期前 2週間
- 誰が: 大学教員（akaghef 自身）
- シーン: 過去の講義スライド・教科書・自分の研究ノートをマップに統合し、15回分の講義設計を立てる
- 関連: N4（スライド→マップ）、N1（教科書マップ化）

### UC2. 学生が試験前に理解を整理
- いつ: 期末試験 1週間前
- 誰が: 学部生
- シーン: 講義スライドと教科書を統合した自分のマップを作り、穴のある場所をクイズで特定
- 関連: N1, N2, N5

### UC3. 教員が学生の理解度を把握
- いつ: 中間試験前後
- 誰が: 大学教員
- シーン: 学生10〜100人のマップを横断俯瞰し、共通の躓き箇所を発見
- 関連: N3, N5

### UC4. 自習者がオンライン講座を受講
- いつ: 平日夜・週末
- 誰が: 社会人学習者
- シーン: Coursera の動画を見ながら、章ごとにマップに転記し、後で復習
- 関連: N1, N2, N5

### UC5. 塾講師が個別指導
- いつ: 週次面談
- 誰が: 塾講師
- シーン: 各生徒のマップを見て、躓いている概念に応じた次回課題を決定
- 関連: N3, N5

### UC6. 研究室ゼミでの輪読
- いつ: 週1回のゼミ
- 誰が: 大学院生＋指導教員
- シーン: 教科書の章を院生が分担、各章のマップを共有し議論
- 関連: N1, collaboration ブレストと連携

## 3. M3E 教育版の世界観

### 3.1 「学ぶこと」をマップで表現する
- 概念ノード = 学習対象
- リンク = 概念間の依存関係（A を理解しないと B が分からない）
- ステータス = 理解度（未学習 / 触れた / 演習済み / 説明できる）
- 子ノード = 具体例・派生・応用

### 3.2 「教えること」もマップで表現する
- 教材ノード = 1コマ分の講義
- 子ノード = 各セクション・スライド
- リンク = 講義間の依存
- 演習ノード = 課題・小テスト

### 3.3 学習者と教育者で **同じマップを別ビュー** で見る
- 学習者ビュー: 自分の理解度ハイライト
- 教育者ビュー: 学生群の理解度ヒートマップ
- → N3 で深掘り

## 4. やらない領域（境界）

ここまで広げない、と決めておく:

- **成績管理 / GPA 計算** — LMS の領域、競合しない
- **動画配信プラットフォーム** — YouTube/Vimeo に任せる
- **試験監督・不正検知** — 教育機関の責任領域
- **学籍管理・出席登録** — 別システム
- **公式採点** — 自動採点はあくまで自習用、評価は人間
