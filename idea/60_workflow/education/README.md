# Education / 教育用途 ブレスト

M3E を **教育コンテキスト**（教科書・講義・学生指導・自習）に転用する切り口の網羅。
研究者個人ツールとしての M3E が、教える側／学ぶ側の双方にどう拡張できるかを並べる。

## 方針

- 採否は決めない、複数案を並べる
- 実装詳細は決めない（コード片は 06_mvp_path.md で参考程度のみ）
- 各論点に ID を振る（N1.x 〜 N5.x、横断は X.x）
- ユースケースは「誰が／いつ／どんなシーンか」を最低3個
- 教育は M3E の二次目標（一次は研究・科研費）。ただし dogfood 機会としては大きい

## 想定読者・関係者

- **教師・大学教員** — 講義準備、教材作成、学生指導
- **学生・受講者** — 講義理解、自習、復習、試験対策
- **塾講師・予備校** — 個別指導、進度管理
- **オンライン講座作者** — Udemy/Coursera 型コンテンツ生成
- **企業内研修担当** — 社内教材化、新人オンボーディング
- **akaghef 自身** — 大学教員としての教育義務（automation_obstacles I2）

## キーメッセージ（中間結論）

1. **N1（教科書）と N4（スライド）は対称構造** — 入力フォーマットが違うだけで、出口（マップ化）は共通基盤に集約できる
2. **N2（クイズ）と N5（進度）は閉ループ** — クイズ結果が進度に反映、進度が次のクイズを決める
3. **N3（教師ビュー）はプライバシー設計が最大の障壁** — policy_privacy 制約と直交、L1〜L5 と要連携
4. **教育用途の MVP は「単一学習者の自習」から始めるのが安全** — 多人数・採点・成績管理に踏み込むと一気に重くなる
5. **既存 M3E スキル（slideshow / cognitive_science / SRS）の流用余地が大きい** — ゼロから作る要素は意外と少ない
