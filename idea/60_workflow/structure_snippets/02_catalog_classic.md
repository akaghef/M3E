# 02. カタログ — ビジネス・分析系の定番

「調べ物・分析」で使う古典的フレームワークを網羅。
各スニペットは「子ノード構造」として M3E に展開されるイメージで記述。

## A1. 5W1H

**用途**: 事象・課題・計画の基本整理。最も汎用。

**構造**:
```
親
├── Who （誰が）
├── What （何を）
├── When （いつ）
├── Where （どこで）
├── Why （なぜ）
└── How （どうやって）
```

**応用**: 既存の記述に対して「5W1H で漏れチェック」としても使える。

## A2. 5W2H / 5W3H

**用途**: 5W1H に定量・金額軸を追加。ビジネス計画向け。

**構造**: 5W1H + `How much` （いくらで） + `How many` （どれだけ）

**推し度**: 企画書用途なら 5W2H が標準。

## A3. SWOT 分析

**用途**: 組織・事業・プロダクトの内外環境分析。

**構造**:
```
親
├── Strengths （内部・プラス）
├── Weaknesses （内部・マイナス）
├── Opportunities （外部・プラス）
└── Threats （外部・マイナス）
```

**マトリクス表示**: 2x2 軸（内外 × +/-）として map_views と相性◎。

## A4. TOWS 分析

**用途**: SWOT の発展形。**組合せ戦略**を導く。

**構造**:
```
親
├── SO 戦略 （強み × 機会）
├── WO 戦略 （弱み × 機会）
├── ST 戦略 （強み × 脅威）
└── WT 戦略 （弱み × 脅威）
```

**連鎖**: SWOT → TOWS と連続展開できると強力（A3 の結果を参照）。

## A5. PEST / PESTLE / STEEPLE 分析

**用途**: マクロ環境分析。事業戦略の前段。

**構造（PEST）**:
```
親
├── Political （政治）
├── Economic （経済）
├── Social （社会）
└── Technological （技術）
```

**PESTLE**: + `Legal` + `Environmental`
**STEEPLE**: + `Ethical` + `Demographic`

## A6. 4P （マーケティング・ミックス）

**用途**: マーケティング施策の4要素。

**構造**:
```
親
├── Product （製品）
├── Price （価格）
├── Place （流通・場）
└── Promotion （販促）
```

## A7. 4C （顧客視点）

**用途**: 4P を顧客視点で書き直したもの。

**構造**: `Customer Value` / `Cost` / `Convenience` / `Communication`

**推し**: BtoC の現代的フレーム。4P と両方当てて比較する使い方が良い。

## A8. 3C 分析

**用途**: 事業戦略の基本軸。

**構造**:
```
親
├── Customer （顧客）
├── Competitor （競合）
└── Company （自社）
```

**拡張**: 4C （+ Co-operator）、5C （+ Context）。

## A9. 5 Forces （ファイブ・フォース）

**用途**: 業界の競争構造分析（ポーター）。

**構造**:
```
親
├── 既存競合の脅威
├── 新規参入の脅威
├── 代替品の脅威
├── 買い手の交渉力
└── 売り手の交渉力
```

## A10. VRIO 分析

**用途**: 経営資源の競争優位性評価。

**構造**:
```
親
├── Value （価値があるか）
├── Rarity （希少か）
├── Imitability （模倣困難か）
└── Organization （組織で活用されているか）
```

**判定**: 4つすべて Yes で「持続的競争優位」。

## A11. AIDA / AIDMA / AISAS

**用途**: 購買プロセスモデル。

**構造（AIDA）**: `Attention` → `Interest` → `Desire` → `Action`
**AIDMA**: + `Memory` （記憶）
**AISAS**: `Attention` → `Interest` → `Search` → `Action` → `Share`

**連鎖タイプ**: 時間軸を持つので、各段階の下に施策ノードを追加する形が合う。

## A12. STP 分析

**用途**: マーケティング戦略策定。

**構造**:
```
親
├── Segmentation （市場細分化）
├── Targeting （標的市場選定）
└── Positioning （ポジショニング）
```

## A13. BCG マトリクス（成長・シェア行列）

**用途**: 事業ポートフォリオ分析。

**構造**:
```
親
├── Stars （花形）
├── Cash Cows （金のなる木）
├── Question Marks （問題児）
└── Dogs （負け犬）
```

**表示**: 2x2（市場成長率 × 市場シェア）。map_views との連携。

## A14. ビジネスモデルキャンバス

**用途**: 事業モデル全体像の可視化。

**構造（9 ブロック）**:
```
親
├── Customer Segments
├── Value Propositions
├── Channels
├── Customer Relationships
├── Revenue Streams
├── Key Resources
├── Key Activities
├── Key Partnerships
└── Cost Structure
```

**配置**: Miro/FigJam で有名な左右対称レイアウト（右=顧客側、左=運営側）。

## A15. リーンキャンバス

**用途**: スタートアップ向け。BMC の軽量版。

**構造（9 ブロック）**:
```
親
├── Problem
├── Solution
├── Key Metrics
├── Unique Value Proposition
├── Unfair Advantage
├── Channels
├── Customer Segments
├── Cost Structure
└── Revenue Streams
```

**差分**: `Key Partnerships` の代わりに `Problem` が中心。

## A16. バリューチェーン（価値連鎖）

**用途**: 企業活動の付加価値分析。

**構造**:
```
親
├── 主活動
│   ├── 購買物流
│   ├── 製造
│   ├── 出荷物流
│   ├── 販売・マーケティング
│   └── サービス
└── 支援活動
    ├── 全般管理
    ├── 人事・労務
    ├── 技術開発
    └── 調達
```

## A17. アンゾフマトリクス（成長マトリクス）

**用途**: 事業成長戦略。

**構造**:
```
親
├── 市場浸透 （既存市場 × 既存製品）
├── 新市場開拓 （新市場 × 既存製品）
├── 新製品開発 （既存市場 × 新製品）
└── 多角化 （新市場 × 新製品）
```

## A18. カスタマージャーニー

**用途**: 顧客体験の時系列マッピング。

**構造**:
```
親
├── Awareness （認知）
├── Consideration （検討）
├── Purchase （購買）
├── Retention （継続）
└── Advocacy （推奨）
```

各段階の下に: 接点 / 感情 / 課題 / 機会 の 4 サブ要素を付けるバリエーションも。

## A19. ペルソナ

**用途**: 顧客像の具体化。

**構造**:
```
親（ペルソナ名）
├── 基本情報 （年齢/性別/職業/家族）
├── 目標
├── 痛み
├── 日常行動
├── 情報源
└── 意思決定プロセス
```

## A20. KPI ツリー

**用途**: 最終目標を測定可能な指標に分解。

**構造**:
```
親（KGI）
├── KPI1 （+ サブKPI の再帰展開）
├── KPI2
└── KPI3
```

**再帰スニペット**: 深さをパラメタで指定できるタイプ。

---

## 横断論点（A）

- **Aa. 日本語 vs 英語ラベル** — SWOT は `Strengths/強み` どちら？多言語ラベルをメタデータで持つ案
- **Ab. マトリクス型の表示** — SWOT/BCG/アンゾフは 2x2 表示が合う、map_views と統合すべき
- **Ac. 連鎖スニペット** — SWOT → TOWS, AIDMA → 施策 など「次に出したい」遷移をスニペットに含める
- **Ad. パラメタ化** — KPI ツリーの「深さN」、PESTLE の「要素選択」などカスタマイズ余地
- **Ae. 業界特化バリアント** — 医療なら SWOT のうち Regulatory を必須化、など

**この章だけで 20 種類**。03・04・05 も同様に 15〜20 種ずつ並べる。
