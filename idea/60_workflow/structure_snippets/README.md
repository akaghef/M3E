# 構造スニペット — 思考フレームワークをパッと呼び出す

研究・思考・分析で使う構造化フレームワーク（5W1H、SWOT、MECE、PICO ...）を
M3E のノード展開時に**ワンアクションで呼び出せる**スニペット機能のブレスト。

思考パターンを「毎回手で書く」→「テンプレで一発展開」にしたい。
種類はとにかく多く。古典的なものからニッチなものまで網羅する。

> メタフォルダ: `idea/60_workflow/`
> 思考フレームワーク自体は研究・学習プロセスの道具なので `60_workflow/` が中心。
> 呼び出し UI は `30_ux/keyboard_modes/` と接続可能だが、ここでは章として内包。

## 方針

- 採用判断はしない — 全部カタログ化する
- 実装は考えない — MVP ファイルで触れる程度
- 複数案を並べる — 分類軸・呼び出し方式を並列に並べる
- **網羅性を優先** — 「種類をとにかくたくさん」のユーザー要望
- 各フレームワークはノード展開時のテンプレ（親→子ノード自動生成）として扱える形で書く

## キーメッセージ

1. **種類 × 呼び出しの 2 軸で考える** — スニペットの豊富さと、その瞬発的呼び出しの両方がないと価値が出ない
2. **構造だけ展開 → 内容はユーザー or AI** — テンプレは空欄の子ノードを作るのが基本、埋めるのは別作業
3. **M3E のノードツリーとの親和性が高い** — SWOT の 4 象限も、5W1H の 6 項目も、ノードの子として自然に表現できる
4. **フレームワークは「思考の借り物」** — 自作スニペット化でユーザー固有のパターンも蓄積可能
5. **ロングテール戦略** — 定番 10 個で 80% のユースケースをカバーしつつ、ニッチ 50 個も用意する
