# 05. カタログ — 創造・意思決定系

発散（アイデア出し）と収束（選択・意思決定）の両側。

## D1. SCAMPER

**用途**: 既存のものを改変してアイデアを出す。

**構造**:
```
親（対象・既存案）
├── Substitute （置換）
├── Combine （結合）
├── Adapt （適応）
├── Modify / Magnify （修正・拡大）
├── Put to other use （転用）
├── Eliminate （削除）
└── Reverse / Rearrange （逆転・並べ替え）
```

**推し**: 発散系の決定版。既存案ノードに当てると即 7 案生まれる。

## D2. 6色ハット（シックス・シンキング・ハッツ）

**用途**: 多視点から同一テーマを検討。デ・ボノ。

**構造**:
```
親（テーマ）
├── 白: 事実・情報
├── 赤: 感情・直感
├── 黒: 批判・リスク
├── 黄: 楽観・利点
├── 緑: 創造・代替案
└── 青: メタ・進行管理
```

**連鎖**: 6 色を順番に「被って」考える。モード切替 UI と親和。

## D3. ディズニー・メソッド

**用途**: 夢想家・現実家・批評家の 3 役割で交互に考える。

**構造**:
```
親（アイデア）
├── Dreamer （夢想家: 理想・制約無視）
├── Realist （現実家: 実行計画）
└── Critic （批評家: 弱点・懸念）
```

**6色ハットと似るが 3 役割に絞って直列適用**するのが特徴。

## D4. ブレインストーミング古典（オズボーン式）

**用途**: 集団発散。4 ルール（批判禁止・自由奔放・量重視・結合改善）。

**スニペット形**:
```
親（テーマ）
├── 発散フェーズ （空ノード N 個を最初から用意）
│   ├── アイデア1
│   ├── アイデア2
│   └── ...
└── 収束フェーズ （後で KJ 法 T13 に接続）
```

## D5. オズボーンのチェックリスト

**用途**: 9 項目の発想刺激リスト。SCAMPER の祖先。

**構造（9 項目）**: 転用 / 応用 / 変更 / 拡大 / 縮小 / 代用 / 置換 / 逆転 / 結合

**SCAMPER との関係**: SCAMPER がより体系化されているが、原型としての価値あり。

## D6. TRIZ 40 の発明原理

**用途**: 技術的問題解決。旧ソ連の発明研究から。

**構造**: 40 原理のうち、問題の「技術矛盾」に対応する数原理を提示。

**スニペットとしての扱い**: フル展開は多すぎるので、**カテゴリ別（分割/抽出/非対称性/...）の下位スニペット**に分けて呼び出す。

## D7. リフレーミング

**用途**: 問題の枠組みを変えて見る。

**構造**:
```
親（現在の捉え方）
├── 別の枠組み1 （例: 問題→機会）
├── 別の枠組み2 （例: 欠点→個性）
└── 別の枠組み3 （例: 失敗→学習）
```

## D8. 類推発想 / アナロジー思考

**用途**: 他領域から発想を借りる。

**構造**:
```
親（問題）
├── 別分野のアナロジー1 （例: 生物界）
├── 別分野のアナロジー2 （例: 自然現象）
├── 別分野のアナロジー3 （例: 別業界）
└── 共通構造の抽出
```

## D9. RICE スコアリング

**用途**: 施策の優先順位付け。

**構造**:
```
親（施策）
├── Reach （到達人数）
├── Impact （影響度）
├── Confidence （確信度）
├── Effort （工数）
└── Score = R × I × C / E
```

**計算ノード**: 数値プロパティと自動計算が欲しい。

## D10. ICE スコアリング

**用途**: RICE の簡易版。

**構造**: `Impact` / `Confidence` / `Ease` → 3 つの積または平均。

**使い分け**: ICE は Reach が見積れない新規施策向け。

## D11. MoSCoW 法

**用途**: 要件の優先度分類。

**構造**:
```
親（要件プール）
├── Must have （必須）
├── Should have （重要）
├── Could have （あれば良い）
└── Won't have this time （今回はやらない）
```

**推し**: プロジェクト管理で現代的に使われる。M3E の開発 TODO にも使える。

## D12. 意思決定マトリクス（Weighted Decision Matrix）

**用途**: 複数選択肢を複数基準で評価。

**構造**:
```
親（決定項目）
├── 基準 （各基準に重み）
│   ├── 基準1 重み: 0.3
│   ├── 基準2 重み: 0.5
│   └── 基準3 重み: 0.2
├── 選択肢 A
│   ├── 基準1 評価
│   ├── 基準2 評価
│   └── 基準3 評価
├── 選択肢 B
└── 選択肢 C
```

**計算**: 各選択肢のスコア = Σ (評価 × 重み)。map_views のマトリクスビューと相性◎。

## D13. ペア比較法 (Paired Comparison)

**用途**: N 案すべての 2 項比較。

**構造**:
```
親（選択肢群）
├── Option A
├── Option B
├── Option C
└── ペア比較
    ├── A vs B → 勝者
    ├── A vs C → 勝者
    └── B vs C → 勝者
```

**定量化**: 勝率でランキング。

## D14. Kepner-Tregoe (KT) 問題分析

**用途**: 問題の正確な定義と原因特定。

**構造**:
```
親（問題）
├── What （何が問題か / 何が問題でないか）
├── Where （どこで起きる / どこでは起きない）
├── When （いつ起きる / いつは起きない）
├── Extent （程度）
└── 差別化要因 （起きる/起きないの違いは？）
```

**推し**: **バグ調査・トラブルシュート**で強力。IS-IS NOT の対比構造が鋭い。

## D15. プレモーテム (Pre-mortem)

**用途**: 「失敗した前提」で振り返りを事前にやる。

**構造**:
```
親（計画）
├── 想定シナリオ: 失敗した
├── なぜ失敗したか？（仮説）
│   ├── 失敗要因1
│   ├── 失敗要因2
│   └── 失敗要因3
└── 予防策
    ├── 対策1
    ├── 対策2
    └── 対策3
```

**兄弟**: 逆転の発想（T17）と似るが、**完了済みを装う想像**が特徴。

## D16. Eisenhower マトリクス（重要度 × 緊急度）

**用途**: タスクの 4 象限分類。

**構造**:
```
親（タスク群）
├── 重要 × 緊急 （即実行）
├── 重要 × 非緊急 （計画的に）
├── 非重要 × 緊急 （委任）
└── 非重要 × 非緊急 （削除）
```

## D17. Impact-Effort マトリクス

**用途**: 施策の 2x2 選別。

**構造**:
```
親（施策群）
├── High Impact × Low Effort （Quick Wins）
├── High Impact × High Effort （Big Bets）
├── Low Impact × Low Effort （Fill-ins）
└── Low Impact × High Effort （Money Pits）
```

## D18. Dot Voting / N/3 投票

**用途**: 集団での案選定。

**スニペットとしての形**: 案ノードの下に「投票」サブノードを各参加者分展開、得票集計。

**M3E 的拡張**: 複数エージェントに投票させる（AI 視点の集約）。

## D19. Cynefin フレームワーク

**用途**: 状況を Clear / Complicated / Complex / Chaotic / Confused の 5 種に分類して対応選択。

**構造**:
```
親（問題）
├── 分類1: Clear （既知・手順で解ける）
├── 分類2: Complicated （専門家の分析必要）
├── 分類3: Complex （試行錯誤）
├── 分類4: Chaotic （即応）
└── 分類5: Confused （分類不能）
```

**推し**: 問題の性質に応じて他のスニペットを選ぶメタスニペットにもなる。

## D20. PDCA / PDSA サイクル

**用途**: 改善サイクル。

**構造**:
```
親（改善対象）
├── Plan
├── Do
├── Check / Study
└── Act
```

**リンク**: Act → Plan へ戻るリンクを自動で（OODA T20 と同様）。

---

## 横断論点（D）

- **Da. 発散系 vs 収束系** — D1-D8 は発散、D9-D18 は収束。スニペット選びは「今どっちが必要か」で変わる
- **Db. スコア計算ノード** — RICE・ICE・決定マトリクスは数値評価が本質。ノードに数値スロット + 自動計算が必要
- **Dc. 役割モード** — 6色ハット・ディズニーメソッドは「今自分が何色」をセッション状態で保持したい
- **Dd. タイマー付きモード** — 635 法（T16）や 6色ハットは時間制限が効果的。タイマー連動スニペット
- **De. 行動テンプレとの接続** — MoSCoW, Eisenhower は map_views の process_lifecycle と接続
- **Df. AI への発散依頼** — D4（ブレスト）の空ノードを AI に埋めさせる「発散エージェント」として呼び出せる
