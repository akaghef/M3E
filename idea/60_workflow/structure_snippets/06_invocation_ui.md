# 06. 呼び出し UI — パッと出す方法のバリエーション

「種類をとにかくたくさん用意したい」と「パッと出したい」は両立が難しい。
60-70 種類のスニペットからどう即座に 1 つ選ぶかを、複数案で並べる。

## U1. コマンドパレット（スラッシュコマンド）

**動き**: ノード選択中に `/` 押下 → パレット表示 → `swot` のようにタイプ → Enter で展開。

**実装イメージ**:
```
/swo       ← 部分入力
┌─────────────────────────┐
│ SWOT 分析                │ ← ハイライト
│ SWOT + TOWS 連鎖         │
│ （類似: ストレングス...） │
└─────────────────────────┘
```

**長所**:
- 名前を部分記憶していれば最速
- 種類が増えても fuzzy 検索で対応可
- Notion/Obsidian のユーザーにはお馴染み

**短所**:
- 名前を知らないと出せない → 初心者に優しくない
- ショートカットを奪う（既存の `/` がある場合は干渉）

**論点 U1a**: 起動キーは何か
- `/` / `:` / `Ctrl+P` / `Ctrl+Shift+P` の選択

**論点 U1b**: 日本語・英語両入力
- `swot` でも `スウォット` でも `弱み強み` でもヒットする辞書

## U2. キーバインド（1 キーまたはチョード）

**動き**: モード突入（例: `s` = snippet mode）→ 2 文字目で選択（`w` = SWOT）。

**例**:
- `s w` → SWOT
- `s p` → PICO
- `s 5` → 5W1H
- `s s` → SCAMPER

**長所**:
- パレット表示がないので最速
- キーボードから手を離さない

**短所**:
- 60 種類を 1-2 キーに割り当てるのは無理。サブカテゴリ階層が必要
- 覚えるのが大変 → チートシート必須

**論点 U2a**: 階層化
- `s a` (analysis) → `s` / `w` / `p` (SWOT/…/PEST)
- `s t` (thinking) → `m` / `w` (MECE/Why)
- `s r` (research) → `p` / `i` (PICO/IMRaD)

**論点 U2b**: M3E の keyboard_modes との統合
- `idea/30_ux/keyboard_modes/` との接続で、モード切替に snippet mode を追加
- または既存の Insert Mode の拡張

## U3. ノード右クリック / コンテキストメニュー

**動き**: ノードを右クリック → 「スニペット展開」サブメニュー → カテゴリ → スニペット選択。

**長所**:
- 発見しやすい（メニューに書いてある）
- マウスユーザー向け

**短所**:
- 速度は遅い
- タッチ操作では長押しで代替する必要

**適した場面**: 初心者・時々ユーザー向けの補助経路。

## U4. 検索ボックス統合

**動き**: 通常のノード検索ボックスに入れると、スニペット名もヒット候補に混ざる。

**長所**:
- 検索行動から自然に発見
- 既存 UI を拡張するだけ

**短所**:
- ノード検索結果と混ざって視覚的に曖昧
- 絞り込みフィルタが必要（「スニペットのみ」トグル）

## U5. 頻出ランキング / Recently Used

**動き**: スニペットパレットに「最近使った」「よく使う」セクション。

**長所**:
- 個人の使用パターンに自動適応
- 上位 5-10 個をショートカット化できる

**短所**:
- 新規ユーザーは使えない（データが溜まってない）

**論点 U5a**: コンテキスト対応ランキング
- 「研究ノード配下では PICO が上位」「事業ノード配下では SWOT が上位」
- ノード種別やタグに応じて並び替え

## U6. AI 呼び出し（自然言語）

**動き**: ノード選択 → AI チャット呼び出し → 「SWOT で展開して」「研究向けに整理」 → AI がスニペット選定 + 内容まで埋める。

**長所**:
- フレームワーク名を覚えていなくていい
- 「原因分析したい」→ AI が fishbone / 5Whys / KT を提示
- 内容も一緒に埋めてくれる（構造 + 内容）

**短所**:
- 応答レイテンシ（数秒）
- LLM コスト
- 意図しないスニペットが使われるリスク
- privacy 制約（policy_privacy）— 送信前にフィルタ必要

**論点 U6a**: AI は「スニペット選定」だけするか、「内容まで埋める」かの 2 段階
- 選定のみ → 確認 → 内容埋める、の対話フロー

**関連**: `idea/20_ai/ai_agent_deep/` と接続。

## U7. ドラッグ & ドロップ（スニペットボード）

**動き**: 画面端に「スニペットボード」を常時表示、ノードにドラッグ→ドロップで展開。

**長所**:
- 視覚的
- タッチ操作・タブレットで強い
- 頻出スニペットを自分で Pin できる

**短所**:
- 画面領域を消費
- マウス必須（キーボード派には遅い）

**バリエーション**: 右側の折り畳みパネル、フローティングツールバー、command bar 下部の専用タブ。

## U8. ショートカット学習 / 予測サジェスト

**動き**: ユーザーが手動で似た構造を複数回作ると、「これスニペット化しますか？」と提案。
または、あるノードのタイトルから「これは SWOT 展開が合いそう」とサジェスト。

**長所**:
- 明示的な呼び出しなしで習慣化
- 自作スニペットが自然に増える

**短所**:
- 実装が重い（パターン認識）
- 過剰な提案はうるさい

**関連**: M3E の `onboarding_tuning`（topic_pool O5）的な AI 補助。

## U9. ガラクタ置き場 (Snippet Drawer)

**動き**: Discord の絵文字ピッカーや Slack の絵文字ショートカット風。
スニペットアイコンだけが並んだ「引き出し」を画面下部に。

**ハイブリッド**: U7 と U5 を組合せ、最近使った 10 個を常時アイコン表示。
あまり使わないものは `...` ボタンの奥。

## U10. ボイスコマンド

**動き**: マイク入力で「スウォットで展開」「PICO 分析で」と発声。

**長所**:
- ハンズフリー、歩きながら・運転中
- モバイル（`capture_ingest` A1 音声入力と接続）

**短所**:
- 精度、騒音環境
- 認識失敗時のリカバリ UX

## U11. URL / ディープリンク

**動き**: `m3e://insert?snippet=swot&target=<nodeid>` のようなリンクで外部から呼び出し。

**用途**:
- 他アプリ（Slack/Notion/メール）から「この構造で M3E に追加」
- 外部ワークフローツールとの連携（例: Zapier）

**関連**: `scope-tree-as-system.md` の URI 設計と連動。

## U12. AI エージェント自律呼び出し

**動き**: 夜間の自動整理エージェント（topic_pool C10）が、整理対象ノードに「このノードは SWOT で展開した方が良さそう」と判断して**勝手に適用**（または提案のみ）。

**長所**:
- ユーザー不在でも整理が進む
- 大量ノードの一括構造化

**短所**:
- 勝手に変えられるのは怖い → 「提案→承認」が安全
- 誤判定のコスト

---

## 横断論点（U）

- **Ua. 最小セット** — 全部実装は過剰。MVP としては `U1（パレット）` + `U5（recently used）` が鉄板
- **Ub. 一貫性** — キーバインド・パレット・右クリックで出せるメニューが同じであるべき（同じスニペット名体系）
- **Uc. コンテキスト依存** — 選択中ノードの種類に応じて「この状況で使いやすいスニペット」を上位に出す
- **Ud. 発見性 vs 速度** — U3（右クリック）は発見性◎速度×、U2（1キー）は逆。両立には U1 パレットが中庸
- **Ue. AI と手動の棲み分け** — U6（AI）は「どれを選ぶか迷う時」、手動（U1/U2）は「決まっている時」
- **Uf. フレームワーク辞書の国際化** — 日英両ラベル、日本語でも英語名で引ける
- **Ug. ウェブ vs デスクトップ vs モバイル** — ショートカットキーはデスクトップ有利、ドロップは全環境、ボイスはモバイル

## 複数案の組合せ例

| ペルソナ | 主な呼び出し経路 |
|---|---|
| キーボード派研究者 | U1 パレット + U2 よく使うもの 1-2 キー |
| マウス派ビジネス | U3 右クリック + U7 ドラッグ |
| AI 依存ユーザー | U6 AI 呼び出し中心、U1 は補助 |
| モバイル利用者 | U7 + U10 ボイス |
| 共同編集セッション | U4 検索 + U12 エージェント提案 |

**MVP で優先**: U1, U5, U3 の 3 点セットから開始が現実的。
