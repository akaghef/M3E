# 個人生産性ワークフロー: M3E を生活の中心に据える体験設計

研究者 akaghef が **毎日 M3E を開く理由** を作るための体験パターンを並べる。
「タスク管理ツール」ではなく「思考の地図」を、生活リズム・エネルギー・習慣の
レイヤーで支える設計案を網羅する。

## 方針

- 採用判断はしない、複数案を並べる
- 実装詳細は MVP ファイルだけに集約
- **「習慣を強制しない、サポートする」設計哲学** を全案に通す
- 既存 idea/keyboard_modes/ M23〜M26 (Sprint/Standup/Retro/Journal) と接続できる形で書く
- automation_obstacles の **A4 自己欺瞞 / A5 エネルギー管理 / J1 燃え尽き** に効く案を優先的に列挙
- 自動化原理主義を避ける。「効率化 ＞ 自動化」を前提

## 論点一覧 (ID 索引)

ユーザーが論点 ID 単位で「これを採用」と言える構造:

### A. デイリー儀式 (G1)
- A1. 朝チェックイン: 1ノード固定 vs 日付ノード生成 vs テンプレ展開
- A2. 昼チェックイン: あり vs 無し / トリガー方式
- A3. 夜チェックイン: 振り返り粒度 (3問 / 1問 / 自由記述)
- A4. 儀式の発動方法: 時刻通知 vs 手動 vs エディタ起動時
- A5. 儀式スキップの設計: 連続スキップしてもよい

### B. 週次レビュー (G2)
- B1. レビュー対象: 全マップ vs 今週触ったノード vs done のみ
- B2. テンプレ構造: KPT / 5W / Five Whys / 自由
- B3. Retro mode (M25) との統合粒度
- B4. 次週への引継ぎ: action item 自動生成 vs 手動

### C. 目標トラッキング (G3)
- C1. OKR/KPI を別ビューにする vs 専用ノード種別
- C2. 進捗の自動算出 vs 手動申告
- C3. 目標と日次タスクのリンク強制度
- C4. 目標の鮮度 (古い目標の整理) — D3 stale 検出と接続

### D. 習慣トラッキング (G4)
- D1. Streak 表示の有無 (やる気 vs プレッシャー)
- D2. 達成バーの粒度 (binary / 段階 / 連続値)
- D3. 「やらない日」の許容設計
- D4. 習慣ノードを通常ノードと混ぜる vs 分離

### E. 時間記録 (G5)
- E1. 外部連携 (Toggl/Clockify) vs M3E 内蔵
- E2. ノード単位計測 vs プロジェクト単位
- E3. 自動検知 vs 手動開始
- E4. プライバシー (policy_privacy: 計測データの扱い)

### F. Pomodoro (G6)
- F1. 1ノード = 1ポモドーロ強制 vs 緩い結びつき
- F2. 25/5 vs 可変 vs ウルトラディアン
- F3. ポモドーロ完了時の状態変化 (status 自動更新)
- F4. 中断・延長の扱い

### G. 集中モード (G7)
- G1. フェード強度 (50%/80%/100%)
- G2. 通知遮断のスコープ (M3E 内のみ vs OS レベル)
- G3. 解除条件 (時間 / 手動 / 完了)
- G4. 集中破壊時のログ取得

### H. エネルギーログ (G8)
- H1. 1-5 スコア vs 自由記述 vs 絵文字
- H2. 入力タイミング (固定時刻 / 変化時 / Pomodoro 終了時)
- H3. 可視化 (時系列グラフ / ヒートマップ / 要約)
- H4. 最適時間帯発見の AI 提案

### I. 横断
- I1. 自動化と「やらない自由」の両立 (A4 自己欺瞞対策)
- I2. ガミフィケーション K 系との関係 (Streak の害)
- I3. M3E 自体を「もう1つの SNS」化しないための歯止め
- I4. データ取得の倫理 (自分のログでも見たくない時がある)

## キーメッセージ (ブレスト前提)

- 「毎日開く理由」は **タスクではなく状態** から生まれる (エネルギー / 気分 / 集中)
- 強制と支援の境界は「**未記入を責めるかどうか**」で判定できる
- 既存 keyboard_modes M23-M26 は本トピックの実装層、本ブレストは UX/概念層
- automation_obstacles の **N7 創発の場** を侵さない設計が必須 (儀式が雑談を奪わない)
- Pomodoro / 集中モードは「**ノード = 単位作業**」に M3E が向いていることを最大化できる
