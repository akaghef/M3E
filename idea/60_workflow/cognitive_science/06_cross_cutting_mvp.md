# 横断観察・組み合わせ・MVP・未決質問

J1〜J5 を **個別機能** ではなく **認知サポート層** として一体運用したときに見えてくる
観察、組み合わせ効果、最小実装の入口、未解決の重要論点をまとめる。

## 横断観察 1: J1〜J5 は同心円構造

```
       [ J5 メタ認知 ]
       ↑ 観察対象
  [ J1 SRS ] [ J2 鮮度 ]
       ↑ スケジュール / 鮮度
  [ J3 複利 ] [ J4 質問 ]
       ↑ 知識の構造化
  [ M3E 既存属性: status / weight / details ]
```

- **基層（既存）**: マップそのもの。今ある属性
- **構造化層（J3, J4）**: 知識の関係性と問いの組み立て
- **時間層（J1, J2）**: スケジュールと鮮度
- **観察層（J5）**: それらを観察するメタ層

→ **下から積む**。J3/J4 を先に整備しないと J1/J2 は出題対象が定まらず、J5 はログる対象がない。

## 横断観察 2: 既存属性で大半が始められる

| 認知機能 | 既存属性で代替できるか |
|---|---|
| J1 出題対象選び | weight + status で 80% |
| J1 評価スコア | annotation に書ける |
| J2 鮮度 | last_modified で 70% |
| J2 重要度 | weight |
| J3 前提リンク | 既存 link 種別 |
| J3 複利指標 | リンク数集計 |
| J4 問い表現 | text に `?` プレフィックス |
| J4 答え | 子ノード or details |
| J5 改稿パターン | history（git）解析 |
| J5 確信度 | 新属性 confidence 必要 |
| J5 気持ち | 新属性 mood 必要 |

→ **新スキーマは confidence と mood の2つだけ**。残りは既存属性 + 集計で済む。
これは MVP のハードルを大きく下げる。

## 横断観察 3: N5（学習）原則との整合検証

`automation_obstacles/04_what_not_to_automate.md` N5: 学習自体は自動化してはいけない。

| 機能 | N5 違反リスク | 緩和設計 |
|---|---|---|
| J1 SRS | 中（AI が答え提示すると違反） | J1.G1「再露出のみ、判定なし」を採用 |
| J2 鮮度 | 低（観察のみ） | J2.D5「annotation 追記で鮮度回復」を採用 |
| J3 推薦 | 中〜高（AI 結論提示は違反） | J3.E4「種別提示」と J3.E5「根拠表示」で本人判断を残す |
| J4 質問生成 | 低〜中 | J4.E2 Five Whys のような **本人が問いを深める** 形を採用 |
| J5 メタ認知 | 低 | J5.C5「質問型」で答えを与えず気づきを促す |

→ J3 が最も注意。「AI が答えを示す」設計にすると N5 違反になる。
**「過去のあなたの言葉を再投入」だけ** に留めるのが安全。

## 横断観察 5: 「研究マップ特化」の歪み

Anki / RemNote / SuperMemo などの SRS は「事実カード前提」。
研究マップに持ち込む時は次の歪みが必須:

1. **正解判定を捨てる** — 「思い出せた / 答え合わせ」は研究的に不適切
2. **進化前提** — ノード内容が変わる、過去版を保つ
3. **問い中心** — 答えではなく問いをカード化
4. **個別性** — 「重要さ」は本人にしか分からない、自動推定は補助
5. **複利志向** — カード単位の最適化より、関係の活性化を優先

これらの歪みが既存 SRS ツールと M3E の差別化になる。

## MVP 提案（最小単位 → 段階導入）

### MVP-1（1日で書ける）: 鮮度色付け

- **対象**: J2.A1 + J2.B7 + J2.F1
- **実装**: viewer.ts で last_modified が N日以前のノードを `?fresh=on` toggle で透明化 / 色温度
- **新スキーマ**: なし（last_modified は既存）
- **行数感**: ~50行

### MVP-2（数日）: 簡易 SRS 専用ビュー

- **対象**: J1.A3 + J1.B3 + J1.C2 + J1.D3 + J1.G1
- **実装**: `?view=srs` で status="key" or weight≥N のノードを5枚ランダム表示、テンキー1〜5 で次回間隔（線形）算出
- **新スキーマ**: `srs_next` (date), `srs_ease` (number)
- **行数感**: ~150行

### MVP-3（1〜2週間）: 質問駆動ノード

- **対象**: J4.A1 + J4.A2 + J4.D2
- **実装**: text が `?` で始まるノードに専用バッジ、`?view=open_questions` で一覧
- **新スキーマ**: `role="question"` (任意付与)
- **行数感**: ~100行

### MVP-4（数週間）: 複利推薦

- **対象**: J3.A6 + J3.B3 + J3.C2 + J3.E3
- **実装**: ノード詳細パネルに「関連を探す」ボタン → AI subagent が3件返す → マップで強調
- **新スキーマ**: なし（AI 推薦なので保存不要）
- **行数感**: ~200行 + プロンプト設計

### MVP-5（並行）: メタ認知ログ基盤

- **対象**: J5.A3 + J5.A4 + J5.D1 + J5.B3
- **実装**: 確信度 1〜5 入力 UI、週次レポート（ノード単位の編集回数 + 確信度推移）
- **新スキーマ**: `confidence` (1-5)
- **行数感**: ~100行

### MVP 順序の推奨

1. **MVP-1（鮮度）** — 即効性、リスク低、データ生成不要
2. **MVP-3（質問駆動）** — 構造化を先に整える
3. **MVP-2（SRS）** — MVP-1 と MVP-3 の上に乗る
4. **MVP-4（複利推薦）** — projection vision 直結なので戦略的に重要
5. **MVP-5（メタ認知）** — 他全てのログを材料にする最上位

## 未決質問（重要なものから）

### Q1. SRS は専用ビュー or 既存マップ統合か

J1.C で 2/3 の選択肢が残った（C2 専用 / C6 テンキー / C3 マップ強調）。
- 専用ビューは map_views 文化と整合
- 既存マップ統合は「文脈保持」で研究らしい
- → 両立できないか（toggle）

### Q2. confidence と weight をどう棲み分けるか

両方とも「重要度」的な数値。
- weight = 客観的・他者から見た重要度？
- confidence = 主観的・自分の確信度？
- → 用途を分ける必要があるが、ユーザに分かるか

### Q3. 「答え」の構造化（J4.C2 子ノード式）は遡及できるか

既存マップに後から「これは問い、これは子答え」を割り当てる工数。
- AI 自動割当は精度問題
- 段階移行（新規だけ問い駆動）は混在マップの分かりにくさ

### Q4. メタ認知ログの保存先

J5.H プライバシー絶対遵守。
- マップ JSON 内（暗号化必須）
- 別ストレージ（同期コスト）
- そもそも永続化しない（観察のみで消す）

### Q5. AI 依存度をどう測るか

J5.A7 で「AI 生成 vs 人間生成」を比率化したいが、どこに記録するか。
- ノード作成時のフラグ（手動 → 嘘がつける）
- AI subagent が touch したノードを自動記録
- 編集パターンから推定（精度低）

### Q6. SRS は「研究マップ全体」 or 「明示マークだけ」

J1.B で揺れた。
- 全体：手間ゼロだがノイズ多
- 明示：質高いが運用コスト
- → 段階的拡大（status="key" → weight 閾値 → 全体）が現実的か

### Q7. 「学習科学」自体の科学的根拠をどこまで要求するか

FSRS / 検索練習 / メタ認知 ... 元論文に当たるか、Anki 流の経験則で十分か。
- 研究者向けツールである以上、根拠は強い方が信用される
- ただし「教育心理学の論文を全部読む」は範囲爆発
- → 採用機能ごとに参照論文を1〜2本だけメモする運用が落とし所か

## 戦略観察

J1〜J5 を **「世界モデル → 射影」を半年で実用化** する project_projection_vision の文脈で見ると:

- **J3（複利）+ B2（科研費テンプレ）** が本丸
- **J4（質問駆動）+ B1（論文ドラフト）** が次点
- **J1（SRS）+ J2（鮮度）** は「過去資産を腐らせない」基盤として裏方
- **J5（メタ認知）** は「半年でちゃんと進んでいるか」自己モニタの軸

→ 半年計画なら **MVP-1 → MVP-3 → MVP-4 → MVP-5** の順序で
3ヶ月までに鮮度・質問駆動・複利推薦が揃い、後半3ヶ月でメタ認知ループに乗せる、が現実的タイムライン。
