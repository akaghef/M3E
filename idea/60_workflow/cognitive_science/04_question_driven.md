# J4. 質問駆動学習（マップ全体を「答えるべき問い」で再構成）

ノードを **「答え」ではなく「問い」** として記述するモードを設計する。
研究は問いの連鎖。マップを問いで埋めると、学習科学的に **検索練習（retrieval practice）** に近い効果が得られる。

## なぜ要るか

- M3E の現状ノードは「結論／メモ／タスク」が混在。問いと答えの区別が曖昧
- 研究本来の駆動力は「問い」だが、マップ上で問いが埋もれる
- 「問いに答える形」でノードを巡ると、検索練習による理解の固定化が起きる
- 申請書・論文も「問題提起 → 解決」構造 → マップが問い駆動なら出力しやすい

## ユースケース具体化

1. **テーマ着手時** — 「このテーマで答えるべき問い」を最初に5個作り、各問いの subtree を育てる
2. **論文 introduction 執筆** — 「Research Question」を集めて並べる、不足を発見
3. **査読対応** — 査読コメント = 新しい問いとして追加、対応ノードを紐付け
4. **学生指導** — 学生に「この subtree を読んで答えよ」と問いを出す
5. **発想の自己点検** — 「自分はこの問いに答えられているか」を SRS 連動で問う
6. **未解決の地図化** — 「答えてない問い」を全マップから抽出、優先度付け

## J4.A. 「問い」の表現方法

| 案 | 表現 | 良い点 | 悪い点 | 推し度 |
|---|---|---|---|---|
| J4.A1 `?` プレフィックス | text 先頭 `?` で問い扱い | 簡単、既存と互換 | 弱い区別 | ★★ |
| J4.A2 role 属性 | `role="question"` 追加 | 構造的、検索容易 | スキーマ追加 | ★★★ |
| J4.A3 ノードタイプ | 専用 type "Q" | 強い区別、UI 別扱い可 | 既存マイグレ | ★★ |
| J4.A4 status 流用 | status="question" | 既存属性 | status 多義化 | ★ |
| J4.A5 タグ | `#q` タグ付与 | 緩い、後付け容易 | 検索性中 | ★★ |
| J4.A6 末尾 `?` 検出 | text が `?` で終わる = 問い | 完全自動 | 偽陽性 | ★ |

**観察**: J4.A2 が中庸で拡張性が高い。J4.A1 + J4.A2 併用も可（既存ノードは ? で、新規は role）。

## J4.B. 問いの粒度

| 案 | 範囲 | 例 | 推し度 |
|---|---|---|---|
| J4.B1 マップ全体 | マップに 1〜10 個の大問い | 「世界モデル → 射影は半年で実用化できるか」 | ★★ |
| J4.B2 subtree | 各 subtree のルートが問い | 「J1 SRS は M3E にどう統合すべきか」 | ★★★ |
| J4.B3 単ノード | 1ノード1問い、答えは details | 「FSRS と SM-2 のどちらを採るか」 | ★★ |
| J4.B4 リンク間 | 2ノード間の関係を問う | 「J1 と J2 の関係は何か」 | ★ |
| J4.B5 階層問い | 大問い → 小問い → 答えの階層 | 自然 | ★★★ |

**観察**: J4.B5（階層問い）が研究のメンタルモデルに最も合う。

## J4.C. 「答え」をどう扱うか

| 案 | 答えの場所 | 推し度 |
|---|---|---|
| J4.C1 details に書く | ノード details が答え | ★★ |
| J4.C2 子ノード群 | 子ノードを答え群と見做す | ★★★ |
| J4.C3 別ノード + answer link | 「answer」リンクで紐付け | ★★ |
| J4.C4 annotation | annotation = 答え | ★ |
| J4.C5 暫定答え + 確定答え | annotation = 暫定、details = 確定 | ★★ |

**観察**: J4.C2 が DAG 的に綺麗。問い → 子答え群 → さらに子問い、で再帰。

## J4.D. 「未解答の問い」の扱い

| 案 | 表示 | 機能 |
|---|---|---|
| J4.D1 視覚的強調 | 答え子なし問いを赤枠 | 一目瞭然 |
| J4.D2 専用ビュー | `?view=open_questions` | 全 open 問い一覧 |
| J4.D3 SRS 連動 | open 問いを優先的に SRS 出題 | J1 と接続 |
| J4.D4 期限属性 | 問いに「いつまでに答える」を付与 | 締切連動 |
| J4.D5 「諦め」属性 | 答えない問いを明示 | 心理安全 |

**観察**: J4.D2 + J4.D3 が研究駆動力を直接押し上げる。

## J4.E. 質問生成の自動化

| 案 | 仕組み | 摩擦 | 推し度 |
|---|---|---|---|
| J4.E1 AI 生成 | LLM が既存ノードから問い候補生成 | 低 | ★★ |
| J4.E2 Five Whys | 任意ノードに「なぜ?」を5回適用 | 中 | ★★ |
| J4.E3 「次に問うべき」 | C2「質問生成器」と統合 | 低 | ★★★ |
| J4.E4 反問パターン | LLM が「逆を問えば」を生成 | 中 | ★ |
| J4.E5 テンプレ問い | 「研究目的 / 方法 / 限界 / 応用」テンプレ | 低 | ★★ |

**観察**: J4.E3 が 00_topic_pool C2 と直結する戦略的接続点。

## J4.F. マップを「問いの順序」で巡る

J4 は slideshow（idea/slideshow/）と接続:

| 案 | 巡り方 | 推し度 |
|---|---|---|
| J4.F1 問いだけツアー | open 問いをスライドショー | ★★ |
| J4.F2 問い → 答え → 次の問い | Q-A-Q のリズム | ★★★ |
| J4.F3 階層降下 | 大問い → 子問い → 孫問いを順に | ★★ |
| J4.F4 ランダム問い5枚 | 朝の儀式 | ★★ |

**観察**: J4.F2 が **論文 introduction の構造** そのもの。出力（B1 論文ドラフト）と接続できる。

## J4.G. 「答えられない / 部分回答」の段階表現

研究の問いは「答え or 未答」の二値ではない:

| 案 | 段階表現 |
|---|---|
| J4.G1 確信度 0〜100 | 答えに対する確信 |
| J4.G2 status: open / partial / answered / archived | 4段階 |
| J4.G3 「答えの版」 | 答えに version を付け、過去版を保持 |
| J4.G4 メタコメント | 「答えだが未検証」「仮説段階」等タグ |

**観察**: J4.G2 + J4.G3 は J5 メタ認知ログとセットで強い。

## J4.H. 既存マップへの遡及適用

既に書かれたマップを問い駆動に変換する移行:

| 案 | 移行戦略 |
|---|---|
| J4.H1 一括変換せず段階適用 | 新ノードのみ問い形式 |
| J4.H2 AI が既存を問い候補化 | LLM が「これを問いにすると」と提案 |
| J4.H3 タグ付けだけで済ます | 既存ノードに `#q` を後付け |
| J4.H4 マップ別運用 | 「問い駆動 workspace」を別に作る |

**観察**: J4.H1 + J4.H4 の併用で、既存資産を壊さず移行できる。
