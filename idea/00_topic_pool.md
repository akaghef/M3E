# 次にブレインストーミングすべきテーマ・プール

「アイデアのアイデア」。
個別の機能案ではなく、**ブレストの題材になる領域** を網羅的に並べる。
気になったテーマを `idea/<topic>/` フォルダで深掘りしていく素材。

凡例:
- 🌱 まだ何も書いていない
- 🌿 部分的に既存ファイルで触れた
- 🌳 すでに idea/ にフォルダがある
- ⭐ 個人的に高レバレッジと思うもの（参考程度）

---

## A. 入力・取り込み（Capture / Ingest）

ノードを **マップに入れる** までの体験。M3E の入り口戦略。

- 🌱 A1. **音声入力フロー** — マイク常時待機、句点で自動分割→scratch、後でtriage
- 🌱 A2. **ブラウザクリップ拡張** — 任意ページから「このノードに紐付けて保存」
- 🌱 A3. **PDF 取り込み** — 論文の章節構造を自動でノード化、引用情報も抽出 ⭐
- 🌱 A4. **OCR / 画像→ノード** — ホワイトボード写真や手書きノートを構造化
- 🌱 A5. **メール→ノード** — 特定アドレスに送ったメールが scratch 化
- 🌱 A6. **Slack/Discord ブリッジ** — チャットの一行に reaction 付けると scratch 化
- 🌱 A7. **コードからノード** — git commit / TODO コメントを自動でノード化
- 🌱 A8. **AIチャットの会話履歴→マップ** — Claude/GPT の対話を構造抽出して取り込み ⭐
- 🌱 A9. **モバイルからのワンタップ捕捉** — iOS/Android の「共有」シート対応
- 🌱 A10. **常駐 quick-add ウィジェット** — Mac/Win メニューバーから即捕捉

## B. 出力・公開（Export / Publish）

マップを **外に出す** 体験。研究者用途では特に重要。

- 🌱 B1. **論文ドラフト自動生成** — IMRAD ビュー → LaTeX/Markdown 出力 ⭐⭐
- 🌱 B2. **科研費・申請書テンプレ生成** — project_projection_vision に直結 ⭐
- 🌱 B3. **スライド自動生成** — マップ → PowerPoint/Reveal.js（slideshow と接続）
- 🌱 B4. **ナレーション付き動画書き出し** — マップツアー + 音声合成
- 🌱 B5. **Web 公開** — マップをそのまま閲覧可能な静的サイトに
- 🌱 B6. **Markdown ブログ生成** — 特定タグ付きノードから記事生成
- 🌱 B7. **Anki/SRS カード書き出し** — 学習サイクルへ
- 🌱 B8. **知識グラフ標準形式エクスポート** — RDF/JSON-LD/SKOS

## C. AI / エージェント深化

既存の ai_subagent.ts を超えた、思考パートナーとしての AI。

- 🌱 C1. **Devil's Advocate モード** — 任意ノードに対して反論・批判を生成 ⭐
- 🌱 C2. **質問生成器** — ノード内容を読んで「次に問うべき質問」を提案
- 🌱 C3. **多視点エージェント** — 異なる立場（実験家/理論家/批評家）が同時にコメント
- 🌱 C4. **自動 link 提案** — 似た内容のノード間にリンク候補を提示
- 🌱 C5. **insight 抽出** — マップ全体を読んで「あなたが気づいていないパターン」
- 🌱 C6. **research assistant フロー** — 「このテーマで論文探して」→ 文献ノード自動投入
- 🌱 C7. **要約の階層化** — 同じ subtree を 1行/3行/1段落 で要約切替
- 🌱 C8. **会話型クエリ** — 「先週決めたことを教えて」→ AI が該当ノードを提示
- 🌱 C9. **エージェント・ペルソナ** — マップを「指導教員」「査読者」「学生」が見たらどう思うか
- 🌱 C10. **自走 AI** — 夜中にマップを掃除・整理・要約しておいてくれる ⭐

## D. メンテナンス・衛生（Hygiene）

マップが大きくなった時の **腐敗防止**。

- 🌱 D1. **重複検出** — 似たノードを発見して merge 候補表示 ⭐
- 🌱 D2. **孤立ノード検出** — リンク無し・親なしノードのアラート
- 🌱 D3. **stale 検出** — N ヶ月触っていないノードの整理候補
- 🌱 D4. **自動アーカイブ** — done から N 日経過で archive 移動
- 🌱 D5. **構造的健全性スコア** — マップの「健康度」を1指標で表示
- 🌱 D6. **過深ツリー警告** — 7階層を超えると分割提案
- 🌱 D7. **巨大ノード警告** — text/details が長すぎたら分割提案
- 🌱 D8. **ガベージコレクション UI** — Cleanup mode と接続

## E. 協調・コラボレーション

複数人 / 複数エージェントでの作業体験。

- 🌱 E1. **リアルタイム共同編集 UX** — カーソル表示・フォロー・他者の編集ハイライト
- 🌱 E2. **コメント／注釈フロー** — Google Docs 風のコメント→解決
- 🌱 E3. **非同期レビュー** — レビュー依頼をエージェント・他者に投げて結果を集約
- 🌱 E4. **権限モデル** — 読み取り/編集/管理のロール、ノード単位 ACL
- 🌱 E5. **プレゼンス表示** — 誰が今どのノードを見ているか
- 🌱 E6. **発言権モード** — プレゼン時に1人だけ操作、他は閲覧
- 🌱 E7. **agent <-> human 引き継ぎ UI** — エージェント完了→人間レビューの流れ

## ブレスト時のテンプレ案

各テーマを掘る時に毎回考えるべき切り口を統一:

```
01_concept.md   — 何か、なぜ、誰が嬉しいか、ユースケース5個
02_options.md   — 実装方式の選択肢（採否決めない）
03_runtime.md   — UI / 体験のバリエーション
04_data.md      — データモデル / 永続化 / 互換性
05_mvp.md       — 最小実装と未決質問
```

slideshow / keyboard_modes / map_views で実証済みのテンプレ。
新テーマも同じ構成にすると比較・選択がしやすい。

## このファイル自体の使い方

- 気が向いたテーマから 🌳 化していく
- 各テーマを掘った時に 🌱→🌿→🌳 を更新
- 新しいテーマを思いついたら追記
- 高レバレッジ候補は時々再評価
- 「飽きたから別」「他とくっつけて」「分割して」も自由
