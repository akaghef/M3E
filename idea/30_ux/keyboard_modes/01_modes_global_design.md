# 01. モード共通設計の論点

各モード固有の操作とは別に、「モードという仕組み」自体に検討すべき共通論点がある。

## モードの入り方の選択肢

| 方式 | 例 | 良い点 | 悪い点 |
|------|------|--------|--------|
| Mod1. プレフィックスキー | `m` → `r` で review mode | Vim 風、覚えやすい | プレフィックス取られる |
| Mod2. コマンドパレット | `Ctrl+P` → "review" | 発見性高い | キー数多い |
| Mod3. メニュー項目 | ハンバーガーから選ぶ | 初学者◎ | 高速操作不可 |
| Mod4. ノードのコンテキストで自動 | proposal ノード選択時に review mode 候補表示 | 文脈マッチ | 意図と外れる場合あり |
| Mod5. URL params | `?mode=review` | リンク共有可 | 編集中の切替が面倒 |
| Mod6. モード固定（モードレス） | 常時いくつかのキーが効く | 速い | キーが枯渇 |

複数併用が現実的。特に Mod1+Mod2 を両立できると良い。

## モードからの抜け方

| 方式 | 良い点 |
|------|------|
| Esc | 普遍的、ユーザ予測通り |
| 同じプレフィックスで再度入る | トグル感 |
| モード操作を1回したら自動で抜ける | "1回だけのつもり" に最適 |
| 別モードへの切替で自動抜け | モード排他なら自然 |

「持続モード」と「ワンショットモード」を区別する設計があると便利。
- 例: `s` でステータス変更1回、`S` でステータス更新モードに入りっぱなし

## モード表示（インジケータ）

| 方式 | 良い点 | 悪い点 |
|------|------|------|
| ステータスバーにラベル | 既存 metaPanel に追加で済む | 視線を下にやらないと見えない |
| カーソル/選択枠の色変化 | 視認性◎ | 色覚配慮要 |
| 画面端のカラーバー | プレゼンでも見える | 場所を取る |
| キャレット形状の変化 | Vim 風 | M3E の「キャレット」概念がない |
| 効くキー一覧をオーバーレイ表示 | 学習に強い | 邪魔 |
| トースト1秒（モード入った瞬間だけ） | 軽い | モード持続を見失う |

組み合わせ推奨: **ステータスバー常時 + 入った瞬間トースト + ?キーで操作一覧**。

## モード排他性

- 排他: 同時に1つだけ（Vim 流）。実装単純、混乱少ない
- 重ね掛け可: review mode + bulk select 同時など
  - 強力だが状態爆発、初学者に厳しい

**MVP は排他**。重ね掛けは後段。

## モード中の安全装置

- 破壊的操作（削除・大きな移動）は1キーでも **2回叩き or 確認トースト**
- Undo を全モードで `u` か `Ctrl+Z` で必ず効くようにする
- モード中の操作は audit_log に必ず吐く（既存 audit_log.ts 流用）
- うっかり押下対策: 一定時間内の同キー連打を block するオプション

## 既存ホットキーとの衝突回避

- 既存 viewer.ts はキーがかなり詰まっている（m3e-shortcuts skill が管理）
- モード中はトップレベルのキーを **ほぼ全部横取り** する
  - Esc / Undo / 矢印（カメラ）など最低限だけパススルー
- モードに入っていない時は既存ホットキー優先

## 学習・発見性

- モード入った直後に「効くキー一覧」を画面下にチラ見せ（数秒で消える）
- `?` でいつでも一覧表示
- `Ctrl+P` のコマンドパレットからもモードに入れる
- 設定に「モード入る時にチートシートを常時表示」オプション

## ロギングと改善ループ

- どのモードがどれだけ使われているか、どのキーが効いていないかを統計
- 使われないモードは消す、頻用モードは固有 UI を強化
- 既存 insights skill と連携可能か検討

## 論点（ID付け）

- **Md1. モード入り口** — Vim 流プレフィックス vs パレット vs 両方
- **Md2. 持続 vs ワンショット** — モード毎にデフォルトを定める
- **Md3. インジケータ** — どこにどう出すか
- **Md4. 排他** — 排他のみ vs 重ね掛け可
- **Md5. 安全装置** — 破壊操作の確認方式
- **Md6. 発見性** — `?` 表示 / 自動チラ見せ / 常時表示
