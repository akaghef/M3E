# 04. K4 ノード進化（育てるとビジュアル変化）

K 系の中で最も「研究行為そのもの」と親和性が高い候補。
ノードに加えた手入れの **積み重ね** をビジュアルで表現する。

研究は対象を「育てる」行為であり、視覚的な成長表現は本質と矛盾しない。
ただし演出が過剰だと「タマゴッチ化」して真面目度を毀損する。
適切な抽象度の探索が論点の中心。

## K4.1 何を「成長」と見なすか

| ID | 成長の定義 | 観察軸 | 取りやすさ |
|----|---------|------|---------|
| K4.1a | 編集回数 | アクション数 | 容易 |
| K4.1b | 内容の長さ・厚み（text/details の文字数）| 量 | 容易 |
| K4.1c | 接続数（incoming/outgoing リンク）| 構造的中心性 | 容易 |
| K4.1d | 子ノード数（subtree のサイズ）| 拡張性 | 容易 |
| K4.1e | 振り返り回数（revisit カウント）| 関心度 | 中 |
| K4.1f | 経年（作成からの日数）| 時間 | 容易 |
| K4.1g | 状態遷移の歴史（status の変化数）| 思考の進展 | 中 |
| K4.1h | 質的成熟度（AI が「完成度」を判断） | AI 評価 | 高 |

→ 単一指標より **複数指標の合成** で「成長」を定義する方が研究行為に近い。
   K4.1c + K4.1e + K4.1g の合成が最も「思考が深まった」を捕まえやすい。

## K4.2 ビジュアル変化の方向性（何が変わるか）

| ID | 変化軸 | 例 | 視認性 | 演出強度 |
|----|------|------|------|------|
| K4.2a | 色（彩度・明度・色相） | 薄→濃 | 高 | 弱 |
| K4.2b | サイズ | 小→大 | 高 | 中（レイアウト影響） |
| K4.2c | 縁取り（border の太さ・スタイル）| 細→太、点線→実線 | 中 | 弱 |
| K4.2d | 形状（角丸→多角形→花型）| 段階的形変化 | 高 | 強（萎えリスク） |
| K4.2e | テクスチャ（パターン・ハッチ）| 透明→ストライプ→塗り | 中 | 中 |
| K4.2f | アイコン付与（小さなマーク）| 葉っぱ・星・冠 | 中 | 中（子供っぽさ） |
| K4.2g | アニメーション（ゆらぎ・脈動）| 完成ノードはわずかに脈動 | 高 | 強（疲れる） |
| K4.2h | 周囲のオーラ／グロー | 重要ノードに微光 | 中 | 中 |

→ 推し組み合わせ: K4.2a + K4.2c。最も静かで研究者向け。
→ 警告: K4.2d, K4.2g は飽きが早い。

## K4.3 「衰退」をどう表現するか

成長があるなら、触らないとどうなるか？ も論点。
「枯れる」表現は K6.3「生き物感」と直結する。

| ID | 衰退表現 | 心理的圧 | UC マッチ |
|----|--------|------|---------|
| K4.3a | 衰退なし、達成は永続 | ゼロ | UC5 マイルストーン記念 |
| K4.3b | 経年で彩度低下（古いものは淡く）| 弱、自然 | 振り返り時の優先度判定 |
| K4.3c | 経年で半透明化、フェード | 中 | 自動アーカイブの予告 |
| K4.3d | 「枯れる」表現（葉が落ちる等メタファ）| 強、子供っぽい | UC4 育成感 |
| K4.3e | サイズ縮小 | 中 | レイアウト的に良い副作用 |
| K4.3f | 周囲ぼかし、視界の端へ寄る | 弱 | 視覚的整理 |

→ 推し: K4.3b（彩度低下）。情報損失なく、視覚的階層が自然に発生する。
   既存の D3 stale 検出と統合可能。

## K4.4 段階の数（離散 vs 連続）

| 案 | 段階 | 解説 |
|----|------|------|
| K4.4a | 2段階（新規／成熟） | 最小、シンプル |
| K4.4b | 3段階（種／苗／樹）| ゲーム的、わかりやすい |
| K4.4c | 5段階 | RPG 的、Lv1〜5 |
| K4.4d | 連続値（成長スコア 0.0〜1.0 を色彩度に直接マップ）| ◎ ジャンプなく自然 |
| K4.4e | ユーザー定義段階 | 自由度高、設定負担 |

→ 推し: K4.4d（連続値）。離散段階は到達瞬間が「Lv up !」と煽る誘惑を生む。

## K4.5 メタファの選択

ノード進化を **何に喩えるか** で印象が大きく変わる。

| ID | メタファ | 印象 | 研究者反応予測 |
|----|--------|------|------------|
| K4.5a | 植物（種→芽→木）| 自然、優しい | ○、最も無難 |
| K4.5b | 結晶／鉱物（粒→石→宝石）| 静的、知的 | ◎、研究者向け |
| K4.5c | 生物（卵→幼体→成体）| ペット感、子供向き | △ |
| K4.5d | 建築（基礎→骨組み→完成）| プロジェクト管理的 | ○ |
| K4.5e | 抽象（点→線→面→立体）| 純粋幾何 | ◎、最もクール |
| K4.5f | メタファなし、純粋な属性表示 | 機能的 | ○ |

→ 研究者向け推し: K4.5b（結晶）または K4.5e（抽象幾何）。
   植物（K4.5a）も汎用的だが「育成ゲーム」連想が強い。

## K4.6 マップ全体の視覚的進化

ノード単位を超えて、マップ **全体** が育つ表現も論点。

| ID | 全体表現 | 例 |
|----|--------|------|
| K4.6a | マップ全体の彩度／密度をヒートマップ風に | 取り組み多い領域が濃く |
| K4.6b | マップに「気候」を表現（晴／曇／雨）| P3 思考の天気と統合 |
| K4.6c | 季節変化（マップが冬→春→夏に）| 経年表現、過剰演出懸念 |
| K4.6d | マップ全体がゆっくり「呼吸」アニメ | 微妙な脈動 |
| K4.6e | 全体表現はせず、ノード単位のみ | シンプル |

→ 推し: K4.6a（ヒートマップ的）。00_topic_pool.md P3 と接続。

## 横断的気づき（K4）

- **「色の彩度」だけで多くの目的が達成可能**。
  K4.2a + K4.3b で「育つ／枯れる」を同時表現。最小投資で最大効果
- **連続値マッピング（K4.4d）が「Lv up !」演出の罠を回避する鍵**。
  離散段階は必ず「次の段階へ進めるために手抜きで触る」誘発を生む
- **メタファ選択は研究者の好みに大きく依存する**。
  プリセット（K4.7e）化して個別に選ばせるべき
- **「衰退」表現（K4.3）は新規性が高い**。
  既存のゲーム化ツールはほぼ全て「集める」のみで、「枯れる」はあまり扱わない
- **K4 は K1/K2 と異なり、副作用を抑えやすい**。
  バッジ／ストリークと違い「達成のための行為」を誘発しにくい
