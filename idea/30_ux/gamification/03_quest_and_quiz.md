# 03. K3 クエストシステム と K5 デイリークイズ

「やるべきことを冒険化」と「過去ノードからランダム出題」を選択肢で並べる。
両者は **AI 連携** が前提になりやすく、C 系（AI/Agent）との接続を強く意識する。

## K3: クエストシステム

### K3.1 クエストとは何か（M3E 文脈での定義）

「タスクリスト + 報酬 + 物語性」の組み合わせ。
真面目研究ツールでの定義を絞ると：

| 定義案 | 解説 | 子供っぽさ |
|------|------|---------|
| K3.1a | 1日の「3つの行動」を朝に提示、夜に達成チェック | 低（todo 管理に近い） |
| K3.1b | 数日〜週間スパンの「挑戦」（例: scratch を 0 にする） | 低 |
| K3.1c | 月間「キャンペーン」（例: 論文1本書き上げる） | 低 |
| K3.1d | RPG 風物語（「あなたは賢者となり…」）| 高、研究者NG |
| K3.1e | 暗黙クエスト（達成すると後から「実はクエストでした」と通知） | 中、面白いが滑り得 |

→ 推し: K3.1a〜c（実用 todo の言い換えに留める）。
→ NG: K3.1d。

### K3.2 クエスト生成主体

| 案 | 主体 | コスト | M3E 適合度 |
|----|------|------|----------|
| K3.2a | 完全手動（ユーザーが自分で設定） | ゼロ | 既存 todo と被る、価値薄 |
| K3.2b | テンプレ提示（「今週の3挑戦」雛形）| 低 | personal_productivity/G2 と接続 |
| K3.2c | AI が現在のマップ状態から提案（C2 質問生成器の応用） | 中 | ◎、独自性あり |
| K3.2d | AI が研究目的（project_projection_vision）から逆算 | 高 | ◎◎、本筋 |
| K3.2e | コミュニティテンプレ共有 | 高 | スコープ外 |

→ K3.2c/d が AI エージェント深化（C 系）と統合され最も価値が高い。

### K3.3 クエスト粒度

| 案 | 例 | 達成判定 |
|----|------|------|
| K3.3a | 微小（1行書く・1リンク張る） | 自動 |
| K3.3b | 小（1ノード完成・1triage完了） | 自動 |
| K3.3c | 中（subtree 整理・1テーマ深掘り） | 半自動 |
| K3.3d | 大（章を書き上げる・review通す） | 手動チェック |

→ 推し: 複数粒度を混ぜる（K3.3a 1個 + K3.3b 1個 + K3.3c 1個）。

### K3.4 クエスト完了後のフィードバック

| 案 | 演出 | 真面目度 |
|----|------|------|
| K3.4a | 達成チェックがつくだけ | ◎ |
| K3.4b | 「今日のクエスト全達成」短いメッセージ | ○ |
| K3.4c | XP 獲得アニメーション | × |
| K3.4d | 完了履歴がカレンダーに残る | ○、振り返り価値あり |
| K3.4e | AI が達成を要約して評価コメント | ◎、研究的価値高 |

### K3.5 失敗の扱い

| 案 | 扱い | 心理的安全性 |
|----|------|---------|
| K3.5a | ペナルティ（XP マイナス・装備破損等）| × M3E 不適 |
| K3.5b | 静かにスキップ、翌日同じクエストが再度出る | ○ |
| K3.5c | 「なぜ達成できなかったか」を尋ねる | ◎、メタ認知促進 |
| K3.5d | 失敗が3回続いたらクエスト粒度を下げる | ○、適応的 |
| K3.5e | 失敗ログを取らない（履歴は成功のみ）| ◎、罪悪感ゼロ |

### K3.6 クエストの「冒険感」を演出するか否か

| 案 | 演出強度 | 例 | 研究者反応予測 |
|----|------|------|------------|
| K3.6a | 演出なし、純粋 todo | 「今日: A, B, C」 | 抵抗ゼロ、ゲーム感ゼロ |
| K3.6b | 名前だけ冒険化 | 「本日のクエスト: 散らかった scratch を片付けよ」 | 軽い遊び感、許容圏 |
| K3.6c | 文脈付き物語 | 「賢者よ、汝の知の森には霧が立ち込めている…」 | ほぼ全員 OFF にする |
| K3.6d | テーマ選択可（真面目／物語／皮肉等） | ユーザーが好きなトーンを選べる | ◎、柔軟 |

→ 推し: K3.6a または K3.6d。

## K5: デイリークイズ

### K5.1 何を出題するか

| 案 | 出題対象 | 学習効果 |
|----|--------|------|
| K5.1a | 古いノード（1週間以上未参照）の text | ○ 忘却防止 |
| K5.1b | done 化したノードの内容 | ○ 達成の定着 |
| K5.1c | リンク元と先の関係（「AとBの関係は？」）| ◎ 構造記憶 |
| K5.1d | タグや属性（「これは何カテゴリだった？」）| △ 浅い |
| K5.1e | ユーザー指定の subtree のみ | ◎ コントロール |

### K5.2 出題形式

| 案 | 形式 | 自動化難易度 |
|----|------|---------|
| K5.2a | ノードタイトルだけ表示、「内容覚えてる？」自己採点 | 低 |
| K5.2b | 内容の一部マスクして穴埋め | 中 |
| K5.2c | AI が選択肢を生成（「次のうち正しいのは」）| 高、AI 必須 |
| K5.2d | 関連ノードを答える（「これと関係あるノードは？」）| 中 |
| K5.2e | フリー回答 → AI 採点 | 高、AI 必須 |

→ MVP は K5.2a（自己採点）が最軽量。

### K5.3 採点と忘却曲線統合

| 案 | 機構 | J 系（認知科学）連携 |
|----|------|----------------|
| K5.3a | 採点なし、ただ表示するだけ | なし |
| K5.3b | 「覚えてた／忘れてた」2択 | J2 忘却曲線可視化 |
| K5.3c | 5段階自己採点（Anki 風）| ◎ J1 SRS統合と直結 |
| K5.3d | 採点結果でノードに「鮮度」属性付与 | ◎ J2 と統合 |

→ K5.3c/d が J 系と統合され学習価値最大化。

### K5.4 タイミング

| 案 | タイミング | 通知必要性 |
|----|---------|--------|
| K5.4a | 起動時に1問 | 低、自然 |
| K5.4b | 朝指定時刻 | 中、通知必要 |
| K5.4c | ランダムにポップアップ | 高、邪魔 |
| K5.4d | サイドバーに常駐、気が向いたら触る | 低、最も自然 |
| K5.4e | スキマ時間検出（5分操作なし）| 中、賢いが UX 難 |

→ 推し: K5.4a + K5.4d。

### K5.5 クイズ結果の活用

| 案 | 活用 | 価値 |
|----|------|------|
| K5.5a | 結果を保存しない | ゼロ、ただ娯楽 |
| K5.5b | 履歴のみ（カレンダー風） | 中 |
| K5.5c | 忘れていたノードを「再学習リスト」に積む | ◎ |
| K5.5d | 弱点 subtree を発見（特定領域の正答率低い）| ◎ メタ認知 |
| K5.5e | クイズ結果に応じて K1 ストリークを進める | K1 統合 |

## K3 × K5 比較表

| 観点 | K3 クエスト | K5 デイリークイズ |
|------|----------|--------------|
| 主目的 | 行動誘発 | 記憶定着 |
| 真面目度との両立 | 中（演出次第で滑る） | 高（学習行為そのもの） |
| AI 依存度 | 高（K3.2c/d） | 中（K5.2c/e で必要） |
| MVP 難易度 | 中（テンプレ程度なら低） | 低（K5.2a + K5.5b） |
| 既存ツール例 | Habitica | Anki, Quizlet |
| 副作用 | 「冒険感」演出失敗、子供っぽさ | 答えられないとマップ嫌いになる懸念 |

## 横断的気づき（K3/K5）

- **K3 は AI なしには成立しない**。手動設定なら既存 todo と同じ。
  AI でこそマップ状態を読み「次に何をすべきか」を提案できる
- **K5 は J 系（J1 SRS統合・J2 忘却曲線）の入り口になりうる**。
  ゲーム化のフリで認知科学の本筋に入っていく経路
- **両者とも「失敗時の心理的安全性」設計が最重要**。
  K3.5e（失敗ログを取らない）/ K5.3a（採点なし）など、罪悪感ゼロ設計が必須
- **演出トーン選択（K3.6d）はゲーム化全体の汎用パターン** になりうる。
  「真面目／カジュアル／物語」をユーザーが選べるなら他の K 系にも展開可能
