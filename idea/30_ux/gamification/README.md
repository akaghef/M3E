# ゲーミフィケーション（K）ブレスト

M3E に「楽しさ・継続性・遊びごころ」を持ち込む際の論点プール。
ただし M3E は研究者の **真面目な思考の場** であり、安易な「報酬で釣る」設計は
本筋を汚しかねない。**真面目さ × 遊び** の境界線をどう引くかが本ブレストの主題。

## 方針

- 採否は決めない。複数案を並べる
- 実装詳細を決めない。MVP ファイルでもコードはスケルトン程度
- 各論点に階層 ID（K1.1, K2.3, ...）を打ち、後でユーザーが ID 単位で選べる構造
- 「研究者の真面目な思考の場 vs ゲーム化」のバランスは常に意識する論点として扱う
- 既存メモリ（project_projection_vision: 半年で世界モデル→射影実用化）と整合
- 既存 `idea/personal_productivity/` `idea/maintenance_hygiene/` と隣接領域あり、
  そちらに近い案は本ブレストのスコープ外として明記する

## キーメッセージ（暫定見立て）

1. **K4（ノード進化）と K5（デイリークイズ）は研究者向けに純度が高い**
   — 育てる楽しさと記憶定着は研究行為そのもの。ゲーム化感が薄い
2. **K1/K2 は「副作用」リスクが高い**
   — ストリーク維持のための薄いノード乱造、バッジ目的の構造汚染
3. **K3（クエスト）は AI エージェント（C 系）と組み合わせて初めて活きる**
   — 単独ではタスクリストの言い換えに留まる
4. **「OFF できる」が必須**
   — 真面目モード／遊びモードの切替を最初から設計すべき
5. **集める方向と捨てる方向の両方をゲーム化すべき**
   — 既存 D 系（hygiene）と接続し、「掃除した分だけ得る」報酬軸も検討
