# 02. 線形・時系列ビュー

ノードを1次元に並べるビュー。時間軸 or 段階軸が代表。

## 時間軸系

### V21. Timeline（横軸=時間）

- ノードを `createdAt` / `dueAt` / `completedAt` で配置
- 用途: 進捗の可視化、過去の意思決定の振り返り
- バリアント:
  - V21a. 横スクロール無限軸
  - V21b. ズーム可能（年→月→日）
  - V21c. 複数タイムライン重ね（やったこと／思いついたこと／引用文献）

### V22. Gantt チャート

- Y軸=タスク、X軸=時間、棒の長さ=期間
- 用途: プロジェクト管理、研究ロードマップ
- M3E 適用: 開発タスクの並行可視化、論文執筆スケジュール

### V23. Spiral / Time-lap（螺旋）

- 中心=現在 or 開始、外側に向かって時間進行
- 用途: 長期プロジェクトの全体俯瞰
- 派生: 内側=未来、外側=過去（年輪のように積み上がる）

### V24. River / Stream（流れ）

- 太さで重要度や活動量、横軸が時間
- 用途: 研究テーマ毎の活動量推移、関心の遷移

### V25. Calendar Heatmap（GitHub 風）

- 日付グリッドに活動量の濃淡
- 用途: 「いつ集中して書いたか」「いつアイデアが降りてきたか」の可視化

### V26. Journal Strip（日記帯）

- 日付ノードを縦に並べ、各日付配下のノードを右に展開
- 用途: 振り返り、日次レビュー
- M3E 適用: M26 Journal mode との連携

## プロセス段階系

### V27. Kanban（ステータス列）

- 列: backlog / todo / wip / review / done
- 用途: 開発タスク管理、定型業務
- M3E 適用: strategy ボードの状態可視化、agent 別レーンへ拡張可

### V28. Funnel（漏斗）

- 段階を上から下に、各段階で件数が減る
- 用途: アイデア → 採択 → 実装 → 完了 の歩留まり分析
- M3E 適用: scratch → strategy → done の通過率可視化

### V29. Pipeline（パイプライン）

- 横長の段階列、各段階にノードがカード状に並ぶ
- 用途: 受注パイプライン的、研究テーマの段階管理

### V30. Roadmap（ロードマップ帯）

- 縦軸: 製品／領域、横軸: 時期（Q1/Q2/Q3/Q4）
- 用途: 半年〜1年単位の見通し
- 既存メモリ project_projection_vision「半年で実用化」と相性◎

### V31. Burndown / Burnup

- X軸=時間、Y軸=残タスク数 or 完了数
- 用途: スプリント・期限への進捗

## イベント・流れ可視化

### V32. Event Stream（活動ログ）

- 時系列に下から積み上がる流れ
- 用途: audit_log の可視化、collab 履歴

### V33. Sankey（流量図）

- 段階間の遷移量を太さで表現
- 用途: 「scratch のうち何割が strategy に上がったか」を可視化

### V34. Decision Tree（フローチャート）

- 分岐を表現する木
- 用途: 意思決定プロセスの可視化、判断ロジックの形式化

### V35. State Machine View

- ノード=状態、エッジ=遷移
- 用途: ワークフロー設計、業務フロー俯瞰

## 順序を持つ閲覧系

### V36. Outline（アウトライン線形）

- ツリーをインデント付きの縦リストに
- 用途: 印刷・共有用、Markdown 出力プレビュー

### V37. Reading List（読書順）

- 手動ピンの順序で並べる
- 用途: 読み返したいノード列、別途 Slideshow とも繋がる

### V38. Dependency Linear（トポロジカル順）

- depends-on リンクからトポロジカルソートして1列に
- 用途: 「何から手を付けるべきか」の自動順序決定

### V39. Critical Path

- 依存グラフから最長パスをハイライト
- 用途: プロジェクトのボトルネック発見

## 共通の論点

- **Lx1. 時間属性の選択** — createdAt / updatedAt / dueAt / completedAt のどれを使うか
- **Lx2. 期間を持つノードの表示** — 点 / 棒 / 帯
- **Lx3. ズームレベル** — 時間軸の単位切替（年/月/週/日/時）
- **Lx4. 過去を切り捨てる閾値** — 古いノードを自動的に視界外
- **Lx5. 未確定の時間** — 「いつかやる」をどこに配置するか
