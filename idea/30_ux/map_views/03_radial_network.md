# 03. 放射・同心円・ネットワーク系ビュー

中心や関係性を主軸にする配置。M3E の素のビュー（mind map）はここに近い。

## 中心構造系

### V40. Concentric Circles（同心円）

- 中心: コアコンセプト
- 各リング: 距離・抽象度・関連の強さに応じて配置
- 用途: 主題からの広がり、PARA の Projects(中心)→Areas(中)→Resources(外)

**使用例**:
```
       [研究全体]
        │
   [テーマA]──[テーマB]
        │       │
   [手法] [文献] [実験] [考察]
        │
   [仮説] [検証]
```

### V41. Sunburst / Hierarchy Wheel

- 中心に root、外に向かって階層展開、扇形で表現
- 用途: ツリーのコンパクトな全体俯瞰
- 利点: 大きさで件数や重要度を表現できる

### V42. Treemap（矩形塗りつぶし）

- 階層をネストした矩形で表現、面積で重み
- 用途: ノード数や見積の総量比較
- M3E 適用: 各ブランチの「ボリューム」可視化

### V43. Radial Tree（放射状ツリー）

- 中心から放射状に枝が伸びる
- mind map に近いが、座標が極座標
- 利点: 360度を均等に使える

### V44. Galaxy / Constellation

- 力学レイアウトされたノード群
- 強い関係でクラスタが形成、星座のように見える
- 用途: 自然なクラスタリング発見、ノード間関係の可視化

## ネットワーク系

### V45. Force-Directed Graph

- リンクを引力、ノード間を斥力としてレイアウト
- 用途: 関係性の自然な配置、密度発見

### V46. Hierarchical Edge Bundling

- ノードを円周配置、リンクを束ねて中心寄せ
- 用途: 多リンクの可視化、絡まりを減らす

### V47. Adjacency Matrix（隣接行列）

- N×N グリッド、セル色でリンク有無/強度
- 用途: グラフが大きすぎてグラフ表示不能な時
- 利点: 規則性・クラスタが行列で見える

### V48. Chord Diagram（弦図）

- 円周配置 + 内部に弦
- 用途: グループ間の関係総量

### V49. Sociogram（社会図）

- 人物・チーム間のやり取りグラフ
- 用途: collab の関係可視化

## 中心可変系

### V50. Ego-network View

- 任意のノードを中心に置き、その周辺だけ表示
- 1ホップ / 2ホップ / 3ホップ の半径切替
- 用途: 「このノードの周辺だけ集中して見たい」

### V51. Multi-center Layout

- 複数のフォーカスノードを画面に置き、それぞれ周辺を展開
- 用途: 比較探索（このトピックとこのトピックを並べて見る）

### V52. Comparison Radial

- 2つのノードを左右に配置、それぞれの周辺を扇状に展開
- 用途: 2案の比較、A/B 検討

## 階層的・分割系

### V53. Pyramid / Tier View

- 上から下へ、または下から上へ階層
- Maslow 風: 基礎→応用、抽象→具体
- 用途: 知識ピラミッド、依存層の可視化

### V54. Iceberg View

- 水面の上=可視、下=隠れた前提・暗黙知
- 用途: ノートに「氷山下」フィールドを設けて整理
- 派生: 「公開／プライベート」の境界線として使う（policy_privacy と相性）

### V55. Onion Diagram（玉ねぎ図）

- 中心=コア、外に向かって表層化
- 用途: 関心領域分け（自分→チーム→組織→社会）

### V56. Zoom Cluster（ズームクラスタ）

- 遠目はクラスタの塊、近づくと中身が展開
- Google Maps 的振る舞い
- 用途: 大マップでも俯瞰と詳細を両立

## 距離・近接系

### V57. Semantic Proximity（意味近傍）

- 埋め込みベクトル距離でノードを配置
- 用途: 似た内容のノード自動集合
- 実装: AI infra と接続、t-SNE / UMAP

### V58. Citation Network

- 引用関係をグラフ化
- 用途: 文献ノード間の系譜可視化、研究マッピング

### V59. Coauthorship Graph

- 共同研究関係
- 用途: collab 領域、人物管理

### V60. Topic Bridge View

- 2つのクラスタの間に「橋渡しノード」を強調
- 用途: 分野横断アイデア発見

## 共通の論点

- **Rx1. レイアウトアルゴリズム** — 物理シミュレーション / 決定論的 / 手動配置
- **Rx2. 中心の選択** — 自動（次数最大）/ ユーザピン / ルートノード固定
- **Rx3. リンクの種類別表示** — 全部表示 / 種類フィルタ / 太さで強度
- **Rx4. ノード密度の処理** — クラスタ折りたたみ / フィルタ / 透明度
- **Rx5. 動きの安定性** — リアクティブ（編集中もぐにゃり）vs 凍結（明示的に再計算）
