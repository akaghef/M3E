# 02 — I1 タイムトラベル と I2 What-if 分岐

「過去に戻る」と「もしも別の選択をしていたら」。
**過去への移動** カテゴリの 2 機能を扱う。両者はデータ的に近いが、
**読み取り専用 vs 書き込み可能** という大きな違いがある。

---

## I1. タイムトラベル（任意時点のマップを再現）

### TT1. 基本動作の選択肢

| ID | 案 | 内容 | 良い点 | 悪い点 | 推し度 |
|---|---|---|---|---|---|
| TT1.A | スライダ式 | 画面下部に時刻スライダ、ドラッグで時点切替 | 直感的、再生も可能 | 高解像度に弱い | ◎ |
| TT1.B | カレンダー式 | カレンダー UI から日付選択 | 日付指定が早い | 操作が重い | ○ |
| TT1.C | コマンドパレット | `/timetravel 2026-04-01 12:00` | 高速、エージェント親和 | 学習コスト | ○ |
| TT1.D | スナップショット一覧 | backup.ts の世代をリスト表示 | 既存資産直結 | 任意時点不可（粗） | ◎ |
| TT1.E | "あの時に戻る" 検索 | 「査読受領時」「あの会議の翌日」など意味検索 | 認知に近い | AI 必須 | ⭐ |
| TT1.F | ノード単位タイムトラベル | 1 ノードだけ過去版を見る | 軽量、安全 | 全体感が無い | ○ |

### TT2. 復元の解像度

| ID | 解像度 | 元データ | コスト |
|---|---|---|---|
| TT2.A | 世代単位（1h × 10）| backup.ts のみ | ほぼ 0（既存資産） |
| TT2.B | 操作単位（add/edit/delete/move）| audit_log.ts を逆再生 | 中（実装必要） |
| TT2.C | 任意秒単位 | snapshot + audit を組み合わせ補間 | 高 |
| TT2.D | "出来事" 単位 | AI が「会議」「査読」などイベント抽出 | 高（AI 必須） |

### TT3. 復元結果の見せ方

| ID | 見せ方 | 良い点 | 悪い点 |
|---|---|---|---|
| TT3.A | フル切替（現在マップが置き換わる） | 没入感 | 「今」を見失う |
| TT3.B | 並列表示（左:過去 / 右:現在） | 比較しやすい | 画面狭い |
| TT3.C | オーバーレイ（過去マップを半透明で重ねる） | 差分が見える | 視認性低い |
| TT3.D | "ゴーストモード"（過去ノードのみ淡色追加表示） | 軽い、現在を維持 | 大規模差分で混乱 |
| TT3.E | 別ウィンドウ／別タブ | 完全独立 | 同期できない |

### TT4. 過去時点での操作

| ID | 案 | 動作 |
|---|---|---|
| TT4.A | **読み取り専用**（推し） | 編集禁止、見るだけ |
| TT4.B | 読み取り＋コピー | 過去から現在に「持ち帰り」可能 |
| TT4.C | フォーク → 別マップ化 | I2 What-if と統合 |
| TT4.D | "差し戻し"（destructive）| 過去状態を現在として上書き |

→ **推し**: TT4.A 基本、TT4.B が "持ち帰り" 用途で必要。TT4.D は基本やらない（destructive）。

### TT5. UI 上の時刻表現

研究者の認知に合わせる工夫:

- **TT5.a 絶対時刻**: `2026-04-01 12:34`（正確だが認知遠い）
- **TT5.b 相対時刻**: `2 週間前`（直感的）
- **TT5.c イベント名**: `査読受領後`（最強だが要 I5 連携）
- **TT5.d 週ラベル**: `2026-W14`（週次レビュー文化と相性）

→ **3 種同時表示** が研究者向きと推測（絶対 + 相対 + イベント）。

### TT6. 物理アクション（キーボード／マウス）

- **TT6.a** `[` / `]` でスナップショット往復
- **TT6.b** `Cmd+Shift+T` でタイムトラベルモード
- **TT6.c** ホイール上下でスライダ移動
- **TT6.d** `Esc` で現在に復帰

## I2. What-if 分岐（仮想ブランチ）

### WI1. ブランチのメタファ選択

| ID | 案 | メタファ | 認知コスト |
|---|---|---|---|
| WI1.A | Git 風 branch / merge | バージョン管理 | 高（一般ユーザー困難） |
| WI1.B | "サンドボックス"（推し）| 一時的に汚さず試す | 低 |
| WI1.C | "下書きレイヤー" | Photoshop のレイヤー | 中 |
| WI1.D | "並列宇宙" | 複数 universe を切替 | 中 |
| WI1.E | "ドラフト世界" | 1 つの仮想世界を編集→破棄 or 採用 | 低 |

→ 研究者向けは **WI1.B サンドボックス** か **WI1.E ドラフト世界**。
  Git の branch UI は「マージ衝突」概念で詰む。

### WI2. 起点の選択肢

What-if は **「いつ／どこから」分岐するか** が論点:

- **WI2.a** 現在マップから（"今からこう変えてみたら"）
- **WI2.b** 過去任意時点から（I1 と組み合わせ）
- **WI2.c** 特定ノード／サブツリーだけを分岐（部分 what-if）
- **WI2.d** "仮想ノード追加"（あるノードを採用していたら、で仮想ノード生成）

### WI3. 仮想空間でできる操作

| ID | 操作 | 推し度 |
|---|---|---|
| WI3.A | 編集（add/edit/delete/move）| ◎ 必須 |
| WI3.B | リンク追加削除 | ◎ |
| WI3.C | AI に「この世界で論文書け」と指示 | ⭐ 強い |
| WI3.D | 別 what-if と比較 | ○ |
| WI3.E | 「採用」して現在マップにマージ | △（destructive、危険） |

### WI4. 仮想空間の表示

| ID | 案 | 説明 |
|---|---|---|
| WI4.A | **オーバーレイ**（推し） | 現在マップの上に半透明で what-if を重ねる |
| WI4.B | 別タブ／別ビュー | 完全独立、混在しない |
| WI4.C | 並列ビュー | 左右で比較 |
| WI4.D | 差分ハイライト | 変更点だけ色付け |

### WI5. 寿命（Lifetime）

仮想空間をいつまで持つか:

| ID | 案 | 良い点 | 悪い点 |
|---|---|---|---|
| WI5.A | セッション限定（リロードで消える） | クリーン | 続きができない |
| WI5.B | 名前付き保存（"案 B 2026-04"） | 比較・引用可 | 数が増える |
| WI5.C | 期限付き（30 日で自動削除） | 自動掃除 | 残したいものを失う |
| WI5.D | "promote" で本物化、それ以外は破棄 | 意思決定促進 | promote 操作の重さ |

### WI6. AI 連携

what-if は AI と組み合わせると爆発的に効く:

- **WI6.a** "この what-if 世界で論文の Abstract を書いて" → AI が仮想マップから生成
- **WI6.b** "現実 vs what-if の差を要約" → AI が比較
- **WI6.c** "what-if 世界が現実より優れている点／劣っている点" → AI 評価
- **WI6.d** "what-if を更に派生" → AI が別の分岐を提案

### WI7. マージ戦略（destructive）

WI3.E（仮想 → 現実マージ）は危険なので慎重設計:

- **WI7.a** マージは **明示的承認 + 差分プレビュー** 必須
- **WI7.b** マージ前に **現在マップの自動 backup**（既存 backup.ts 活用）
- **WI7.c** マージ単位は **ノード／サブツリー単位**（全部マージは禁止案）
- **WI7.d** マージ後 30 日は undo 可能

## I1 × I2 の関係

| 軸 | I1 タイムトラベル | I2 What-if |
|---|---|---|
| 方向 | 過去のみ | 現在 or 過去から **分岐** |
| 編集 | 不可（推し）| 可（仮想空間内）|
| 永続 | 元データを保持 | サンドボックス、寿命あり |
| 起点 | スナップショット／audit | 任意の時点 |
| AI | 過去の文脈再現 | 仮想実験パートナー |

→ **I1 のフォーク → I2** という流れが自然。
  「あの時の状態に戻って、別の判断をしたらどうなるか試す」フローが王道シナリオ。

## TT × WI 横断の論点

- **共通基盤として "現在マップを汚さない仮想表示" 機能** が要る
  （TT3.D ゴーストモード ≒ WI4.A オーバーレイ）
- **時刻指定 UI（TT5）** は WI2.b でも使える
- **AI 連携（WI6）** は I5 履歴 AI 要約 と機能的に隣接
- **マージ／差し戻しの destructive 性** は安全装置（既存 backup, 04_safety 的）必須
