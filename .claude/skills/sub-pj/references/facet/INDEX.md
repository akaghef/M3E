# facet タイプ一覧

PJ の対象を分解する切り口（facet）の汎用パターン集。
PJ01 の 12 facets を汎用化したもの。PJ ごとに必要なものだけ選べ。

## 全 facet 共通ルール

- **scope をまたぐ link は張るな。** 異なる scope 間の関係は alias で示せ
- **ノードには必ず summary 属性を付けろ。** ラベルだけのノードを作るな
- **最初は 3〜5 facet に絞れ。** 全部定義するな。Phase 0 で使うものだけ

## facet タイプ

| タイプ | ファイル | 何を分解するか | 使う場面 |
|--------|---------|---------------|---------|
| **実装構造** | [class_diagram.md](class_diagram.md) | クラス、モジュール、ファイルの内部構造 | コード解析、移植 |
| **処理フロー** | [flow.md](flow.md) | 時系列の処理手順、呼び出しチェーン | アルゴリズム理解、デバッグ |
| **依存グラフ** | [dependency.md](dependency.md) | 静的な依存関係（import、参照） | 影響範囲分析、移植順序 |
| **概念モデル** | [concept.md](concept.md) | ドメインの概念と関係 | 世界モデル構築、オントロジー |
| **型・階層** | — | 継承、分類体系、is-a 関係 | 型設計、分類整理 |
| **マッピング** | — | 2 つの世界の対応関係 | 概念↔実装、理論↔実験 |
| **データフロー** | — | データの変換・流れ | ETL、計算グラフ |
| **検証** | — | テスト、ground truth、ベンチマーク | 品質保証 |
| **設計** | — | ターゲットのアーキテクチャ・API | 新規設計 |
| **タスク管理** | — | 作業項目、進捗、マイルストーン | プロジェクト管理 |

「—」のタイプは具体ファイルが未作成。PJ で必要になった時に追加する。

## PJ ごとの選び方

- **コード移植 PJ**: 実装構造 + 依存グラフ + 概念モデル + 処理フロー
- **文献整理 PJ**: 概念モデル + 型・階層（分類）+ マッピング（主張↔根拠）
- **新規開発 PJ**: 設計 + タスク管理 + 検証
- **調査・分析 PJ**: 概念モデル + データフロー

PJ01（MATLAB→Julia 移植）は上記のほぼ全タイプを使った極端な例。通常は 3〜5 で十分。
