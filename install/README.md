# M3E

## マインドマップとは

考えごとをするとき、頭の中では複数のアイデアが絡み合っています。
マインドマップは、その絡まりを「ノード」と「つながり」で
目に見えるかたちに広げる手法です。

中心にテーマを置き、そこから枝分かれさせて、
関連する考えを親子の木構造として書き出していきます。
紙に書き出すだけで思考が整理される、あの感覚をソフトウェアにしたものです。

### 木構造の基本用語

マインドマップの骨格は「木（ツリー）」と呼ばれる構造です。
ここで使われる用語を整理しておきます。

```
        研究テーマ          ← 根（ルート）：木の起点。すべてはここから始まる
        ├── 仮説 A          ← 子（child）：直接ぶら下がるノード
        │   ├── 根拠 1      ← 孫。仮説 A の子であり、研究テーマの子孫
        │   └── 根拠 2
        └── 仮説 B
            └── 反例 1      ← 葉（リーフ）：その先に子を持たないノード
```

- **ノード** — ひとつの考えやメモを入れる箱。マップ上の丸や四角がこれにあたります
- **根（ルート）** — 木の一番上にあるノード。マインドマップでは中心テーマがこれです
- **親と子** — あるノードから枝分かれした先のノードが「子」、元のノードが「親」です。
  「仮説 A」は「研究テーマ」の子であり、「根拠 1」の親です
- **兄弟（きょうだい）** — 同じ親を持つノード同士。「仮説 A」と「仮説 B」は兄弟です
- **子孫と祖先** — 子、孫、ひ孫…とたどれるすべてが子孫。逆にさかのぼれるすべてが祖先です
- **葉（リーフ）** — 子を持たない末端のノード。これ以上枝分かれしていない考えです
- **深さ** — 根からの距離。根が深さ 0、その子が深さ 1、孫が深さ 2 です。
  深いほど具体的・詳細な考えになる傾向があります
- **折りたたみ** — あるノードの子孫をまとめて隠すこと。
  深い枝を一時的にしまうことで、今注目したい部分だけを見渡せます

M3E の操作はこの木構造を基本にしています。
「子を追加する」「兄弟を追加する」「折りたたむ」といった操作は、
すべてこの用語に対応しています。

ただし、ふつうのマインドマップツールは「きれいに整理すること」が目的になりがちです。
M3E が目指すのは、その先にあるものです。

## M3E が目指すもの

M3E は、科学研究のための思考支援ツールです。

研究では、論点を分解し、仮説を立て、前提を比較し、設計を判断する
という過程を繰り返します。この過程には構造があります。
しかし、頭の中だけでやっていると、構造はすぐに崩れ、
前提を見落とし、同じところを何度も回ってしまいます。

M3E は、この「構造を持った思考」を外に出し、目で見て、手で触れるようにします。
頭の中の世界モデルを画面上に構築し、可視化し、外在化する。
そうすることで、意思決定を加速できると考えています。

チャット型の AI は会話の流れで思考を進めますが、
構造は対話の中に埋もれてしまいます。
M3E は構造そのものを直接操作できるようにすることで、
AI との対話とは異なるかたちで思考を前に進めます。

## 3つの思考モード

人の思考には、場面によって異なる速度と粒度があります。
M3E はこれを 3 つのモードとして扱います。

- **Flash** — ひらめきや割り込みをすばやく捕まえるモード。
  構造より速度を優先し、思いついたことをまず置く場所です。
- **Rapid** — 日常の作業モード。
  ノードの追加・編集・並べ替えを軽快にこなし、考えを育てていきます。
- **Deep** — 設計レベルの深い思考モード。
  前提の明確化、比較の構造化、外部 AI に渡すための文脈整理など、
  じっくり取り組む場面で使います。

これらは同じデータを異なる密度で見るモードであり、
別々のファイルやデータベースではありません。
思考の段階に合わせて切り替えながら、ひとつの構造を育てていきます。

## M3E の設計思想

- **科学研究が第一** — 汎用メモツールではなく、研究の思考過程を支えることに集中します
- **構造は親子が主軸** — 思考の骨格は親子ツリーで表現します。補助的なつながりは骨格を壊さない範囲で扱います
- **スコープは認知境界** — フォルダのような整理箱ではなく、「今の思考に要らないものは見せない」ための仕組みです
- **AI は提案、人間が確定** — AI が勝手に内容を書き換えることはありません。提案は差分として見え、採否は常に人間が決めます
- **消えないことが最優先** — 便利さよりも、データが壊れないこと・消えないことを重視します

## はじめかた

### 1. セットアップ

`install\setup.bat` をダブルクリックしてください。

画面の案内に従うだけで、必要なものはすべて自動で準備されます。
Node.js などの事前インストールは不要です。

途中でデータの保存先を聞かれます。
特にこだわりがなければ Enter を押してデフォルトのままで構いません。

### 2. 起動

セットアップが完了すると、デスクトップに **M3E** ショートカットが作られます。
ダブルクリックするとブラウザが開き、すぐに使い始められます。

### 3. 基本操作

起動すると、キャンバス上にノードが表示されます。

- **ノードをクリック** して選択
- **Tab** で子ノードを追加
- **Enter** で兄弟ノードを追加
- **F2** または直接入力でノードのテキストを編集
- **Delete** で選択ノードを削除
- **スペース** でノードの折りたたみ/展開
- マウスドラッグでキャンバスを移動、スクロールで拡大縮小

画面右側のテキストエディタでは、インデント付きテキストとして
ツリー全体を一括編集することもできます。

### 4. ファイルの読み書き

- ツールバーの読み込みボタンから `.mm` ファイルを開けます
- 編集した内容は保存ボタンでエクスポートできます
- Freeplane の `.mm` ファイルをそのまま読み込めるので、既存の資産をそのまま使えます

### 5. 終了

ターミナルウィンドウを閉じるか、Ctrl+C で停止します。

## 設定

設定ファイルは `%LOCALAPPDATA%\M3E\m3e.conf` にあります。

```
M3E_DATA_DIR=D:\MyDocs\M3E
M3E_PORT=38482
```

設定をやり直したいときは `m3e.conf` を削除して `setup.bat` を再実行してください。

## 困ったとき

| 症状 | 対処 |
|------|------|
| セットアップ中にダウンロードが止まる | インターネット接続を確認して再実行 |
| ビルドに失敗する | `setup.bat` をもう一度実行 |
| ブラウザが開かない | `http://localhost:38482/viewer.html` を手動で開く |
| ポートが使用中と出る | ランチャーが自動で対処します。解決しない場合は PC を再起動 |
