# Todo Pool

## 目的

確定前の粗い TODO を一時プールし、正式タスク化前の取りこぼしを防ぐ。

## 運用ルール

1. 粗いメモでも登録してよい（未分解可）。
2. 実施完了の詳細は daily note に残す。
3. `Current_Status.md` には現在の重要項目のみ反映する。
4. 同じ項目が正式化されたら `Link` に反映先を記録する。

## 記法

- Date
- Topic
- Owner
- State (`pooled` | `ready` | `doing` | `blocked` | `done`)
- Link
- Note
