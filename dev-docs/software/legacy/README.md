# Legacy

このディレクトリには、現行方針では採用しない旧設計要件を置く。  
ここにある文書は歴史的な経緯の保管が目的であり、現行実装の基準ではない。

## 目的

- 以前の設計判断を失わない
- 現行文書との混在を避ける
- どこが obsolete になったかを明示する

## 現在 legacy 扱いするもの

- Canvas 中心でアプリ全体を構成する前提
- 無限平面を起点に UI と描画を同一レイヤーで考える前提
- 描画技術の理想形を初期段階から優先する前提
