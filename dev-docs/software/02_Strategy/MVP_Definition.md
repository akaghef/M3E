# MVP Definition

## MVP の目的

M3E の MVP は、独自 UI を完成させることではない。  
Freeplane 上の構造を入力として受け取り、科学研究の思考支援として有効な変換と提案を返せることを示す。

## MVP で必須のこと

### 1. 構造を読める

- Freeplane のマップ構造を読み取れる
- ノード階層を安定して取得できる
- 最低限の属性を扱える

### 2. コンテクストを作れる

- 選択ノード以下を自然言語化できる
- 研究論点・比較軸・前提・未確定点を抽出できる
- AI に渡しやすい形へ圧縮できる

### 3. 提案を返せる

- 追加ノード案
- 再整理案
- 要約案
- 比較観点案
- 仮説整理案
- 研究設計上の抜け漏れ指摘

### 4. 人間が採用判断できる

- 提案は差分として見える
- どこが変わるかが明確
- 承認前に正本は変わらない

## MVP で後回しにすること

- 高性能な独自レンダリング最適化
- 独自ドラッグ操作
- 高度なリアルタイム同期
- マルチユーザー
- 課金や公開サービス化
- 完全な Deep 差分履歴

## 独自実装へ進む場合の最小 UI 方針

- パネル、ツールバー、検索、設定は React コンポーネントで構築する
- マップ描画面は独立した rendering surface として切り出す
- rendering surface の内部実装は Canvas / SVG のいずれでもよい
- コアデータモデルと操作ロジックは React 非依存の TypeScript に置く

## 成功条件

- 日常の科学研究で繰り返し使える
- 提案が思考を前に進める
- 操作が不安定でない
- 正本データを壊さない

## 非目標

- Freeplane を置き換えること
- 最初から汎用マインドマップ製品を作ること
- あらゆるノートツールを同時統合すること
- 初期段階で研究以外の幅広い用途へ最適化すること
